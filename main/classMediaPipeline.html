Detailed Description

Data-driven builder and runtime for a DAG of MediaIO stages.

MediaPipeline consumes a MediaPipelineConfig, instantiates every stage as a MediaIO (either by registered backend name or by filesystem path), connects frameReady / frameWanted / writeError signals according to the declared routes, and drives frames from source stages to sink stages signal-driven on the owner's EventLoop — no dedicated pumper thread.

Lifecycle:

  • build — validates the config, creates every MediaIO, resolves file paths via MediaIO::createForFileRead / MediaIO::createForFileWrite.
  • open — opens each stage (topologically ordered so sinks are ready when sources emit their first frame).
  • start — wires signals and primes the drain so the first source read kicks off the flow.
  • stop — cancels pending commands, disconnects signals.
  • close — closes each stage in reverse topological order and releases the underlying MediaIO instances.

Fan-out is supported: a single source may appear as the from of multiple routes, and each destination back-pressures independently via MediaIO::writesAccepted and frameWanted. Fan-in on a single stage is not supported in this implementation (the first stage that would receive frames from more than one producer is rejected at build time).

◆ close()

Error MediaPipeline::close ( bool  block = true)

Closes every stage and releases the MediaIO instances.

Uses a graceful cascade: the true source stages (no upstream in the pipeline) are closed first via MediaIO::close with block=false. As each source's synthetic EOS reaches drainSource and latches upstreamDone, every direct downstream consumer of that source is closed in turn — so intermediate stages only see close() after all their input frames have been written to them. The pipeline emits closedSignal (and the repurposed finishedSignal) once every stage has emitted its own MediaIO::closedSignal.

Safe to call in any state:

  • Empty / Closed: no-op.
  • Built: stages are destroyed without opening.
  • Open / Stopped: stages are closed in parallel (no drain to propagate through).
  • Running: full cascade through the DAG.
Parameters
blockWhen true (default), blocks until the cascade has completed. When false, returns immediately and the caller learns completion through closedSignal.
Returns
Error::Ok on success or Error::Ok after successful async submit. Errors reported by individual stages are aggregated into the closedSignal payload rather than the return value; the return value only reflects up-front failures such as the pipeline already closing.

◆ injectStage()

Error MediaPipeline::injectStage ( const String &  name,
MediaIO *  io 
)

Registers an externally-constructed MediaIO for a stage that build would otherwise create.

Some backends cannot be instantiated through MediaIO::create because they need resources the library does not manage — the SDL player, for example, needs live SDLVideoWidget / SDLAudioOutput pointers. Callers pre-build those instances on the main thread and register them via injectStage before calling build. Injected stages are not deleted by the pipeline in close — ownership stays with the caller.

Must be called while the pipeline is State::Empty or State::Closed. When build sees the stage's name in the injection map it skips construction and uses the registered pointer instead.

Parameters
nameThe stage name declared in the config.
ioThe external MediaIO (must outlive the pipeline).
Returns
Error::Ok or Error::Busy when the pipeline is not in a settable state.

Emitted once every stage's MediaIO::closedSignal has fired and the pipeline has transitioned to State::Closed.

Fires for every completed close path — explicit close, natural EOF auto-cascade, and error-triggered teardown — so consumers have a single completion signal to wait on regardless of what drove the cascade. Emitted once the pipeline has finished draining AND every stage has fully closed.

Fires alongside closedSignal at the end of the cascade (natural EOF or explicit close). clean is true for natural EOF with no errors, false if an error interrupted the flow. Sinks have flushed their in-flight writes by this point — consumers that previously relied on finishedSignal firing as soon as sources hit EOF now see it after sink drain too.

◆ stop()

Error MediaPipeline::stop ( )

Cancels pending commands and halts the drain.

Pre-condition: the pipeline is State::Running.

Note
stop is a one-shot transition into State::Stopped — the terminal pre-close state. The pipeline cannot be re-started in place: signal connections were established against the live MediaIO instances at start time, and there is no mechanism today to disconnect and re-connect them. To run again, tear the pipeline down with close and rebuild via build / open / start.

◆ DefaultCloseTimeoutMs

constexpr unsigned int MediaPipeline::DefaultCloseTimeoutMs = 3000
staticconstexpr

Default close-watchdog timeout, milliseconds.

A clean close cascade relies on frames draining from each stage's input to its output. If a stage gets stuck on an external signal that never fires — a FrameBridge output waiting for a consumer that never arrives, for example — the cascade never reaches the stuck stage and the pipeline stalls indefinitely. The watchdog escalates to a forced close on every still-open stage once this deadline expires.
