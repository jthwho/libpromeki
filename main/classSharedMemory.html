Detailed Description

Cross-platform named shared memory region.

Wraps a named region of shared memory that can be shared between processes. One side creates the region (the owner) and the other side(s) open it by name. When the owner closes or destroys its SharedMemory, the region is unlinked from the system's name table; existing mappings remain valid until each holder unmaps it.

On POSIX the region is backed by shm_open (typically surfaced under /dev/shm on Linux). On other platforms support may be unavailable — see isSupported.

Naming
POSIX shared-memory names must begin with "/" and must not contain any further path separators. A leading slash is added automatically if missing; any embedded slash is rejected as Error::Invalid. Maximum length is platform-defined (SHM_NAME_MAX on BSDs; usually NAME_MAX on Linux).
Cross-user access
The owner passes a file mode to create (default 0600 — owner only). For cross-user access, use wider permissions such as 0660 and set groupName to a group both parties belong to. If groupName is non-empty, the underlying object is chown'd to that group immediately after creation; failures propagate.
Cleanup
If the owning process crashes, the region name is orphaned and remains in the system's name table until it is reused, explicitly unlinked (for example by an admin's "rm /dev/shm/foo"), or the system reboots. Holders that map the orphaned region continue to see a valid mapping until they unmap it.
Example
// Producer side.
SharedMemory shm;
Error err = shm.create("/my-region", 4096);
void *p = shm.data();
// ... write into p ...
// Consumer side (potentially in another process).
SharedMemory mapped;
err = mapped.open("/my-region", SharedMemory::ReadOnly);
const void *q = mapped.data();
// ... read from q ...

◆ unlink()

static Error SharedMemory::unlink ( const String &  name)
static

Removes a shared memory name from the system table.

Intended for higher-level code that has independently determined a prior owner is no longer live and needs to recycle the name before a fresh create. Equivalent to shm_unlink on POSIX. Returns Error::Ok when the name is gone afterwards (including when it never existed) and an error only if the unlink itself failed for a non-ENOENT reason.

Existing mappings in this or other processes remain valid until each holder unmaps them — only the name is removed.

nameRegion name (leading "/" added if missing).
