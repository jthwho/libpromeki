Detailed Description

Per-channel (AudioStreamDesc, ChannelRole) assignment.

AudioChannelMap is an ordered list of channels, with each channel carrying both a ChannelRole (positional / functional role) and an AudioStreamDesc (the logical stream the channel belongs to). The pairing lets a single audio block carry multiple program streams simultaneously — a 5.1 Main bed alongside a Commentary mono channel, for instance — without needing one buffer per stream.

Construction

Several constructors are offered for common shapes:

  • From a ChannelRole list (or std::initializer_list) — every channel takes the AudioStreamDesc::Undefined stream.
  • From a (stream, roles) pair — every channel takes the same explicit stream.
  • From a list of fully-formed Entry pairs — full per-channel control over both stream and role.

Convenience factories defaultForChannels seed the canonical layouts (Mono / Stereo / 5.1 / 7.1 / 7.1.4 / FOA) for both the Undefined-stream case and a caller-supplied stream.

String form

A map round-trips through a per-channel comma-separated string:

"Stereo" // single Undefined-stream well-known
"Main:5.1" // single explicit stream well-known
"FrontLeft,FrontRight" // explicit per-channel, all Undefined
"Main:FrontLeft,Main:FrontRight,Commentary:Mono" // mixed streams
"FrontLeft,Commentary:Mono" // mixed streams (first Undefined)

Each per-channel item is either "StreamName:RoleName" or just "RoleName" (in which case the stream is Undefined). toString prefers the well-known shortcut when the map matches one (single stream throughout, role list matches a well-known layout), falling back to the per-channel comma form otherwise.

Equality

Two maps compare equal when they are the same length and every entry's stream and role match in the same order.

Thread Safety

Distinct instances are independent; concurrent mutation of one instance must be externally synchronized.

◆ defaultForChannels() [1/2]

static AudioChannelMap promeki::AudioChannelMap::defaultForChannels ( size_t  channels)
static

Returns the canonical layout for channels.

Returns the first entry in wellKnownLayouts whose role list has channels entries. Several channel counts have more than one well-known layout — the registration order in the dictionary defines which layout wins:

  • 1 → Mono
  • 2 → Stereo
  • 3 → 2.1 (before 3.0)
  • 4 → Quad (before 4.0, FOA)
  • 5 → 5.0
  • 6 → 5.1 (before 6.0)
  • 7 → 6.1 (before 7.0)
  • 8 → 7.1 (before 5.1.2)
  • 10 → 5.1.4 (before 7.1.2)
  • 12 → 7.1.4

Counts not listed return channels Undefined-stream / ChannelRole::Unused entries — the caller is expected to fill in roles explicitly.

Callers that need a non-default layout (e.g. "3.0" for a 3-channel buffer or "5.1.2" for 8 channels) must build the map explicitly via fromString or the AudioChannelMap constructors rather than relying on this helper.

Every entry in the result uses the AudioStreamDesc::Undefined stream.

◆ fromString()

static Result< AudioChannelMap > promeki::AudioChannelMap::fromString ( const String &  str)
static

Parses a string into an AudioChannelMap.

Accepted forms:

  • Well-known layout shortcut: "Stereo", "5.1", "7.1.4", "FOA", … — every channel takes AudioStreamDesc::Undefined.
  • Stream-prefixed well-known shortcut: "Main:5.1", "Commentary:Mono", … — every channel takes the named stream.
  • Per-channel comma list: "FrontLeft,FrontRight" or "Main:FrontLeft,Main:FrontRight,Commentary:Mono". Each item is "StreamName:RoleName" or just "RoleName" (Undefined stream).

Whitespace around commas / colons is permitted.
