Detailed Description

Generic name/value configuration container for the media subsystem.

Thin subclass of VariantDatabase<MediaConfigTag> that adds a canonical catalog of well-known ID constants for every configurable knob in libpromeki. All media components — MediaIO backends, Image::convert, CSCPipeline — share this single registry so:

  • A key set at one layer can be forwarded to another without any translation or key mapping.
  • There is exactly one place to look up the string name, default value type, and semantic of a knob.
  • New callers get autocomplete and a compile-time spelling check on every key they touch.

Follows the same layout as Metadata: a class inheriting from VariantDatabase so standard key IDs can live as static inline const members while instance methods (set / get / contains / merge / JSON + stream serialization) are inherited from the base unchanged.

Example
MediaConfig cfg;
cfg.set(MediaConfig::Filename, String("/tmp/clip.mov"));
cfg.set(MediaConfig::VideoSize, Size2Du32(1920, 1080));
cfg.set(MediaConfig::JpegQuality, 95);
cfg.set(MediaConfig::JpegSubsampling, ChromaSubsampling::YUV444);
Image jpeg = rgb.convert(PixelDesc(PixelDesc::JPEG_RGB8_sRGB),
rgb.metadata(), cfg);

const ID MediaConfig::RtpSdp {"RtpSdp"}
inlinestatic

Polymorphic reader-side SDP input. Accepts either:

  • String: interpreted as a filesystem path. The MediaIO calls SdpSession::fromFile at open time.
  • SdpSession: consumed directly, no filesystem access. The MediaIO populates per-stream destinations, payload types, clock rates, and (for JPEG XS) geometry from the m= / a=rtpmap / a=fmtp lines. Explicit per-stream config keys still override anything discovered via SDP.
