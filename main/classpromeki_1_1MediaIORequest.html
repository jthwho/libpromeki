Detailed Description

Asynchronous handle for a MediaIO operation.

Every public operation on MediaIO, MediaIOSource, MediaIOSink, and MediaIOPortGroup that goes through the strand returns a MediaIORequest — a thin facade over the underlying MediaIOCommand. The request adds no completion state of its own; it just forwards wait / isReady / then / cancel to the command's own MediaIOCommand::waitForCompletion / MediaIOCommand::isCompleted / MediaIOCommand::setCompletionCallback / MediaIOCommand::cancelled hooks.

Why non-templated
Every command carries an Error in MediaIOCommand::result, so the request just proxies that — there is one canonical "outcome" type and no need to template over a payload. Typed Output fields (e.g. MediaIOCommandRead::frame, MediaIOCommandSeek::currentFrame, MediaIOCommandParams::output) live on the concrete command subclass and are reached via commandAs<MediaIOCommandRead>().
Lifetime
MediaIORequest is a value type — it holds a MediaIOCommand::Ptr (a shared pointer) plus a small Error sentinel for the resolved(Error) factory path. Copying the handle copies the shared pointer. When every handle goes away, the command is freed (the strand keeps its own reference until the command runs, so dropping every caller-side handle is safe).
Cancellation
cancel() is best-effort and has three observable behaviors depending on lifecycle stage:
  1. Not yet dispatched — the strategy completes the command immediately with Error::Cancelled; wait returns the cancellation. The backend's executeCmd is never called.
  2. Dispatched and executing on a non-blocking strategy (InlineMediaIO, SharedThreadMediaIO) — the strategy ignores late cancellation requests and lets the command complete normally. Cancellation is not preemptive on shared-pool workers because doing so would require backend cooperation we cannot rely on.
  3. Dispatched and executing on DedicatedThreadMediaIO — the strategy invokes the backend's cancelBlockingWork() virtual on a thread distinct from the worker. Backends override cancelBlockingWork() to interrupt syscalls (close fds, signal condvars, set a shutdown flag, etc.).
Callback marshalling
then captures the calling thread's EventLoop at the time of registration and dispatches the callback via that loop's postCallable when the request resolves. When then is called on a thread without an active EventLoop, the callback runs synchronously on the thread that resolves the request (the strategy worker thread). When then is called after the request has already resolved, the callback fires immediately (still marshalled through the captured loop, if any).

template<typename CmdT >
const CmdT * promeki::MediaIORequest::commandAs ( ) const
inline

Returns the underlying command typed as CmdT.

Returns a raw pointer because MediaIOCommand uses the SharedPtrProxy path, so a shared dynamic-cast across the hierarchy isn't available. The returned pointer's lifetime is tied to this MediaIORequest handle (or another copy of it) — keep a request alive as long as the pointer is in use.

Unchecked static_cast — callers built the command knowing its concrete type, so a runtime mismatch is a programmer error rather than a recoverable condition. Returns nullptr when the request has no command (the resolved(Error) sentinel path).

MediaIORequest req = source->readFrame();
if (req.wait().isOk()) {
Frame::Ptr frame = req.commandAs<MediaIOCommandRead>()->frame;
}

Builds a request that is already resolved with the result baked into cmd.

Useful for short-circuit returns where the public API can fully service the request without going through the strand and wants to surface a typed output. The caller pre-populates cmd's MediaIOCommand::result (and any Output fields), then this factory marks the command completed so subsequent wait / then calls fire immediately.

◆ stats()

MediaIOStats promeki::MediaIORequest::stats ( ) const
inline

Telemetry snapshot for this request.

Returns the MediaIOStats container the framework + backend populated for this command. Same accessor for both contexts:

  • For standard commands (open/close/read/write/ seek/params): per-command keys (dispatch duration, queue wait, bytes processed, plus any backend-specific keys).
  • For MediaIOCommandStats — instance-wide cumulative aggregate (frames dropped, bytes per second, queue depth, etc.).

Returns an empty container when the request has no command attached (the resolved(Error) sentinel path).

◆ wait()

Error promeki::MediaIORequest::wait ( unsigned int  timeoutMs = 0)
inline

Blocks until completion or timeoutMs elapses.

Returns Error::Invalid when the request is empty (default-constructed), Error::Timeout when the deadline expired before the command resolved, or the underlying command's MediaIOCommand::result on completion. When the request was built from resolved(Error err) (no command), err is returned immediately regardless of timeoutMs.

Parameters
timeoutMsMaximum time to wait in milliseconds. A value of 0 (the default) waits indefinitely.
