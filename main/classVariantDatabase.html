Detailed Description

template<typename Tag>
class VariantDatabase< Tag >

A collection of named Variant values keyed by string-registered IDs.

Template Parameters
TagA tag type that distinguishes this database's ID namespace. The Tag also serves as the StringRegistry tag, so each VariantDatabase type gets its own independent ID space.

VariantDatabase maps string names to Variant values using integer IDs for fast lookup. The nested ID type is a StringRegistry::Item scoped to this database's Tag.

Supports serialization to/from JSON, TextStream, and DataStream. All serialization formats use the string names (not integer IDs), so data can be safely persisted and loaded across runs.

Warning
The integer IDs within ID are assigned in registration order and must not be persisted or used outside the current process.
Example
struct ConfigTag {};
using Config = VariantDatabase<ConfigTag>;
Config cfg;
Config::ID width("video.width");
Config::ID height("video.height");
cfg.set(width, 1920);
cfg.set(height, 1080);
int w = cfg.get(width).get<int32_t>(); // 1920
bool has = cfg.contains(width); // true
// Serialize to JSON
JsonObject json = cfg.toJson();
// Deserialize from JSON
Config cfg2 = Config::fromJson(json);
