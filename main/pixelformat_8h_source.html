336 // -- Full-range uncompressed YCbCr --
337 //
338 // The library-wide YCbCr naming convention is that
339 // the unsuffixed form (e.g. YUV8_422_Rec709) is
340 // limited-range (16..235 Y, 16..240 Cb/Cr) to match
341 // broadcast / SDI / ST 2110 defaults. The explicit
342 // "_Full" suffix opts in to full-range 0..255 Y and
343 // 0..255 Cb/Cr, which is what JPEG / JFIF uses and
344 // what most consumer video decoders (ffplay,
345 // browsers, libjpeg-turbo) expect. These
346 // PixelFormats exist both as general-purpose
347 // full-range YCbCr storage and as encode-source
348 // intermediates for the full-range JPEG variants
349 // below.

355 // -- Full complement of JPEG YCbCr variants (matrix × range) --
356 //
357 // Following the library-wide YCbCr convention: the
358 // unsuffixed JPEG names (existing
359 // JPEG_YUV8_422_Rec709 / JPEG_YUV8_420_Rec709) are
360 // limited-range to stay consistent with the
361 // uncompressed YCbCr defaults. Full-range variants
362 // take the explicit "_Full" suffix. All eight
363 // combinations (matrix × range × subsampling) are
364 // provided; pick the one that matches what the
365 // downstream decoder expects to interpret:
366 //
367 // - Rec.601 + full range (strict JFIF) → ffplay,
368 // browsers, libjpeg-turbo, most consumer apps
369 // - Rec.709 + full range → modern cameras with
370 // an ICC profile; good for high-quality still
371 // storage that a colour-managed viewer honours
372 // - Rec.709 + limited range (existing default) →
373 // broadcast / SDI JPEG pipelines (ST 2110 JPEG XS
374 // style)
375 // - Rec.601 + limited range → legacy broadcast
376 JPEG_YUV8_422_Rec601 = 147,
