Detailed Description

template<typename T>
class promeki::Span< T >

Non-owning view over contiguous storage, wrapping std::span.

Provides a Qt-inspired API over std::span with consistent naming conventions matching the rest of libpromeki. Non-owning view — no PROMEKI_SHARED_FINAL.

Thread Safety
Distinct Span instances may be used concurrently. A Span itself is just (pointer, size); its const operations are thread-safe. Concurrent access to the underlying storage that the Span views follows the storage's own thread-safety contract — Span does not provide synchronization for the data it points at.
