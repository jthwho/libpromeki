Detailed Description

Abstract base class for stateful video encoders.

VideoEncoder is the push-frame / pull-packet counterpart to ImageCodec for codecs that are fundamentally temporal — H.264, HEVC, AV1, VP9, ProRes, and so on. Unlike ImageCodec, which treats every image as an independent round-trip, VideoEncoder holds a long-lived session: frames submitted via submitFrame feed an internal pipeline, and encoded output comes back out of receivePacket zero, one, or many frames later depending on the codec's coding structure and configured look-ahead.

Session lifecycle
  1. Create an instance via createEncoder.
  2. Call configure with a MediaConfig holding bitrate, GOP length, preset, and any other well-known knobs.
  3. For each source frame, call submitFrame.
  4. After each submit (and any time after), drain with receivePacket until it returns a null Ptr. Packets may arrive with PTS out of order (B-frames); the DTS on each packet reflects decode order.
  5. When the input stream is exhausted, call flush and keep draining until receivePacket returns a packet with the MediaPacket::EndOfStream flag set.
  6. Destroy the encoder.

Implementations are not required to be thread-safe. Each pipeline thread should own its own encoder instance; typical use is one encoder per stream.

Example
VideoEncoder *enc = VideoEncoder::createEncoder("H264");
MediaConfig cfg;
cfg.set(MediaConfig::BitrateKbps, 8000);
cfg.set(MediaConfig::VideoRcMode, VideoRateControl::CBR);
cfg.set(MediaConfig::GopLength, 60);
cfg.set(MediaConfig::VideoPreset, VideoEncoderPreset::LowLatency);
enc->configure(cfg);
for(const Image &frame : source) {
enc->submitFrame(frame);
while(auto pkt = enc->receivePacket()) {
sink.write(pkt);
}
}
enc->flush();
while(auto pkt = enc->receivePacket()) {
sink.write(pkt);
if(pkt->isEndOfStream()) break;
}
delete enc;

◆ configure()

virtual void VideoEncoder::configure ( const MediaConfig &  config)
virtual

Applies encoder parameters from a MediaConfig.

Each backend reads the well-known keys it understands (e.g. BitrateKbps, VideoRcMode, GopLength, VideoPreset) and silently ignores the rest, so the same MediaConfig can be reused across pipeline stages without filtering. Calling configure() while an encoder session is active is allowed; the backend applies what it can at runtime and may defer the rest to the next IDR. The default implementation is a no-op.

Parameters
configCaller-supplied configuration database.

Reimplemented in JpegVideoEncoder, and JpegXsVideoEncoder.

virtual Error VideoEncoder::flush ( )

Signals end-of-stream to the encoder and asks it to emit remaining packets.

After flush(), the caller drains with receivePacket until it sees a packet with the MediaPacket::EndOfStream flag or a null Ptr. After the flush completes, the encoder may be reused for a fresh stream by calling reset followed by configure; or it may be destroyed.

◆ submitFrame()

virtual Error VideoEncoder::submitFrame ( const Image &  frame,
const MediaTimeStamp &  pts = MediaTimeStamp() 
)
pure virtual

Submits one uncompressed frame for encoding.

The encoder takes a logical copy of what it needs from frame before returning; the caller may reuse or release the frame afterward. The pts is recorded in the encoder's internal queue so that the matching output packet can carry it back out. Submitting does not guarantee a packet will be available immediately — B-frames and look-ahead both introduce output delay.

Parameters
frameThe source image in one of the supported uncompressed pixel formats.
ptsPresentation timestamp for this frame. May be invalid if the pipeline does not care about presentation ordering.
Returns
Error::Ok on success, or an error code describing why the frame was rejected.

Implemented in JpegVideoEncoder, and JpegXsVideoEncoder.
