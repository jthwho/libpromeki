◆ PROMEKI_SHARED_ABSTRACT

#define PROMEKI_SHARED_ABSTRACT (   INTERMEDIATE)
Value:
public: \
INTERMEDIATE *_promeki_clone() const override = 0;

Macro for abstract intermediate classes in a shared hierarchy.

Use this on a class that derives from a base already marked with PROMEKI_SHARED (or a hand-rolled equivalent) and that stays abstract — concrete leaves below supply the actual clone. The macro adds a covariant pure-virtual override of _promeki_clone so SharedPtr<INTERMEDIATE>::modify() detaches directly into an INTERMEDIATE pointer without a downcast, and sharedPointerCast<INTERMEDIATE>(basePtr) yields a typed Ptr whose modify() is valid without further work at the call site.

Example:

class VideoPayload : public MediaPayload {
PROMEKI_SHARED_ABSTRACT(VideoPayload)
public:
/ concrete leaves provide _promeki_clone
};

Macro for non-copyable derived shared classes.

Same role as PROMEKI_SHARED_DERIVED, but the virtual _promeki_clone override aborts instead of copy-constructing. Use this when DERIVED adds non-copyable state (e.g. an owned Promise, std::atomic, or UniquePtr) on top of a shared base. As with PROMEKI_SHARED_NOCOPY, the matching SharedPtr<BASE, false> (CoW disabled) is required so the abort path is never taken at runtime.

Macro for non-polymorphic native shared objects.

Use this instead of PROMEKI_SHARED when the class will never be subclassed (e.g. private inner Data classes). It avoids the overhead of a vtable pointer by using non-virtual _promeki_clone(). The class does not need a virtual destructor.

Example:

class Data {
PROMEKI_SHARED_FINAL(Data)
public:
int value = 0;
};

Macro for non-polymorphic, non-copyable native shared objects.

Same role as PROMEKI_SHARED_FINAL, but the _promeki_clone stub aborts instead of copy-constructing. Use this when TYPE holds a member that cannot be copied (e.g. Promise<T>, std::atomic-only state, UniquePtr) — the macro-generated copy-clone path would otherwise fail to compile.

Callers must use SharedPtr<TYPE, false> (CoW disabled); the default SharedPtr<TYPE> path triggers _promeki_clone inside modify() and would hit the abort at runtime.

Example:

struct Pending {
PROMEKI_SHARED_FINAL_NOCOPY(Pending)
Promise<HttpResponse> promise; // non-copyable
// ...
};
using PendingPtr = SharedPtr<Pending, false>;

Macro for non-copyable polymorphic shared base classes.

Same role as PROMEKI_SHARED, but the virtual _promeki_clone stub aborts instead of copy-constructing — pick this when BASE (or any of its members) is genuinely non-copyable (e.g. holds a Promise<T>, UniquePtr, or std::atomic-only state) and copy-on-write semantics therefore cannot be supported.

Callers must use SharedPtr<BASE, false> (CoW disabled); derived classes that are copyable can override the clone via PROMEKI_SHARED_DERIVED, otherwise pair this with PROMEKI_SHARED_DERIVED_NOCOPY in the leaves.
