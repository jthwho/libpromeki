Detailed Description

Abstract polymorphic base for any encoded bitstream packet.

A MediaPacket represents one compressed access unit (AU) flowing through a pipeline. The common fields — a byte range on a shared Buffer, presentation / decode timestamps, nominal duration, and a Metadata container — apply to every kind of packet we model; codec-identity, keyframe/parameter-set/etc. flags, sample counts, and other per-family properties live on concrete subclasses such as VideoPacket, AudioPacket, and future Ancillary/SDI/ST2110 packet types.

Subclassing

Subclasses use PROMEKI_SHARED_DERIVED to participate in SharedPtr-based ownership, override kind() to identify themselves for cheap dispatch without RTTI, and override clone() to return the correct derived type during copy-on-write. Concrete subclasses are not marked final — further specialization (for example, H264VideoPacket or MpegVideoPacket deriving from VideoPacket) is an intentional extension point.

Stream-level metadata

End-of-stream and corruption marking are carried in metadata() rather than as per-packet flags so that the same keys work uniformly across packets, Frame, Image, and Audio:

  • Metadata::EndOfStream (bool) — terminal packet in the stream.
  • Metadata::Corrupt (bool) — payload is known to be corrupt.
  • Metadata::CorruptReason (String) — human-readable explanation.

Convenience accessors isEndOfStream, isCorrupt, corruptReason, markEndOfStream, and markCorrupt are provided so callers don't need to remember the keys.

Ownership

MediaPacket uses the library's native shared-object pattern (PROMEKI_SHARED). Callers manage lifetime via MediaPacket::Ptr . Polymorphic construction goes through takeOwnership:

MediaPacket::Ptr pkt = MediaPacket::Ptr::takeOwnership(
new VideoPacket(view, PixelFormat::H264));
