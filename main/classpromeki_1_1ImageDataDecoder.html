Detailed Description

Companion decoder for ImageDataEncoder.

Reads back the 64-bit payloads written by ImageDataEncoder from a raster UncompressedVideoPayload, with enough robustness to survive compression, scaling, and other "real-world" image-pipeline indignities. See Image Data Encoder Wire Format for the wire format specification — anything described there is normative; this class is the reference implementation of a compatible reader.

Algorithm
For each band the caller hands in:
  1. The relevant scan lines are extracted from the source image and converted to RGBA8_sRGB via the libpromeki CSC pipeline. This is the slow but format-agnostic step; future revisions can add hand-rolled fast paths for the common formats and bypass the CSC entirely.
  2. The R channel of the converted scan-line strip is vertically averaged into a single 1D row of length imageWidth. Multi-line averaging is essentially free SNR: a 16-line band gives sqrt(16) = 4× improvement against Gaussian-style noise. Single-line mode (MiddleLine) is available as a faster, less robust alternative.
  3. The averaged row is binarised against a threshold computed using Otsu's method — the textbook variance-minimising threshold for bimodal signals — so the decoder picks up whatever white / black levels actually exist in the channel after compression has had its way with them.
  4. The sync nibble at the start of the binarised row supplies the bit pitch. Its four runs are measured and averaged into a sub-pixel-accurate bitWidth value. The decoder validates the result against imageWidth/76 ± 50% and rejects rows that fall outside that band as Error::CorruptData.
  5. The 76 bit cells are sampled at their centres at the established pitch, the sync nibble is verified, the CRC is recomputed against the catalogued CRC-8/AUTOSAR parameters, and the payload is returned.
Lifetime and reuse
Construct one decoder per (PixelFormat, image dimensions) pair and reuse it across many frames — the per-decode allocation is just a scratch RGBA8 conversion buffer, which gets re-allocated lazily for each band.
Example
ImageDataDecoder dec(img.desc());
if(!dec.isValid()) {
// Image too narrow for any 76-cell row at this format's
// alignment quantum.
}
List<ImageDataDecoder::Band> bands;
bands.pushToBack({ 0, 16 }); // first 16 lines
bands.pushToBack({ 16, 16 }); // next 16 lines
List<ImageDataDecoder::DecodedItem> items;
dec.decode(img, bands, items);
for(const auto &it : items) {
if(it.error.isOk()) {
// it.payload is the decoded 64-bit value.
}
}

enum class promeki::ImageDataDecoder::SampleMode
strong

How the decoder samples the image rows in a band.

Enumerator
AverageBand 

Convert every scan line in the band and vertically average the R channel. Default — gives sqrt(N) SNR improvement on noisy / compressed input.

MiddleLine 

Convert just the middle scan line of the band. Faster but does not benefit from multi-line averaging.

Decodes a list of bands from an image.

out is cleared before decoding starts. After the call returns, out contains exactly bands.size() DecodedItem entries — one per supplied band, in the same order, including failed ones. Inspect each item's error field to tell success from failure.

Parameters
payloadPayload to read from. Must have the same descriptor as the one supplied to the constructor.
bandsBands to decode.
outOutput list of decoded items.
Returns
Error::Ok on success (every band attempted and the result list is fully populated), or an error code if the payload descriptor does not match the decoder.
