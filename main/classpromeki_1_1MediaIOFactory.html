Detailed Description

Polymorphic factory that creates MediaIO instances for a given backend identity.

Each backend declares one concrete XxxFactory subclass that returns identity, role flags, config specs, discovery surfaces, and constructs the matching MediaIO instance from a MediaConfig. Lookup is name- / extension- / scheme- / path-based (see the findBy* statics).

Lifetime
Factories are owned by the registry and live for the entire process. The single instance per backend is allocated by PROMEKI_REGISTER_MEDIAIO_FACTORY at static init time and deleted at process exit.
Thread safety
Factory implementations must be thread-safe — multiple threads may query the registry concurrently. Practically every override is a pure function returning a constant, so this is trivially satisfied by stateless subclasses.

Returns the names of config keys in cfg that are not recognized by the backend or the global registry.

Walks cfg's stored keys and filters out any that have a VariantSpec — either in the backend's spec map or in the global MediaConfig spec registry. The result is the set of leftover key names, sorted lexicographically for stable logging.

Pure detection primitive. Callers that want a ready-made error policy should use validateConfigKeys instead.

Parameters
typeNameThe registered backend name. When the backend is unknown or has no spec map, only the global registry is consulted.
cfgThe configuration to check.
Returns
The sorted list of unrecognized key names; an empty list when every key is recognized.

)
static

Validates a config's key set against the registered specs.

Calls unknownConfigKeys and then applies the error policy selected by mode:

  • ConfigValidation::Lenient — emit a warning for each unknown key and return Error::Ok.
  • ConfigValidation::Strict — emit a warning for each unknown key and return Error::InvalidArgument.

Warnings are emitted via promekiWarn and include the optional contextLabel so callers can make each line self-identifying (e.g. "mediaplay: input[TPG]").

Parameters
typeNameThe registered backend name.
cfgThe configuration to check.
modeThe error policy to apply.
contextLabelOptional prefix for log messages.
Returns
Error::Ok or Error::InvalidArgument per mode. Always Error::Ok when there are no unknown keys.
