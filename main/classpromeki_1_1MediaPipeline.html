Detailed Description

Data-driven builder and runtime for a DAG of MediaIO stages.

MediaPipeline consumes a MediaPipelineConfig, instantiates every stage as a MediaIO (either by registered backend name or by filesystem path), connects frameReady / frameWanted / writeError signals according to the declared routes, and drives frames from source stages to sink stages signal-driven on the owner's EventLoop — no dedicated pumper thread.

Lifecycle:

  • build — validates the config, creates every MediaIO, resolves file paths via MediaIO::createForFileRead / MediaIO::createForFileWrite.
  • open — opens each stage (topologically ordered so sinks are ready when sources emit their first frame).
  • start — wires signals and primes the drain so the first source read kicks off the flow.
  • stop — cancels pending commands, disconnects signals.
  • close — closes each stage in reverse topological order and releases the underlying MediaIO instances.

Fan-out is supported: a single source may appear as the from of multiple routes, and each destination back-pressures independently via MediaIOSink::writesAccepted and frameWanted. Fan-in on a single stage is not supported in this implementation (the first stage that would receive frames from more than one producer is rejected at build time).

Thread Safety
Inherits ObjectBase — thread-affine. Pipeline lifecycle (build, open, start, stop, close) is driven from the owning EventLoop's thread. Per-stage execution happens on the strand worker that the stage's MediaIO uses; the pipeline coordinates them via signal-driven notifications.

◆ close()

Error promeki::MediaPipeline::close ( bool  block = true)

Closes every stage and releases the MediaIO instances.

Uses a graceful cascade: the true source stages (no upstream in the pipeline) are closed first via MediaIO::close with block=false. As each source's synthetic EOS reaches drainSource and latches upstreamDone, every direct downstream consumer of that source is closed in turn — so intermediate stages only see close() after all their input frames have been written to them. The pipeline emits closedSignal (and the repurposed finishedSignal) once every stage has emitted its own MediaIO::closedSignal.

Safe to call in any state:

  • Empty / Closed: no-op.
  • Built: stages are destroyed without opening.
  • Open / Stopped: stages are closed in parallel (no drain to propagate through).
  • Running: full cascade through the DAG.
Parameters
blockWhen true (default), blocks until the cascade has completed. When false, returns immediately and the caller learns completion through closedSignal.
Returns
Error::Ok on success or Error::Ok after successful async submit. Errors reported by individual stages are aggregated into the closedSignal payload rather than the return value; the return value only reflects up-front failures such as the pipeline already closing.

◆ injectStage()

Error promeki::MediaPipeline::injectStage ( MediaIO *  io)

Registers an externally-constructed MediaIO for a stage that build would otherwise create.

Some backends cannot be instantiated through MediaIO::create because they need resources the library does not manage — the SDL player, for example, needs live SDLVideoWidget / SDLAudioOutput pointers. Callers pre-build those instances on the main thread and register them via injectStage before calling build. Injected stages are not deleted by the pipeline in close — ownership stays with the caller.

Must be called while the pipeline is State::Empty or State::Closed. When build sees a stage whose name matches an injected entry it skips factory construction and uses the registered pointer instead.

Name resolution
Picks the stage name in the following order, then stamps it onto the IO via MediaIO::setName so io->name() reports the resolved name on return:
  1. If io->name() is non-empty and does not collide with an already-injected stage, use it verbatim.
  2. If io->name() is non-empty and collides, append an incrementing suffix (e.g. "sdl", "sdl2", "sdl3", ...).
  3. If io->name() is empty, generate a role-based default: "src&lt;N&gt;", "sink&lt;N&gt;", or "xfm&lt;N&gt;" when the role is determinable from MediaIO::isSource / MediaIO::isSink (post-open) or the MediaIO::describe probe (pre-open via the registered factory's role flags). Falls back to "stage&lt;N&gt;" when the role is unknown.
Parameters
ioThe external MediaIO (must outlive the pipeline).
Returns
Error::Ok or Error::Busy when the pipeline is not in a settable state, or Error::InvalidArgument when io is null.

◆ setStatsInterval()

void promeki::MediaPipeline::setStatsInterval ( Duration  interval)

Configures the periodic stats tick.

When interval is Duration::zero (default) the tick is disabled and neither statsUpdatedSignal nor PipelineEvent::Kind::StatsUpdated fires. A positive interval is rounded up to the nearest millisecond and starts a millisecond-precision timer on this object's EventLoop while the pipeline is State::Running. The timer is stopped during stop / close and restarted on the next start when interval is still non-zero.

Safe to call from any state.

Parameters
intervalTick interval (zero disables).

◆ stats()

MediaPipelineStats promeki::MediaPipeline::stats ( )

Collects a stats snapshot from every live stage.

For each stage the snapshot contains:

  • the cumulative aggregate returned by MediaIO::stats (instance-wide standard keys plus any backend-specific cumulative keys); and
  • the per-MediaIOCommand::Kind windowed breakdown sourced from the stage's MediaIOStatsCollector when stats collection is enabled (see MediaPipelineConfig::statsWindowSize).

Stages whose underlying MediaIO is not open appear as records with a default-constructed cumulative and an empty windowed map.

◆ stop()

Error promeki::MediaPipeline::stop ( )

Cancels pending commands and halts the drain.

Pre-condition: the pipeline is State::Running.

Note
stop is a one-shot transition into State::Stopped — the terminal pre-close state. The pipeline cannot be re-started in place: signal connections were established against the live MediaIO instances at start time, and there is no mechanism today to disconnect and re-connect them. To run again, tear the pipeline down with close and rebuild via build / open / start.

◆ subscribe()

int promeki::MediaPipeline::subscribe ( EventCallback  cb)

Registers cb to receive every future PipelineEvent.

The subscriber's EventLoop is captured at the call site (EventLoop::current). Each event is dispatched by posting a callable onto that loop, so callbacks always run on the subscriber's thread regardless of which thread triggered the underlying pipeline transition.

Subscriptions can be installed in any pipeline state and survive across build / open / start / stop / close cycles. The first subscription lazily installs a Logger listener so subsequent promekiInfo / promekiWarn / promekiErr calls are mirrored as PipelineEvent::Kind::Log events; the listener is removed when the last subscriber leaves.

Parameters
cbCallback invoked once per event. An empty callback is rejected and yields -1.
Returns
Non-negative subscription id usable with unsubscribe, or -1 when cb was empty or no EventLoop is available on the calling thread.

◆ closedSignal

Signal< Error > promeki::MediaPipeline::closedSignal

Emitted once every stage's MediaIO::closedSignal has fired and the pipeline has transitioned to State::Closed.

Fires for every completed close path — explicit close, natural EOF auto-cascade, and error-triggered teardown — so consumers have a single completion signal to wait on regardless of what drove the cascade. The payload is the first non-Ok per-stage close error, or Error::Ok when every stage closed cleanly.

◆ DefaultCloseTimeoutMs

constexpr unsigned int promeki::MediaPipeline::DefaultCloseTimeoutMs = 3000
staticconstexpr

Default close-watchdog timeout, milliseconds.

A clean close cascade relies on frames draining from each stage's input to its output. If a stage gets stuck on an external signal that never fires — a FrameBridge output waiting for a consumer that never arrives, for example — the cascade never reaches the stuck stage and the pipeline stalls indefinitely. The watchdog escalates to a forced close on every still-open stage once this deadline expires.

◆ finishedSignal

Signal< bool > promeki::MediaPipeline::finishedSignal

Emitted once the pipeline has finished draining AND every stage has fully closed.

Fires alongside closedSignal at the end of the cascade (natural EOF or explicit close). clean is true for natural EOF with no errors, false if an error interrupted the flow. Sinks have flushed their in-flight writes by this point — consumers that previously relied on finishedSignal firing as soon as sources hit EOF now see it after sink drain too.
