Detailed Description

Statistics container for MediaIO backends.

A distinct VariantDatabase type for runtime stats reported by backends. Has its own StringRegistry so stats keys don't collide with config or param keys. Backends populate instances of this in executeCmd(MediaIOCommandStats&); users receive them from MediaIO::stats().

Standard key names are defined as static members; backends are free to add their own backend-specific keys for data not covered by the standard set.

Member Function Documentation

◆ toString()

String MediaIOStats::toString ( ) const

Renders the standard telemetry keys as a compact log line.

Formats whichever standard keys are present into a single space-separated line suitable for dumping to a terminal or logger. Keys are emitted in a fixed order so periodic telemetry output stays scannable:

BytesPerSecond FramesPerSecond FramesDropped FramesRepeated FramesLate AverageLatencyMs/PeakLatencyMs QueueDepth/QueueCapacity PendingOperations LastErrorMessage

Counters that are still zero (FramesRepeated, FramesLate) and the empty LastErrorMessage are elided to keep the line quiet under normal operation. Backend-specific keys are not included — callers that want a full dump should iterate via forEach.

Returns
A single-line summary, or an empty String if the container holds no standard keys.
