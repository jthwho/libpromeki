Detailed Description

Compressed video payload — one encoded access unit whose codec identity lives in the base ImageDesc's compressed PixelFormat.

The access unit is carried in the base's plane list — usually one plane for a single contiguous bitstream, though multiple planes (SPS / PPS / IDR fragments sharing a single backing Buffer via non-overlapping BufferView offsets) are supported for encoders that emit concatenated NAL units in one locked buffer.

Codec-specific state
  • frameType classifies the access unit as I / P / B / IDR / BRef (or FrameType::Unknown when the producer doesn't set it). The accessor is virtual so codec-specific subclasses (future NALBitstreamPayload, ProResPayload, …) can map their internal slice / picture types onto this common vocabulary without requiring every consumer to dispatch on the concrete type.
  • inBandCodecData optionally carries codec-private bytes attached to this specific packet (an in-band SPS/PPS update, for example). Stream-level codec-private data normally lives on the producer's MediaConfig; this per-payload override is null in the common case.

The MediaPayload::ParameterSet concept from the older packet type lives here as the ParameterSet flag — packets that carry only parameter sets (no actual slice data) set it so muxers know to treat them specially.

CompressedVideoPayload is intentionally not final — codec- specific subclasses are a supported extension point.

Example
ImageDesc desc(1920, 1080, PixelFormat(PixelFormat::H264));
auto pkt = CompressedVideoPayload::Ptr::create(desc, encodedBuffer);
pkt.modify()->setPts(pts);
pkt.modify()->addFlag(MediaPayload::Keyframe);
pkt.modify()->setFrameType(FrameType::IDR);

◆ isSafeCutPoint()

bool promeki::CompressedVideoPayload::isSafeCutPoint ( ) const
inlineoverridevirtual

Returns true when stopping the stream before this access unit leaves the decoder in a coherent state.

  • Intra-only streams (JPEG, JPEG XS, ProRes, DNxHD) are always safe — every access unit is an independent random-access point.
  • Temporal streams (H.264, HEVC, AV1) are safe only when the MediaPayload::Keyframe flag is set.
  • A compressed payload whose descriptor has no codec identity is treated as unsafe — we have no evidence the stream can be truncated here.
