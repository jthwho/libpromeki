Detailed Description

JPEG XS image codec using SVT-JPEG-XS.

Encodes and decodes JPEG XS (ISO/IEC 21122), a modern low-complexity intra-only codec designed for visually-lossless contribution and transport. JPEG XS targets the broadcast / ST 2110 / RFC 9134 RTP workflow: sub-frame latency, constant bitrate, and native support for 10- and 12-bit YCbCr.

Supported input formats:

  • YUV8_422_Planar_Rec709, YUV10_422_Planar_LE_Rec709, YUV12_422_Planar_LE_Rec709 (Rec.709 limited range)
  • YUV8_420_Planar_Rec709, YUV10_420_Planar_LE_Rec709, YUV12_420_Planar_LE_Rec709 (Rec.709 limited range)
  • RGB8_Planar_sRGB (sRGB full range)

Decoder targets are the matching uncompressed planar formats. Interleaved RGB (e.g. RGB8_sRGB) reaches the codec via the CSC fast path (RGB8_sRGB ↔ RGB8_Planar_sRGB) — the full encode/decode chain through Image::convert() is transparent to callers.

JPEG XS is constant-bitrate rather than quality-targeted: configure the encoder with a bits-per-pixel budget via setBpp (or MediaConfig::JpegXsBpp). Typical visually-lossless broadcast values are 2-6 bpp; the codec default is 3 bpp (≈ 8:1 compression for 8-bit 4:2:2).

Configuration (bpp, decomposition depth) is set once after construction and applies to all subsequent encode() calls. Changing parameters between calls is legal; the internal SVT encoder is re-initialised lazily on the next encode.

This class is not thread-safe. Each thread should use its own instance. The underlying SVT encoder is multi-threaded internally, so a single instance per producer thread is already parallel.

Example
JpegXsImageCodec codec;
codec.setBpp(4); // 4 bits/pixel constant bitrate
Image compressed = codec.encode(sourceImage);

Reads JPEG XS-specific knobs from config.

Honored keys:

  • MediaConfig::JpegXsBpp (int or float) — sets setBpp. Values <= 0 are ignored.
  • MediaConfig::JpegXsDecomposition (int) — sets setDecomposition. Clamped to 0-5.

Missing or malformed keys leave the corresponding setting at its current value.
