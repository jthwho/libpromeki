Detailed Description

Sample-domain binary data decoder for PCM audio payloads.

Audio analogue of ImageDataDecoder. Recovers the 64-bit payload + CRC + sync nibble emitted by AudioDataEncoder from a contiguous run of audio samples on a single channel. The decoder works at sample-accurate alignment when no resampling has occurred and tolerates ±25 % per-bit drift introduced by ordinary sample-rate conversion via a sync-nibble run-length measurement and Manchester integrate-and-compare demodulation.

Algorithm
  1. Extract sampleCount samples of the requested channel into a normalized float array (AudioFormat::samplesToFloat handles every PCM format uniformly).
  2. Locate the sync nibble: search a small window around firstSample for the first positive zero-crossing-going transition, then measure the next three transitions ((+→-, -→+, +→-)). Reject if any run is more than 25 % off the average — a strong signal that the band is corrupted or mis-located.
  3. Validate the recovered samples-per-bit against the constructor hint with a ±50 % tolerance band.
  4. For each of the 76 bit positions, integrate the first half-bit and the second half-bit. bit '1' if first > second, bit '0' otherwise. Integration is what makes Manchester robust to the low-pass filtering an SRC introduces — the smoothed transition averages out symmetrically.
  5. Verify the recovered sync nibble equals 0xA, recompute the CRC over the 8 payload bytes (big-endian), and compare to the decoded CRC.
Lifetime and reuse
Construct one decoder per (AudioDesc, samplesPerBit hint) pair and reuse it across many payloads. decode is reentrant on a single instance only with respect to itself.
Example
AudioDataDecoder dec(payload->desc());
AudioDataDecoder::Band band{0, payload->sampleCount(), 0};
auto item = dec.decode(*payload, band);
if(item.error.isOk()) {
// item.payload, item.decodedCrc, item.samplesPerBit
}

Decodes a codeword from a single-channel float buffer.

Bypasses the per-format AudioFormat::samplesToFloat extraction step the PcmAudioPayload overloads perform — useful for one-shot decode of a known band already in normalized float form. Decodes only the first complete codeword in the buffer; downstream callers that need to drain a streaming buffer should use decodeAll instead.

The buffer is treated as the entire search window: sample 0 is the start of the band, sample count-1 is the last valid sample. Sub-sample-accurate sync localisation works exactly as in the PcmAudioPayload overloads.

Parameters

countNumber of samples in samples.
Returns
A populated DecodedItem. On failure error names the failure mode and partial fields may be populated for diagnostics.

Decodes every complete codeword reachable in the rolling buffer of state, after appending count new samples.

Streaming entry point for callers that keep a cross-call rolling buffer (the inspector, network receivers, etc.) and need to decode every codeword the buffer contains, including ones that straddle the boundary between two consecutive decodeAll calls.

The append-then-decode loop:

  1. Append newSamples to state.buffer.
  2. Run sync localisation and bit demodulation on the buffer. If sync isn't found, leave the buffer alone and return — more samples may complete the codeword on the next call.
  3. If sync is found but the buffer doesn't have enough samples for the full 76-bit codeword, same: leave the buffer and return.
  4. Otherwise emit a DecodedItem to out (whether successful or not — CRC and sync-byte failures are reported via DecodedItem::error so the caller can distinguish), advance the buffer past the packet, and loop.
  5. After the loop, trim the buffer if it exceeds kStreamBufferMaxSamples (drops the oldest samples, advances the anchor).

Each emitted DecodedItem carries a stream- absolute DecodedItem::streamSampleStart (sample 0 = the very first sample ever pushed through this StreamState). Items remain in insertion order, so the last item is the most recently completed packet.

Parameters
statePer-channel rolling state.
newSamplesPointer to count fresh samples to append. May be nullptr when count is zero — useful for pumping decodes through a pure trim cycle.
countNumber of samples in newSamples.
outFilled with one DecodedItem per completed packet (success or failure). Cleared on entry.

Maximum samples held in a single StreamState before the oldest half is trimmed.

Bounded so a sustained decode failure (silent channel, hostile input) can't grow memory without limit. Sized to comfortably hold a few codewords plus one frame's worth of headroom.
