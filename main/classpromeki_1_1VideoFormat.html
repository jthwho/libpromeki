Detailed Description

Combined video raster, frame rate, and scan mode.

VideoFormat identifies a video signal by the triple (active_raster, frame_rate, scan_mode). It is a first-class Variant type and can round-trip to and from the industry-standard SMPTE-style short form ("1080i59.94", "720p59.94", "1080psf23.98").

String format

toString() emits the form "<raster><scan><rate>":

  • <raster> is the SMPTE height-only prefix ("486", "576", "720", "1080", "2160", "4320") when the full raster matches the standard broadcast width at that height (720×486, 720×576, 1280×720, 1920×1080, 3840×2160, 7680×4320). Otherwise the explicit "WxH" form is emitted ("2048x1080", "4096x2160", "405x314").
  • <scan> is "p", "i", or "psf".
  • <rate> is the frame rate as formatted by the FrameRate registry ("23.98", "29.97", "59.94", "24", "25", "30", "60", ...) for progressive and PsF. For interlaced modes <rate> is the SMPTE field rate (twice the frame rate), matching industry convention:
Frame rate Progressive Interlaced PsF
23.98 1080p23.98 — 1080psf23.98
24 1080p24 — 1080psf24
25 720p25 1080i50 1080psf25
29.97 1080p29.97 1080i59.94 1080psf29.97
30 1080p30 1080i60 1080psf30
59.94 720p59.94 — —

Set StringOptions::useNamedRaster to prefer well-known raster names ("HDp29.97", "UHDp60", "2Kp24") in place of the SMPTE height prefix. Rasters with no registered name fall back to explicit "WxH" in both modes.

String parsing

fromString() is case-insensitive and accepts:

  • SMPTE height prefixes — "1080i59.94", "720p60".
  • Well-known raster names — "HDp29.97", "UHDp60", "2Kp24", "4Kp24", "NTSCi59.94", "PALi50".
  • Explicit rasters — "1920x1080p29.97", "405x314p30".
  • "psf" suffix — "1080psf23.98".
  • All frame-rate strings that FrameRate::fromString accepts, including "23.976" (alias for 24000/1001) and explicit rationals like "30000/1001".
  • Whitespace and cosmetic separators ('@', ',', ';') between the raster, scan, and rate tokens — "1920x1080 @ 29.97", "HD, 29.97", "1080 i 59.94" all parse.
  • Scan marker positioned before or after the rate — "1080 29.97p", "1080 59.94i", "1080 24psf", "1080 29.97 p" all work in addition to the canonical "1080p29.97" inline placement. If no scan character is present anywhere, the scan mode defaults to Progressive.

By default (ParseOptions::strictInterlacedFieldRate = true) the rate suffix for an interlaced form is interpreted as the field rate, so "1080i59.94" yields a frame rate of 30000/1001. With strictInterlacedFieldRate set to false, the parser falls back to interpreting the suffix as a direct frame rate when halving it does not yield a well-known frame rate — this tolerates non-SMPTE notation like "1080i29.97".

A default-constructed VideoFormat is invalid.

◆ WellKnownFormat

enum promeki::VideoFormat::WellKnownFormat

Identifiers for well-known video formats.

The two sentinel values precede the generated list:

  • Invalid — the underlying VideoFormat is not valid at all (invalid raster or rate).
  • NotWellKnown — the VideoFormat is valid but its raster / rate / scan combination does not match any entry in PROMEKI_WELL_KNOWN_VIDEO_FORMATS.

Every other value names a specific well-known format and can be passed to the VideoFormat(WellKnownFormat) constructor to instantiate it.

◆ frameRateString()

String promeki::VideoFormat::frameRateString ( ) const

Returns the frame-rate portion of the formatted string.

For interlaced modes this is the SMPTE field rate (twice the frame rate). For progressive and PsF modes it is the frame rate itself. Well-known rates are emitted as their common-name form ("29.97", "59.94", "24", "60"); arbitrary rates are emitted in exact-rational form ("30000/1001").

Returns an empty String when the frame rate is invalid.

◆ isIntegerCadence()

bool promeki::VideoFormat::isIntegerCadence ( ) const
inline

Returns true if this format has an integer frame-rate cadence.

True for rates whose exact rational has denominator 1 (24, 25, 30, 48, 50, 60, 100, 120), false for the NTSC-derived 1000/1001 family (23.98, 29.97, 59.94, …) and for invalid frame rates. Works on any VideoFormat, not just well-known ones.
