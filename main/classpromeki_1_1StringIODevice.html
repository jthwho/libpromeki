Detailed Description

IODevice backed by a promeki String.

StringIODevice provides seekable, random-access I/O over a String pointer. It is useful wherever an IODevice interface is expected but the data lives in a String — for example, wiring a TextStream to a String for in-memory text formatting and parsing.

Ownership
StringIODevice does not own the String; the caller must ensure the String outlives the device.
Write growth
When opened for writing, writes append to or overwrite the String content at the current position. The String grows as needed.
Thread Safety
Inherits IODevice — thread-affine. A single instance must only be used from the thread that created it.
