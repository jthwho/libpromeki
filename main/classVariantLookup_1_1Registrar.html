Detailed Description

template<typename T>
class VariantLookup< T >::Registrar

Fluent builder returned by registrar.

All methods side-effect into the per-T static registry and return a fresh Registrar so registrations can be chained. The instance itself is stateless; it exists only so users can write the PROMEKI_LOOKUP_REGISTER block as a single expression.

Note
Registration methods are marked const because Registrar carries no per-instance state — the const here denotes "no @c Registrar members are modified", not "no side effects". Each call takes a write lock on the per-T static registry and inserts the handler there.

Registers a child composition (no index).

Looking up "Name.rest" calls get on the instance, then forwards rest to VariantLookup<U>::resolve. A null get result yields Error::IdNotFound. specFor on the same path delegates to VariantLookup<U>::specFor(rest), so U's declared types flow up transparently.

Registers a prefixed database binding.

Looking up "Prefix.DbKey" calls get on the instance then resolves DbKey through the returned VariantDatabase. The remainder is treated opaquely: it is handed to VariantDatabase<DbName>::ID::find without further parsing. A null get result yields Error::IdNotFound; an unregistered key also yields Error::IdNotFound. Spec lookup delegates to VariantDatabase<DbName>::specFor(rest), so declared types for keys like Meta.Timecode surface uniformly.

Template Parameters
DbNameThe compile-time name of the target VariantDatabase. The binding is typed so the right per-DbName StringRegistry is consulted.

Registers an indexed child composition.

Looking up "Name[N].rest" calls get on the instance with N, then forwards rest to VariantLookup<U>::resolve. A null get result yields Error::OutOfRange. Spec lookup ignores the index (spec is a type-level property) and delegates to VariantLookup<U>::specFor.
