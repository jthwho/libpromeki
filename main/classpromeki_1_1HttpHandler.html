Detailed Description

Abstract base class for HTTP request handlers.

Use this when a handler holds non-trivial state (a database connection, a configuration snapshot, a backing data store) that's cleaner to express as a class than to capture in a lambda. For lambda-style handlers, prefer HttpHandlerFunc directly.

HttpFileHandler and the auto-mounted reflection adapters (HttpServer::exposeDatabase, HttpServer::exposeLookup) are the primary consumers in-tree.

Thread Safety
Handlers are typically invoked on the HttpServer's owning EventLoop thread. Concurrent invocation of serve on the same handler instance is only safe if the concrete subclass documents that it is — most handlers can assume single-threaded dispatch.
