Detailed Description

Library-provided Enum types that are commonly used as config values throughout the ProAV pipeline. Each class inherits from TypedEnum<Self> so its values carry a compile-time type identity:

  • A Type member holds the registered Enum::Type handle.
  • A set of static const wrapper-class constants, one per registered value, for ergonomic use at call sites:
    cfg.set(MediaConfig::VideoPattern, VideoPattern::ColorBars);

    Enum VideoPattern — selected test pattern.
    Definition mediaconfig.h:127
  • Functions that want compile-time type checking take the concrete class (e.g. "const VideoPattern &") instead of a bare "const Enum &"; runtime compatibility with Variant and any Enum-based API is preserved via public inheritance.

The integer values are stable and match the corresponding C++ enum inside the subsystem that originally defined them (e.g. VideoTestPattern::Pattern), so code that still holds the C++ enum internally can convert in either direction via a plain static_cast on Enum::value().

The registered string names use the same CamelCase spelling as the C++ constants on the wrapper struct (e.g. "ColorBars", "BottomCenter", "LTC") rather than the legacy all-lowercase config strings. Call sites that previously wrote "colorbars" / "bottomcenter" / "tone" / "422" need to be updated to the CamelCase form or, better, to use the typed constants directly.
