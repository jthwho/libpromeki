Detailed Description

Library-provided Enum types that are commonly used as config values throughout the ProAV pipeline. Each struct follows the pattern documented on Enum:

  • A Type member holds the registered Enum::Type handle.
  • A set of static inline const Enum constants, one per registered value, for ergonomic use at call sites:
    cfg.set(MediaConfig::VideoPattern, VideoPattern::ColorBars);

The integer values are stable and match the corresponding C++ enum inside the subsystem that originally defined them (e.g. VideoTestPattern::Pattern), so code that still holds the C++ enum internally can convert in either direction via a plain static_cast on Enum::value().

The registered string names use the same CamelCase spelling as the C++ constants on the wrapper struct (e.g. "ColorBars", "BottomCenter", "LTC") rather than the legacy all-lowercase config strings. Call sites that previously wrote "colorbars" / "bottomcenter" / "tone" / "422" need to be updated to the CamelCase form or, better, to use the Enum constants directly.
