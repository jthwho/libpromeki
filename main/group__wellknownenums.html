Detailed Description

Library-provided Enum types that are commonly used as config values throughout the ProAV pipeline. Each class inherits from TypedEnum<Self> so its values carry a compile-time type identity:

  • A Type member holds the registered Enum::Type handle.
  • A set of static const wrapper-class constants, one per registered value, for ergonomic use at call sites:
    cfg.set(MediaConfig::VideoPattern, VideoPattern::ColorBars);
    List
    Dynamic array container wrapping std::vector.
    Definition list.h:40
    List::set
    bool set(size_t index, const T &val)
    Sets an item in the list by index.
    Definition list.h:541
  • Functions that want compile-time type checking take the concrete class (e.g. "const VideoPattern &") instead of a bare "const Enum &"; runtime compatibility with Variant and any Enum-based API is preserved via public inheritance.

Subsystems that consume these values (e.g. VideoTestPattern) alias the well-known class directly via using, so there is no parallel C++ enum to keep in sync.

The registered string names use the same CamelCase spelling as the C++ constants on the wrapper class (e.g. "ColorBars", "BottomCenter", "LTC"). Use the typed constants directly at call sites.
