Detailed Description

Asymmetric bound on the error envelope of a clock's now() reading relative to ground truth.

The error is defined as reportedTime - trueTime.

  • minError is the most negative value the clock is expected to return (early bias). Typically <= 0.
  • maxError is the most positive value the clock is expected to return (late bias). Typically >= 0.

The envelope is asymmetric by design. For a steady clock like std::chrono::steady_clock the envelope is symmetric and tiny (~1 ns on either side). For a clock derived from an audio device's consumed-byte counter the envelope is strictly one-sided late ({0, bufferPeriod}) because the counter reads stale between callbacks and jumps forward only when the device fires its next callback.
