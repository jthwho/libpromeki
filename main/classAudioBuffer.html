Detailed Description

Thread-safe ring-buffered audio sample FIFO with format conversion on push.

AudioBuffer stores interleaved PCM samples in a fixed output format. Callers push audio in any compatible input format (differing bit depth, endianness, integer vs float are handled automatically) and pop samples in the storage format. The buffer is designed as the efficient bridge between an Audio producer that works in one format (e.g. native float at the video frame rate) and a consumer that wants a different framing or format (e.g. a container writer that writes fixed-size chunks of s16 little-endian).

Format conversion

When inputFormat() differs from format() only in DataType (bit depth / endian / float vs int), push() converts on the fly via AudioDesc's samplesToFloat / floatToSamples helpers.

When the input and output sample rates differ and PROMEKI_ENABLE_SRC is available, push() resamples on the fly via an internal AudioResampler. Use setResamplerQuality() to control the quality mode (default: SrcQuality::SincMedium). When PROMEKI_ENABLE_SRC is not available, push() returns Error::NotSupported for rate mismatches.

When the input and output channel counts differ, push() returns Error::NotSupported (channel-map hook TBD).

Drift correction

When the producer and consumer run on independent clocks (e.g. ALSA capture vs. video frame timer), their rates drift relative to each other. Over time the buffer either fills up (producer faster) or drains (consumer faster).

Call enableDriftCorrection() to engage the resampler with automatic ratio adjustment. On each push(), the buffer measures its fill level against a caller-supplied target and adjusts the resampling ratio via a PI (proportional–integral) controller:

error = (available - target) / target
integral += error
ratio = nominalRatio * (1.0 - Kp * error - Ki * integral)

The proportional term (Kp = gain) reacts immediately to fill level deviations. The integral term (Ki = gain / 1000) accumulates over time, eliminating any steady-state offset caused by a constant clock drift — without the integral, a proportional-only controller would allow the buffer to stabilize above or below the target.

When the buffer is overfull the ratio decreases (fewer output samples per input), draining it back to the target. When it is underfull the ratio increases, filling it back up. The gain parameter controls how aggressively the correction is applied — small values (0.001 – 0.01) give inaudible corrections.

Drift correction works whether the input and output sample rates are the same or different. When rates are the same, the nominal ratio is 1.0 and only the drift adjustment moves it. When rates differ, the nominal ratio handles the fixed conversion and the drift adjustment rides on top.

Thread safety

All push, pop, and query methods are internally synchronized via a Mutex. Push methods wake any thread blocked in popWait() after writing. This allows a producer thread and a consumer thread to operate on the same AudioBuffer without external locking.

Capacity and ownership

The buffer is sized via reserve() (or constructor). Pushes that would exceed capacity return Error::NoSpace — the caller is expected to size the buffer for their expected peak residency. The class is move-only; copying a FIFO is almost always a mistake.

Example
AudioBuffer fifo(AudioDesc(AudioFormat::PCMI_S16LE, 48000, 2));
fifo.reserve(48000); // 1 second of headroom
fifo.setInputFormat(AudioFormat::NativeFloat, 48000, 2);
// Producer thread
Audio chunk(nativeDesc, 1602);
// ... fill chunk ...
fifo.push(chunk);
// Consumer thread — blocks until 1600 samples are ready
Audio slice(outputDesc, 1600);
auto [got, err] = fifo.popWait(slice, 1600, 200);
if(err == Error::Timeout) { ... }

Disables clock-drift correction.

The internal resampler is reset but kept alive if the input and output sample rates still differ (fixed rate conversion continues). If rates are the same, the resampler is no longer used until drift correction is re-enabled.

When PROMEKI_ENABLE_SRC is off this is a no-op.

Error AudioBuffer::enableDriftCorrection ( size_t  targetSamples,
double  gain = 0.001 
)

Enables clock-drift correction on push.

When enabled, the buffer dynamically adjusts the resampling ratio on each push() to steer the fill level toward targetSamples. This compensates for clock drift between a producer and consumer running on independent oscillators.

Drift correction works for both same-rate and different-rate configurations. When rates are the same, the resampler is created with a nominal ratio of 1.0 and only the drift adjustment moves it.

Parameters
targetSamplesDesired steady-state fill level in samples. A good default is half the reserved capacity.
gainProportional gain for the correction. Small values (0.001 – 0.01) give smooth, inaudible corrections. Larger values track drift faster but may cause audible pitch shifts. Default: 0.001.
Returns
Error::Ok on success, Error::NotSupported if PROMEKI_ENABLE_SRC is off.

◆ setInputFormat()

void AudioBuffer::setInputFormat ( const AudioDesc &  input)

Sets the expected input format for push().

If input differs from format() only in DataType, push() converts on-the-fly via samplesToFloat / floatToSamples. If the sample rate differs and PROMEKI_ENABLE_SRC is available, push() resamples transparently. If the channel count differs, push() returns NotSupported (channel-map hook TBD).

Changing the input format resets any internal resampler state so filter history from the previous format does not bleed through.
