Detailed Description

Per-stat-ID WindowedStat container for telemetry breakdowns.

Holds one WindowedStat per MediaIOStats::ID — the natural shape of a "windowed" view onto per-command telemetry. Created when the previous Map<Kind, MediaIOStats> design ran into a type-system mismatch: registered specs declare ExecuteDuration as TypeDuration, but the windowed view of the same key holds a WindowedStat of nanoseconds — strict spec validation rightly rejects that swap. Pulling the windowed slot into its own type removes the conflict without disabling validation anywhere.

The WindowedStatsBundle does not carry its own ID namespace; the keys are MediaIOStats::ID values so collectors and consumers keep using the same well-known identifiers (ExecuteDuration, QueueWaitDuration, BytesProcessed, ...) the MediaIOStats already declares.

Serialization round-trips through JSON (each entry is the canonical "cap=N:[...]" string parsed by WindowedStat::fromString) and DataStream (one TypeWindowedStat tag per entry).

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized.

◆ ValueFormatter

using promeki::WindowedStatsBundle::ValueFormatter = std::function<WindowedStat::ValueFormatter(ID)>

Renders every entry as a multi-line summary.

One line per stat ID: "<name>: <window summary>", where the window summary comes from WindowedStat::toString. The optional formatter is forwarded to each window so callers can humanise values per stat ID — useful for unit-aware output (ms, MB) where the picker depends on the ID name.
