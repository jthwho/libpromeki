Detailed Description

Abstract polymorphic base for any media payload flowing through a pipeline.

A MediaPayload represents one unit of media — an uncompressed video frame, an uncompressed audio block, a compressed access unit, a metadata packet, etc. — carried between pipeline stages. The common fields apply to every kind:

  • A BufferView holding 1..N planes of bytes. A single entry covers the contiguous case (interleaved RGB, interleaved PCM, a compressed bitstream); multiple entries cover planar video (Y/U/V) and planar audio (one buffer per channel). The views may alias regions of one shared Buffer (e.g. a planar frame pulled in by a single read) or reference independently allocated buffers.
  • Presentation / decode timestamps and a nominal duration.
  • A Metadata container for payload-intrinsic keys.
  • A stream index so payloads from distinct tracks in the same Frame can be told apart.
  • A bitmask of Flag values that apply generically to every subclass.

Subclasses introduce the fields specific to their family: VideoPayload carries an ImageDesc, AudioPayload an AudioDesc, and the concrete uncompressed / compressed leaves add whatever per-payload state their family needs. "Is this compressed?" is answered by isCompressed, which the concrete classes implement (typically by delegating to the descriptor's own compressed flag).

Subclassing

Intermediate abstracts (VideoPayload, AudioPayload) leave _promeki_clone and kind pure-virtual; the concrete leaves override them and provide covariant clones. The concrete classes are intentionally not final — codec-specific specializations (for example, an H.264 or ProRes wrapper of CompressedVideoPayload) are a supported extension point.

Ownership

MediaPayload uses the library's native shared-object pattern. Callers manage lifetime via MediaPayload::Ptr . Polymorphic construction typically goes through a concrete subclass's Ptr::create helper; the converting constructor on SharedPtr lets you hand a VideoPayload::Ptr into code that accepts MediaPayload::Ptr without cloning.

MediaPayload::Ptr p = UncompressedVideoPayload::Ptr::create(desc, planes);
if(p->kind() == MediaPayloadKind::Video) {
auto *vp = p->as<VideoPayload>(); // safe dispatch, no RTTI miss
...
}

Thread Safety
MediaPayload::Ptr is atomically refcounted, so handing a payload between pipeline stages running on different threads is safe. A single MediaPayload instance is conditionally thread-safe: const accessors (descriptor, kind, metadata read) are safe; mutators require external synchronization. The intended pipeline pattern is "produce on stage A, hand off to stage B by Ptr, then mutate copy-on-write from B" — this is race-free as long as no thread keeps a non-Ptr pointer to a payload it has handed off.

◆ Flag

enum promeki::MediaPayload::Flag : uint64_t

Generic bit flags that apply to every payload kind.

Values are bitwise-OR'able. Replace the entire mask via setFlags, or manipulate individual bits via addFlag / removeFlag / setFlag / hasFlag. The convenience predicates (isKeyframe, isDiscardable, isCorrupt, isEndOfStream, isIntraRefresh) read from this mask.

Flags specific to one payload family (for example, ParameterSet on compressed video) live on the concrete subclass rather than the base. The mask is 64 bits wide so subclass flags never have to compete for the same bit range as the base flags.

Enumerator
None 

No flags set.

Keyframe 

Self-contained decode entry point (trivially true for uncompressed payloads).

Discardable 

Non-reference payload — safe to drop without affecting later decode.

Corrupt 

Payload is known to be corrupt; pair with Metadata::CorruptReason when a message is available.

EndOfStream 

Terminal payload in the stream.

IntraRefresh 

Intra-coded but not a random-access point — part of a gradual refresh cycle (distinct from Keyframe).

◆ duration()

virtual Duration promeki::MediaPayload::duration ( ) const
inlinevirtual

Returns the wall-clock duration spanned by this payload, or a zero Duration when no duration is available.

Polymorphic hook — the base default returns Duration() because MediaPayload does not itself own any state from which a duration could be derived. Subclasses override:

  • AudioPayload computes sampleCount() / desc().sampleRate() — one source of truth, no storage needed.
  • VideoPayload returns a native duration field populated by the enclosing pipeline (typically MediaIO stamps one frame of the session frame rate when the producer did not).

A zero Duration from a payload whose hasDuration returns true means "no value has been stamped yet" — MediaIO uses that combination as its fill trigger.

Reimplemented in promeki::AudioPayload, and promeki::VideoPayload.

◆ ensureExclusive()

void promeki::MediaPayload::ensureExclusive ( )
inline

Detaches every plane's backing Buffer so writes through this payload don't propagate to other holders of the same Buffer.

Walks data() and, when a plane's backing Buffer is shared (reference count > 1), clones it via Buffer::Ptr::modify. Whole- buffer views keep their offset=0, size=full invariant; sub-range views retain offset and size but point at the exclusive clone.

Calls ensureExclusiveExtras after the plane detach so subclasses can CoW-detach any extra Buffer fields they own (codec private data, auxiliary metadata buffers, …).

◆ isSafeCutPoint()

virtual bool promeki::MediaPayload::isSafeCutPoint ( ) const
inlinevirtual

Returns true when stopping the stream before this payload leaves downstream consumers in a coherent state.

Distinct from isKeyframe — a payload can be a decode entry point without being a safe cut (H.264 open-GOP, where an I-frame's trailing B-frames still reference the previous GOP), or be a safe cut without being the codec's "keyframe" concept (audio codecs whose PacketIndependence is Every). Subclasses override with codec- or family-specific rules; the default falls back to isKeyframe as a conservative baseline.

Used by Frame::isSafeCutPoint and MediaPipeline to honour a pipeline-wide frame count without truncating a GOP or stranding a non-decodable audio packet.

Reimplemented in promeki::CompressedAudioPayload, promeki::CompressedVideoPayload, promeki::PcmAudioPayload, and promeki::UncompressedVideoPayload.

◆ metadata() [1/2]

virtual const Metadata & promeki::MediaPayload::metadata ( ) const
pure virtual

Returns the payload's metadata container.

Pure virtual on the base — each concrete subclass decides where the metadata lives:

  • VideoPayload and AudioPayload forward to desc().metadata(), so VideoPayload::metadata returns the ImageDesc metadata (the FrameRate / colorimetry / timecode keys) and AudioPayload::metadata returns the AudioDesc metadata. The metadata lives next to the descriptor it annotates.
  • Non-AV subclasses (subtitle, ancillary, custom) maintain their own Metadata member and return a reference to it.

This lets markCorrupt, corruptReason, VariantLookup's Meta.* binding and any generic consumer hold a MediaPayload reference and reach the right metadata store without knowing the concrete type.

Implemented in promeki::AudioPayload, and promeki::VideoPayload.
