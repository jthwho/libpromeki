282 // Generic image cache: a small fixed array of slots
283 // shared by all patterns. Slot meanings are local to
284 // each pattern's branch in create() — for example
285 // AvSync uses slot 0 for "white marker" and slot 1 for
286 // "black non-marker", while a static pattern uses slot
287 // 0 for its pre-rendered background. Patterns are
288 // mutually exclusive at any given time, so reusing the
289 // same slot index across patterns is fine.
290 //
291 // Any change to _pattern, _solidColor, or the
292 // requested ImageDesc dumps the entire cache; the next
293 // create() call rebuilds whatever it needs. Two
294 // renderSolid() calls (the most expensive thing the
295 // current patterns can do on a rebuild) are essentially
296 // free, so a simpler dump-on-change policy is the right
297 // tradeoff over per-key invalidation.
298 static constexpr int CacheSlotCount = 2;
