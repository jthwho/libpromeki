Detailed Description

Auto-scaling unit formatter for byte counts, durations, frequencies, and sample counts.

Centralizes the metric-prefix (and IEC-binary-prefix) formatting that was previously scattered across String::fromByteCount (now removed), Duration::toString, and ad-hoc helpers in benchmark code.

Every static method picks the scale factor and unit suffix automatically so the caller only provides a raw numeric value.

Thread Safety
Fully thread-safe. Units exposes only static formatter functions over caller-provided values; no shared state.
Example
Units::fromByteCount(1500000); // "1.5 MB"
Units::fromByteCount(1048576, 3, ByteCountStyle::Binary); // "1 MiB"
Units::fromDuration(0.0015); // "1.5 ms"
Units::fromDurationNs(8970); // "8.97 us"
Units::fromFrequency(48000.0); // "48 kHz"
Units::fromSampleCount(96000, 48000); // "2 s"

Formats a raw count (frames, items, queue depth) with metric prefixes.

Auto-scales with k/M/G/T suffixes (1000-based). Unlike fromItemsPerSec this never returns the "-" placeholder — zero counts render as "0" — so it is suitable for telemetry rows where every stat slot is always populated.

Example
Units::fromCount(0); // "0"
Units::fromCount(127); // "127"
Units::fromCount(1500); // "1.5k"
Units::fromCount(2340000ULL); // "2.34M"

Formats a duration given in milliseconds.

Auto-scales to ns, us, ms, s, m, or h via the same table fromDurationNs uses. Provided so callers holding ms values (latency telemetry, timer intervals) don't have to scale up to ns at the call site.

Example
Units::fromDurationMs(0.05); // "50 us"
Units::fromDurationMs(4.2); // "4.2 ms"
Units::fromDurationMs(1500.0); // "1.5 s"
