Detailed Description

Well-known Enum type for HTTP request methods.

Covers the nine methods defined by RFC 9110. The integer values are arbitrary internal IDs — the wire-format method is the registered string name (e.g. "GET", "POST"), available via Enum::valueName and round-trippable via the TypedEnum string constructor.

Example
HttpMethod m = HttpMethod::Post;
String onWire = m.valueName(); // "POST"
HttpMethod parsed{ String("PUT") }; // HttpMethod::Put

◆ allowsBody()

bool promeki::HttpMethod::allowsBody ( ) const

Whether the method has a meaningful request body.

True for HttpMethod::Post, HttpMethod::Put, and HttpMethod::Patch; false for the rest. Used by HttpConnection to decide whether to wait for a body even when no Content-Length header is present.

◆ wireName()

String promeki::HttpMethod::wireName ( ) const

Returns the canonical wire-form method name.

Convenience wrapper around valueName() that returns the upper-case ASCII string sent on the request line ("GET", "POST", ...). An invalid HttpMethod returns an empty string.
