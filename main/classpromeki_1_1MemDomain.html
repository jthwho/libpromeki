Detailed Description

Coarse classification of memory address spaces.

MemDomain identifies the class of a memory address space — Host, CudaDevice, FpgaDevice, etc. — independently of the specific MemSpace within that class. Multiple MemSpaces can belong to the same MemDomain (e.g. SystemHost, SystemSecure, and CudaPinnedHost all live in MemDomain::Host), and a Buffer can be mapped for access from any registered domain via Buffer::mapAcquire.

TypeRegistry
Mirrors the MemSpace registration pattern: well-known IDs are pre-populated; user code can register additional domains with registerType followed by registerData. The wrapper is a lightweight handle to an immutable Data record looked up by ID.
Thread Safety
Fully thread-safe. The registry is built once at startup and read freely thereafter; the wrapper is a value type carrying a single pointer.

Identifies a memory domain.

Well-known domains have named enumerators. User-defined domains obtain IDs from registerType, which begins at UserDefined.

Enumerator

Host CPU memory (system RAM, including page-locked / secure variants).

CudaDevice 

CUDA device memory (GPU VRAM).

FpgaDevice 

FPGA on-board memory or buffer-index space.

Default
