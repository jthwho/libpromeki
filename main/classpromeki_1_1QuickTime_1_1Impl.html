Detailed Description

Abstract backend for a QuickTime instance.

Readers and writers derive from this base. The base provides default "not implemented" behavior for every virtual so that a partially-implemented backend still compiles and returns meaningful errors.

Reads count consecutive samples starting at startSampleIndex as a single concatenated payload.

Useful for pulling a block of PCM audio in one go without N individual readSample calls. The returned Sample's metadata reflects the first sample in the range, and data contains the concatenated bytes of all count samples.

The default implementation calls readSample in a loop; backends can override for efficiency.

Returns
Error::Ok on success.

◆ setFlushSync()

void promeki::QuickTime::Impl::setFlushSync ( bool  enable)
inline

Enables or disables forcing an fdatasync after each successful flush().

When enabled, each fragment boundary becomes a durable checkpoint: the data is guaranteed to have reached stable storage before flush() returns. Useful for crash-critical capture workflows where losing the last fragment is unacceptable. Expensive — adds latency at every fragment boundary.

Default: false.

◆ startTimecode()

const Timecode & promeki::QuickTime::Impl::startTimecode ( ) const
inline

Returns the anchor timecode resolved from the container's tmcd track, or an invalid Timecode if no timecode track is present.

Per-frame timecode is computed by advancing this anchor by the requested frame index — only the starting timecode (and the tmcd flags such as drop-frame) is parsed during open().
