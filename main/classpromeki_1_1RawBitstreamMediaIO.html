Detailed Description

MediaIO sink that writes encoded packet payloads verbatim to a file.

RawBitstreamMediaIO is the minimum possible backend for capturing the output of a VideoEncoderMediaIO — every CompressedVideoPayload on an incoming Frame is appended to the target file exactly as it arrived, with no container framing, no timestamps, no muxing, and no modification of the payload. The resulting file is a raw elementary stream — for H.264 / HEVC output from NVENC, that's Annex-B byte-stream form, which is exactly what ffplay / ffprobe / mpv will accept.

Registered with the extension list "h264" / "h265" / "hevc" so mediaplay's file-path auto-detection routes -d foo.h264 through this backend without requiring the caller to say RawBitstream explicitly.

Only sink mode (MediaIO::Sink) is supported in this first iteration — reading an elementary-stream file back into CompressedVideoPayloads requires a codec-specific NAL-unit parser that belongs with the decoder work.

◆ describe()

Error promeki::RawBitstreamMediaIO::describe ( MediaIODescription *  out) const
overridevirtual

Populates out with a snapshot of this MediaIO's identity, role, and format landscape.

The wrapper fills in:

  • Backend identity (name, description) from the registered MediaIOFactory.
  • Role flags (canBeSource / canBeSink / canBeTransform) from the same MediaIOFactory.
  • Instance identity (name) from this MediaIO.
  • Cached canSeek, frameCount, frameRate, containerMetadata if open.

Then dispatches to the backend's describe to add format-specific fields (producibleFormats, acceptableFormats, preferredFormat) and any pre-open probe results.

Synchronous; safe to call any time (pre-open, while open, after close). Cheap when open (cached state is reused) and as cheap as the backend's probe implementation otherwise.

Parameters
outThe description to populate (overwritten).
Returns
Error::Ok on success, or the probe error from the backend (also stamped into out's probeStatus field).
