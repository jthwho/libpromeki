Detailed Description

Result of decoding one Band or one streaming packet.

error reports the outcome. On success (error.isOk()) all fields are valid. On a decode failure, error names the failure mode and any partially-recovered values are populated for diagnostics (e.g. decodedSync may be non-zero when only the CRC mismatched).

◆ packetSampleCount

int64_t promeki::AudioDataDecoder::DecodedItem::packetSampleCount = 0

Number of samples this packet occupied, computed from the measured pitch as round(samplesPerBit * BitsPerPacket). Reflects the actual on-wire span the codeword took — a number lower than BitsPerPacket * AudioDataEncoder::DefaultSamplesPerBit indicates the packet was compressed in time (downstream rate change), higher indicates expansion. Zero when no sync was found (the samplesPerBit measurement isn't available).

◆ streamSampleStart

int64_t promeki::AudioDataDecoder::DecodedItem::streamSampleStart = -1

Stream-absolute sample position of the first positive sync sample, measured from sample 0 of the StreamState's first decodeAll call. Populated only by decodeAll; -1 for items returned by the per-band decode entry points. Signed because downstream consumers may rebase the anchor against an external reference (e.g. an audio MediaTimeStamp), at which point packets that straddle the rebase point land before the origin.
