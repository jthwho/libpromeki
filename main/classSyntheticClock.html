Detailed Description

Frame-count driven synthetic Clock.

SyntheticClock reports raw time as currentFrame × framePeriod. It has no wall-time component — the counter only moves when the caller calls advance, setCurrentFrame, or reset.

This is the right clock for any pipeline where each output frame is exactly one FrameRate period, decoupled from real time: file writers that need a pristine frame rate, offline conversion, and tests that want deterministic timing.

Properties
  • Domain defaults to ClockDomain::Synthetic. Override via the constructor.
  • resolutionNs returns 1 (the stored counter is ns-accurate).
  • jitter returns {0, 0} — a SyntheticClock is perfect.
  • rateRatio returns 1.0.
  • Cannot be paused.
sleepUntilNs

The base-class sleep is a no-op here — the counter is authoritative and only advance / setCurrentFrame move time forward.

Invoked by sleepUntil after domain validation and offset translation. If targetNs is in the past, implementations should return Error::Ok immediately. Implementations backed by a disappearing resource (audio device, remote clock) return the relevant error when the wait cannot complete.
