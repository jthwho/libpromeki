Detailed Description

Frame-count driven synthetic Clock.

SyntheticClock reports time as currentFrame × framePeriod. It has no wall-time component — its notion of "now" only moves when the caller explicitly advances the frame counter via advance or setCurrentFrame.

This is the right clock for any pipeline where each output frame is exactly one FrameRate period, decoupled from real time:

  • File writers that must emit a fixed, pristine frame rate.
  • Offline / batch conversion.
  • Tests that want deterministic, reproducible timing.

When handed to a FrameSync, the sync object advances the frame counter by 1 for every output frame it emits, which is how "output frame count drives the clock" gets wired up.

Properties
  • domain defaults to ClockDomain::Synthetic. Override with setDomain if cross-stream correlation is needed.
  • resolutionNs returns 1 (the stored counter is ns-accurate).
  • jitter returns {0, 0} — a SyntheticClock is perfect.
  • rateRatio returns 1.0.
sleepUntilNs

sleepUntilNs is a no-op. It does not advance the internal counter; it does not block. This is deliberate: the counter is the authoritative state, and only advance or setCurrentFrame move it.

◆ rateRatio()

double SyntheticClock::rateRatio ( ) const
overridevirtual

Ratio of actual tick rate to nominal tick rate.

For clocks that are their own nominal reference (WallClock, PTP, SyntheticClock) the ratio is 1.0. For clocks derived from an external rate source (audio device drain, a remote stream's packet rate), the ratio reports the actual rate measured against that reference, which callers can use as a drift-correction signal (e.g. feeding an audio resampler ratio).

The default implementation returns 1.0.

Reimplemented from Clock.

void SyntheticClock::sleepUntilNs ( int64_t  targetNs)
overridevirtual

Blocks until the clock reaches targetNs.

Implementation determines the waiting mechanism: an OS sleep for wall clocks, polling a queue depth for audio clocks, waiting on a hardware timer for PTP.

If targetNs is in the past, the call returns immediately. A SyntheticClock implements this as a no-op — its notion of "now" is driven by explicit advance calls, not by sleeping.

targetNsTarget time in nanoseconds from the clock's epoch.
