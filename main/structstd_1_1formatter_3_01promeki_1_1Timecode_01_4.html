Detailed Description

std::formatter specialization for promeki::Timecode.

Demonstrates the bespoke-formatter pattern for a library type whose toString() takes a parameter and returns a non-String type (here, Result<String>).

Format spec syntax
{} // default SMPTE e.g. "01:00:00:00"
{:smpte} // explicit SMPTE
{:smpte-fps} // SMPTE with frame rate, e.g. "01:00:00:00/24"
{:smpte-space} // SMPTE with space-separated frame rate
{:field} // field-style separator, e.g. "01:00:00.00"

The hint may be followed by a colon and a standard string format spec to apply width / fill / alignment to the rendered timecode:

{:smpte:>16} // right-justified, width 16
{:>16} // default SMPTE, right-justified width 16
{:smpte:*<16} // left-justified, width 16, '*' fill

Unrecognized hints fall through to the standard string format parser, so a stray spec like {:>16} still works.
