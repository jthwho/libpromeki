Detailed Description

Drift-free frame pacer with pluggable clock and phase control.

FramePacer blocks the calling thread until the next frame deadline, using rational arithmetic via FrameRate::cumulativeTicks to compute absolute deadlines from an origin time. This avoids the cumulative drift that arises from repeatedly adding a floating-point frame period, which matters for NTSC rates (29.97, 59.94, etc.) over long runs.

Clock sources

By default the pacer uses the system's monotonic wall clock (steady_clock). Call setClock() to supply a custom Clock for PTP, audio, or other time sources. The clock is not owned — the caller must keep it alive for the lifetime of the pacer.

Phase alignment (ST 2110)

For broadcast applications where frame boundaries must align to a network epoch, call reset(originNs) with the desired origin in the clock's time domain. For example, an ST 2110 sender would compute the PTP time at which frame 0 of the stream begins and pass that as the origin:

FramePacer pacer(FrameRate(FrameRate::FPS_29_97));
pacer.setClock(&ptpClock);
pacer.reset(streamPhaseNs); // PTP time of frame 0
while(running) {
auto r = pacer.pace();
if(r.framesToDrop > 0) sender.reportDrops(r.framesToDrop);
sender.transmitFrame(r.frameIndex);
}

Pace result and frame dropping

pace() returns a PaceResult that tells the caller everything it needs to react:

  • frameIndex — which frame this pace corresponds to.
  • framesToDrop — how many frames the caller should drop to get back in sync. Advisory only — the pacer does not skip frames on its own.
  • error — signed wake-up error relative to the ideal deadline. Positive = late, negative = early.

The pacer always advances by exactly one frame per pace() call. If the caller drops frames, it tells the pacer via advance(), which jumps the frame counter forward. If the caller doesn't drop, the pacer's error compensation handles it naturally: when the accumulated error exceeds the remaining time, pace() returns immediately (zero sleep) so the pipeline runs as fast as it can until the error drains below a frame period. The timeline stays anchored — deadlines are always computed from the origin — so the caller catches up as quickly as its processing allows.

Error compensation

After each sleep the pacer measures the actual wake-up time and accumulates the signed error (positive = late, negative = early). On the next pace, it biases the sleep target by the accumulated error so that small oversleeps are corrected on the following frame.

Typical wall-clock usage
FramePacer pacer(FrameRate(FrameRate::FPS_29_97));
pacer.reset();
while(running) {
auto r = pacer.pace();
if(r.framesToDrop > 0 && canDrop()) {
pacer.advance(r.framesToDrop);
// ... skip r.framesToDrop frames ...
}
presentFrame(r.frameIndex);
}

◆ pace()

PaceResult FramePacer::pace ( )

Paces one frame and returns the result.

Advances the frame counter by 1 and sleeps until the deadline for that frame (biased by accumulated error). If the deadline has already passed, returns immediately and reports the lateness and recommended drop count in the result.

On the first call after reset(), captures the origin (unless an explicit origin was provided) and returns immediately with frame index 0.

Returns a default PaceResult if the frame rate is invalid.

Returns
A PaceResult describing the outcome.
