Detailed Description

Identifier for a logical audio stream within a channel map.

Every entry in an AudioChannelMap pairs a ChannelRole (positional / functional role) with an AudioStreamDesc (the logical stream the channel belongs to). This lets a single audio block carry multiple program streams simultaneously — a 5.1 Main stream alongside a Commentary mono stream, for instance — without needing a separate buffer per stream.

Identity

An AudioStreamDesc is a thin wrapper around a uint64_t identifier registered in the AudioStreamDescRegistry. ID 0 is reserved as Undefined — the default stream for code that does not need to distinguish multiple streams (the dominant case). Non-zero IDs are assigned by the registry from the FNV-1a hash of the stream's name, so a given name always resolves to the same ID within a process and across runs of the same build.

Built-in stream names

The library pre-registers a small set of common stream names at static-init time so application code can refer to them symbolically:

  • Main, Music, Dialog, Effects, Commentary, DescriptiveAudio, Background, Foreground, VoiceOver

Applications register additional names ad-hoc by constructing AudioStreamDesc("MyStreamName") — the registry adds the name on first use.

Name constraints

Stream names must not contain the ':' or ',' characters, since those are the delimiters used by AudioChannelMap::toString and AudioChannelMap::fromString (e.g. "Main:5.1" or "Main:FrontLeft,Commentary:Mono"). Constructing an AudioStreamDesc from a name containing either character logs a warning via promekiWarn and yields the Undefined sentinel; the same restriction applies to lookup.

Example
AudioStreamDesc undef; // Undefined (id == 0)
AudioStreamDesc main("Main"); // built-in
AudioStreamDesc cust("MyExtra"); // app-registered
assert(main.name() == "Main");
assert(undef.isUndefined());

Thread Safety

AudioStreamDesc instances are plain values — copy / compare concurrently without synchronization. The backing StringRegistry is internally synchronized for the registration path.

◆ Undefined

constexpr ID promeki::AudioStreamDesc::Undefined = 0
staticconstexpr

Reserved ID for the Undefined stream.

Default-constructed AudioStreamDesc has id() == Undefined and isUndefined returns true. The library guarantees 0 is never assigned to a registered name (FNV-1a's offset basis is non-zero, and the registry probes past collisions), so 0 is permanently free.
