Detailed Description

One detected discontinuity in the inspector's frame stream.

Inspector continuity tracking compares the previous frame's picture-side metadata against the current frame's, and emits one InspectorDiscontinuity for every property that changed in an unexpected way. The triple (kind, previousValue, currentValue) is intended to be self-describing — the description field is a pre-rendered human-readable string suitable for logging.

enum InspectorDiscontinuity::Kind

Categorical type of the discontinuity.

Enumerator
FrameNumberJump 

Frame number from the picture data band did not advance by 1.

StreamIdChange 

Stream ID from the picture data band changed between frames.

PictureTcJump 

Picture timecode did not advance by 1 frame compared to the previous frame.

LtcTcJump 

LTC timecode did not advance by 1 frame compared to the previous frame.

ImageDataDecodeFailure 

Picture data band failed to decode (CRC mismatch / sync error / etc.). After at least one previous successful decode this counts as a continuity event because the stream's picture-side identity disappeared mid-flight.

LtcDecodeFailure 

LTC failed to decode in this frame's audio after a previously-successful decode.

SyncOffsetChange 

The picture-vs-LTC sync offset moved by more than the configured tolerance from the previous frame. Audio and video are locked in pro workflows, so any change from a previously-stable offset is a real fault rather than a measurement artifact.
