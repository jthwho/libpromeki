Detailed Description

Default Clock for a MediaIOPortGroup without a backend-supplied clock.

MediaIOClock synthesizes raw time from its bound port group's current frame position and frame rate:

rawNs = group->currentFrame() * group->frameRate().frameDuration().nanoseconds()

The group is tracked via an ObjectBasePtr so that when the owning MediaIOPortGroup is destroyed, raw returns Error::ObjectGone and the clock's now / nowNs propagate that error to callers.

Late-bind construction
The clock can be constructed with a null group and later bound via setGroup. This supports the CommandMediaIO::addPortGroup helper, which needs a clock to construct the group, and a group pointer to populate the clock — the helper allocates the clock with a null group, constructs the group with the clock, then ties the two together via setGroup.
Properties
  • Domain: ClockDomain::Synthetic.
  • Resolution: one frame period (or 1 ns if the frame rate is not yet valid).
  • Jitter: symmetric, ± half a frame period — the underlying source can only report time at frame boundaries.
  • Cannot be paused.
sleepUntilNs
A no-op — MediaIOPortGroup::currentFrame is the authoritative driver, so there is no wall-clock to wait for. Callers that need real-time pacing should use a WallClock or the device-supplied clock from a task backend.
Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized. The clock reads its driving frame counter from the bound MediaIOPortGroup and inherits that object's threading contract.
