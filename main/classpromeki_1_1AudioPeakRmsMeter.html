Detailed Description

Default per-channel peak + RMS meter.

Tracks two values per channel, both in linear amplitude:

  • peak — the maximum absolute sample value seen since the last reset, with optional exponential decay between consecutive process calls (configured via setPeakDecay). Decay is a multiplicative factor in [0, 1] applied at the start of each block before the block's own peak is folded in; 1.0 disables decay (peak is a strict running maximum).
  • rms — the running root-mean-square over the most recent samples within a sliding window whose length is configured via setRmsWindow (frames per channel). Internally the meter accumulates a sum of squares and the count, draining the oldest samples by exponential averaging — cheap, no ring buffer, and converges to the true sliding RMS within a few window lengths.

Both values are stored in std::atomic<float> for lock-free readout from any thread. The meter does not hold internal state beyond the per-channel atomics, so it is safe to query concurrently with process.

Convert to dBFS via 20 * log10(value) at the call site if needed; the meter intentionally exposes linear values so callers picking different reference levels (e.g. EBU, SMPTE, K-weighted) don't pay a log they then have to undo.

void promeki::AudioPeakRmsMeter::setPeakDecay ( float  decay)
inline

Sets the peak decay factor.

Multiplied into the running peak before each block's own peak is folded in. 1.0 (default) disables decay; values below 1.0 cause the displayed peak to drop toward zero between transients, useful for VU-style meters. Typical values: 0.999 for slow decay, 0.99 for moderate, 0.9 for fast.
