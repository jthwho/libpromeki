Detailed Description

Well-known Enum type for audio test pattern generator modes.

Used as the element type for the MediaConfig::AudioChannelModes EnumList — the TPG consumes a list of AudioPattern values, one per audio channel, so each channel can carry a different diagnostic signal. If the configured list is shorter than the stream's channel count, the remaining channels are silenced.

  • Tone — continuous sine at the configured tone frequency.
  • Silence — constant zero.
  • LTC — Linear Timecode audio encoding this channel.
  • AvSync — one-shot tone burst on tc.frame() == 0, silent otherwise. Pairs with the picture AvSync marker so inspectors can measure audio-to-video drift.
  • SrcProbe — 997 Hz reference tone. 997 Hz is prime relative to common sample rates so any downstream sample rate conversion shifts the observed frequency by a detectable amount.
  • ChannelId — channel-unique sine whose frequency encodes the channel index, so a receiver that sees the audio out of order can identify which channel is which without metadata. See MediaConfig::AudioChannelIdBaseFreq and MediaConfig::AudioChannelIdStepFreq.
  • PcmMarker — sample-domain framing of a 64-bit payload (frame BCD64 timecode when available, otherwise a per-instance monotonic counter). Enables bit-exact round-trip verification of the audio pipeline — the analog of the picture-side ImageDataEncoder / ImageDataDecoder.
  • WhiteNoise — flat-spectrum noise generated from a cached pseudo-random buffer. The buffer is built once from a fixed seed in AudioTestPattern::configure and shared across WhiteNoise channels, with a per-channel read offset so the channels stay decorrelated.
  • PinkNoise — -3 dB/octave noise generated from a cached buffer (Paul Kellet's IIR filter applied to white noise). Same caching strategy as WhiteNoise.
  • Chirp — continuous logarithmic sine sweep from MediaConfig::AudioChirpStartFreq to MediaConfig::AudioChirpEndFreq over MediaConfig::AudioChirpDurationSec. Phase is integrated across create() calls so chunk boundaries don't introduce clicks; the sweep loops once it reaches the end frequency.
  • DualTone — two simultaneous sines summed on the channel. The default configuration reproduces the SMPTE IMD-1 reference (60 Hz + 7 kHz at 4:1 amplitude) for intermodulation-distortion measurement. See MediaConfig::AudioDualToneFreq1, MediaConfig::AudioDualToneFreq2, and MediaConfig::AudioDualToneRatio.
  • Sweep — linear frequency sweep from AudioTestPattern::sweepStartFreq to AudioTestPattern::sweepEndFreq over AudioTestPattern::sweepDurationSec. Complements the logarithmic Chirp for frequency-response measurements where a constant Hz/s sweep rate is preferred.
  • Polarity — positive-going half-sine impulse repeated at AudioTestPattern::polarityPulseHz. If the received waveform goes positive first, the path is non-inverting; negative first means an odd number of polarity inversions.
  • SteppedTone — discrete frequency steps cycling through AudioTestPattern::steppedToneFreqs, each held for AudioTestPattern::steppedToneStepSec. Default 100 Hz / 1 kHz / 10 kHz covers the low / mid / high bands for per-band level alignment.
  • Blits — simplified EBU Tech 3304 BLITS (Broadcast Loudness and Identification Tone Sequence). Channel-aware cycle: all-channel tone, sequential per-channel identification, polarity check (odd channels inverted), then silence.
  • EbuLineup — EBU line-up tone: 1 kHz at −18 dBFS with a 3 s on / 2 s off cadence. The standard reference for broadcast level alignment.
  • Dialnorm — broadband pink noise calibrated to a target loudness (default −24 dBFS RMS, approximating −24 LKFS per ITU-R BS.1770). Reads from the same cached pink-noise buffer as PinkNoise but at its own amplitude.
  • Iec60958 — biphase-mark-encoded IEC 60958 professional channel-status block (192 bits) in the PCM domain. Enables bit-exact verification of digital audio transport paths.
