Detailed Description

Abstract base class for stateful audio encoders.

AudioEncoder is a single push-frame / pull-packet codec session: UncompressedAudioPayload frames pushed via submitFrame feed an internal pipeline, and encoded CompressedAudioPayload access units come back out of receivePacket zero, one, or many submits later depending on the codec's frame size and look-ahead.

Encoders expose no codec-family metadata directly — the caller asks the codec wrapper returned by codec() instead. The codec wrapper knows its own name, description, FourCC list, compression type, bitrate mode, and all other spec-level facts; per-backend facts (specific uncompressed input formats, bitrate ceilings, …) live on the registered BackendRecord and are reachable via AudioCodec::encoderSupportedInputs.

The session contract is intentionally identical to VideoEncoder (configure, submit/receive, flush, reset) so pipeline glue (MediaIOTask, QuickTime muxer, RTP packetiser, …) can treat audio and video encoders symmetrically.

Session lifecycle
  1. Resolve a session via AudioCodec::createEncoder.
  2. Call configure with a MediaConfig. (Skip when a config was already supplied to createEncoder.)
  3. For each UncompressedAudioPayload frame, call submitFrame.
  4. After each submit, drain with receivePacket until it returns a null Ptr. Each emitted packet carries this codec's codec() value so downstream code can route it without consulting the encoder again.
  5. When the input stream is exhausted, call flush and keep draining until receivePacket returns a packet whose MediaPayload::Flags::EndOfStream flag is set.
  6. Destroy the encoder.

Implementations are not required to be thread-safe.

virtual void AudioEncoder::requestKeyframe ( )
virtual

Asks the encoder to start a fresh independently decodable run at the next packet.

Mirrors VideoEncoder::requestKeyframe. Most audio codecs are packet-independent and don't need anything here; codecs with internal state (FLAC sync points, Opus DTX boundaries, MP3 bit reservoir) override to force the relevant reset on the next submit. Default implementation is a no-op so pipeline glue can call this unconditionally.
