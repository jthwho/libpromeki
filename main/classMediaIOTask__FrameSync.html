Detailed Description

MediaIOTask backend that resyncs media to a target clock cadence.

ReadWrite MediaIO that wraps a FrameSync instance. The write side pushes source frames into the FrameSync; the read side calls FrameSync::pullFrame, which blocks on the configured clock until the next deadline and synthesises one output frame.

By default a SyntheticClock is used, making the task a non-blocking frame-rate converter suitable for offline / file-based pipelines. For real-time playback, call setClock before opening to substitute a WallClock or device-paced clock (e.g. SDLAudioClock). Callers using an external clock will typically construct the task directly and hand it to MediaIO::adoptTask.

Mode support

Only MediaIO::Transform is supported.

Threading

FrameSync::pullFrame blocks the MediaIO read strand until the clock deadline. This is intentional — the FrameSync task is the pacing stage that governs the pipeline's temporal cadence.

Key Type Default Description
MediaConfig::OutputFrameRate FrameRate Invalid (inherit source) Output frame rate.
MediaConfig::OutputAudioRate float 0 (inherit source) Output audio sample rate in Hz.
MediaConfig::OutputAudioChannels int 0 (inherit source) Output audio channel count.
MediaConfig::OutputAudioDataType Enum AudioDataType Invalid (inherit source) Output audio sample format.

All output format keys default to "inherit from source" — set only the components you want to override. The overflow policy defaults to DropOldest; callers that need Block should construct the task directly, call .setInputOverflowPolicy, and use MediaIO::adoptTask.

MediaIO::Config cfg;
cfg.set(MediaConfig::Type, "FrameSync");
MediaIO *io = MediaIO::create(cfg);
io->setExpectedDesc(upstreamDesc);
io->open(MediaIO::Transform);
io->writeFrame(sourceFrame);
Frame::Ptr outputFrame;
io->readFrame(outputFrame);
io->close();
delete io;
