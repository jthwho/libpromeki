Detailed Description

Immutable descriptor for an audio codec.

Populated by the library for well-known codecs, or by applications via registerData for custom codecs. Captures codec-family facts that do not depend on any specific backend implementation — per-backend capabilities live on the AudioEncoder / AudioDecoder backend records.

Member Data Documentation

◆ frameSizeSamples

List<int> AudioCodec::Data::frameSizeSamples

Frame sizes the codec accepts, in samples.

Populated when the codec has discrete frame size restrictions (Opus at 48 kHz: 120/240/480/960/1920/2880). Empty means "any size the backend can accumulate".

◆ rateControlModes

List<RateControlMode> AudioCodec::Data::rateControlModes

Rate-control modes the codec can be driven in.

Same shape as VideoCodec::Data::rateControlModes. Empty means "constant-quality / fixed codec" (PCM-in-container, FLAC, etc.).

◆ supportedChannelCounts

List<int> AudioCodec::Data::supportedChannelCounts

◆ supportedSampleFormats

List<int> AudioCodec::Data::supportedSampleFormats

Uncompressed AudioFormat IDs the codec accepts / produces.

Stored as int for header-layering reasons (audioformat.h pulls in audiocodec.h itself, so the dependency can't go the other way). Callers wrap each entry as AudioFormat(id) at use time. Empty means "no spec-level constraint on sample format."

◆ supportedSampleRates

List<float> AudioCodec::Data::supportedSampleRates

Sample rates the codec spec permits, in Hz.

Populated only when the codec spec actually restricts its input (Opus: 8000/12000/16000/ 24000/48000). Empty means "any rate".
