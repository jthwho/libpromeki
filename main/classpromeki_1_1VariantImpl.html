Detailed Description

template<typename... Types>
class promeki::VariantImpl< Types >

Type-safe tagged union that can hold any of the types listed in PROMEKI_VARIANT_TYPES.

Variadic template that implements the Variant machinery.

VariantImpl is a thin wrapper around std::variant that adds a Type enum, human-readable type names, and automatic type-conversion logic via the templated get() method. It is not intended to be used directly; instead use the Variant type alias which is instantiated with the concrete type list.

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance — including any combination of set() / get() / reset() — must be externally synchronized. The underlying std::variant is not internally synchronized.
Example
Variant v = 42;
String s = v.get<String>(); // "42"
v.set(String("hello"));
bool valid = v.isValid(); // true
Variant::Type t = v.type(); // Variant::TypeString

Resolves the stored value to an Enum of the given type.

Unlike get<Enum>(), which has no context about the intended enum kind and therefore requires a fully qualified "TypeName::ValueName" string, this method takes a target Enum::Type handle and uses it to resolve several more flexible forms. Accepted inputs:

  • TypeInvalid (default-constructed / missing config key) → returns Enum(enumType), i.e. the type's registered default value, with Error::Ok. This makes the registered default double as the config fallback so callers don't need to repeat it.
  • TypeEnum holding an Enum of enumType → returned directly.
  • TypeEnum holding an Enum of a different type → error, returns an invalid Enum.
  • TypeString "TypeName::ValueName" (qualified) → parsed via Enum::lookup; must match enumType.
  • TypeString "ValueName" (unqualified) → resolved by name against enumType.
  • TypeString decimal integer (e.g. "100", "-1") → wrapped as Enum(enumType, int). The result may be out-of-list but is still valid and round-trippable.
  • Any integer Type (bool/u8..s64) → wrapped as Enum(enumType, int).
  • Anything else → error, returns an invalid Enum.

This is the canonical way to pull a typed Enum out of a config value that may have been authored as a string, an integer, or a typed Enum.

Example
Variant v = cfg.get(ConfigVideoPattern);
Error err;
Enum pat = v.asEnum(VideoPattern::Type, &err);
if(err.isError() || !pat.isValid()) pat = VideoPattern::ColorBars;

Error *  err = nullptr 
) const
inline

Formats the stored value using a type-specific format spec.

Builds the format string "{:<spec>}" and routes it to std::vformat with the held value, so the type's own std::formatter specialization is invoked. This means rich types use their native style keywords (Timecode accepts "smpte" / "smpte-fps" / "field", VideoFormat accepts "smpte" / "named", etc.) and primitives use the standard std::format spec grammar ("x", ".2f", "05d", ...).

An empty spec is short-circuited to get<String>(), which reproduces the existing default toString() behavior for every supported type. Variant alternatives that have no std::formatter specialization fall back to formatting the spec against the value's String form, so width / fill / alignment still work for them.

If spec is malformed (caught as std::format_error), the default String form is returned and err is set to Error::Invalid.

Parameters
specType-specific format spec, without the surrounding {} or leading ':'.
errOptional error output, set to Error::Ok on success or Error::Invalid if the spec could not be applied.
Returns
The formatted String.

static

Constructs a VariantImpl from a JSON value, inferring the best native type.

Mapping rules:

  • null -> invalid (default-constructed) variant
  • boolean -> bool
  • unsigned integer -> uint64_t
  • signed integer -> int64_t
  • floating-point -> double
  • string -> String
  • array -> VariantList (recursive, every element converted via fromJson)
  • object -> VariantMap (recursive, every value converted via fromJson)

Arrays and objects produce a typed nested tree rather than the flattened-string fallback used historically — callers can walk the tree, validate it against a VariantSpec, and round-trip back to JSON without losing structure.

Defined out-of-line below the VariantList / VariantMap class definitions so the body can construct those types.

valThe JSON value to convert.

Converts the stored value to the requested type To.

The method uses std::visit with a compile-time conversion matrix. If the stored type is already To, the value is returned directly. Otherwise an appropriate conversion is attempted (numeric casts, string parsing, toString() calls, etc.). When no conversion path exists, a default-constructed To is returned and err (if non-null) is set to Error::Invalid.

◆ operator==()

template<typename... Types>
bool promeki::VariantImpl< Types >::operator== ( const VariantImpl< Types > &  other) const

Returns true if both variants hold equal values.

Comparison is performed in three tiers:

  1. Same type — uses the type's own operator==.
  2. Cross-type numeric (including bool) — promotes to a common representation: double when either operand is floating-point, otherwise a safe signed/unsigned integer comparison via uint64_t with a negative-value guard.
  3. Cross-type convertible — attempts to convert one operand to the other's type via get<T>(). Both directions are tried; if either conversion succeeds and the converted values compare equal, returns true.
Example
// Same type — direct comparison
Variant(int32_t(42)) == Variant(int32_t(42)); // true
// Cross-type numeric promotion
Variant(int32_t(42)) == Variant(uint32_t(42)); // true
Variant(int32_t(3)) == Variant(3.0); // true
Variant(int32_t(-1)) == Variant(uint32_t(0)); // false (negative guard)
// Cross-type convertible
Variant(int32_t(42)) == Variant(String("42")); // true

Borrows the held value when its type is exactly T.

Returns a non-owning pointer into the underlying std::variant alternative when T matches the current type, or nullptr otherwise. Cheap (no copy) and especially valuable for the heavy alternatives — VariantList, VariantMap, String, Buffer and similar container types — where calling get repeatedly during a tree walk would deep-copy each container.

Example
if (const VariantMap *m = v.peek<VariantMap>()) {
// walk *m without paying for a deep copy
m->forEach([](const String &k, const Variant &v) { ... });
}

Converts complex types to their String representation, leaving simple types unchanged.

Numeric types (integers, floating point, bool) and the Invalid alternative are returned as-is. Every other Variant alternative is converted to String via get<String>(). This is driven by a type trait so new alternatives added to PROMEKI_VARIANT_TYPES are picked up automatically without editing this method.

Returns
A new VariantImpl containing either the original value or its String representation.
