Detailed Description

Fixed-capacity ring of double samples with descriptive statistics.

Used by the MediaIO telemetry path to track recent per-command metrics (read/write execute duration, queue wait, bytes processed, backend-specific keys) so the framework can surface min / max / average / stddev / sampleCount over a bounded window without growing without bound. The container holds plain doubles so it stays cheap to copy and serialize even when many metrics are tracked per stage.

Storage
A single List<double> backs the ring. When fewer than capacity samples have been pushed the list grows; once full, push overwrites the oldest entry. values returns the samples in oldest-first order regardless of the internal write head, so callers see a stable timeline.
String form
The canonical string form is "cap=N:[v1,v2,...,vK]" — capacity plus the bracketed sample list (oldest first). An empty / zero- capacity instance round-trips as "cap=0:[]". This compact form is what the Variant integration uses for JSON / DataStream round-trips, matching every other compound Variant type (AudioStreamDesc, AudioChannelMap, etc.). Sample values are formatted with "%g" so trailing zeros are elided.
Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance — including any combination of push / clear / values — must be externally synchronized. Stats accessors (min, max, average, stddev) walk the buffer once per call and recompute from scratch — there is no cached aggregate, so an external mutex protects every reader equally.

using promeki::WindowedStat::ValueFormatter = std::function<String(double)>

Per-value formatter used by toString to render each numeric component with custom units.

The formatter is invoked once per scalar (avg, stddev, min, max) so callers can humanise values via the Units helpers (e.g. ms, MB) without WindowedStat needing to know the underlying physical unit. When empty (the default), toString falls back to String::number for every value.

Parses the canonical "cap=N:[...]" form produced by toSerializedString.

Accepts the legacy bare-list form "[v1,v2,...]" (capacity inferred from sample count) so older snapshots remain readable. Whitespace inside the brackets is ignored. Returns Error::Invalid on any parse failure; the value is a default-constructed WindowedStat in that case.

◆ push() [1/2]

bool promeki::WindowedStat::push ( const Variant &  v)

Pushes a numeric value extracted from a Variant.

Promotes any of the integer / floating-point Variant types to double and pushes the result. Two domain-specific types receive bespoke handling:

  • Duration is exposed in nanoseconds (matches the unit MediaIOCommand::executeDurationNs uses internally so windowed values stay comparable across strategies and backends).
  • FrameCount honours its sentinel states: FrameCount::unknown / FrameCount::infinity return false rather than push a misleading negative or sentinel value.

Any other Variant type (String, Enum, Variant-of- object, Invalid) returns false without modifying the ring, so the call is safe to apply blindly to heterogeneous stat maps.

Returns
true if the variant was numeric and a sample was pushed; false if it was non-numeric or a non-finite sentinel.

Resets the capacity, dropping the oldest samples if necessary.

Computes every descriptive statistic in a single pass.

Cheaper than calling each accessor individually when a caller wants more than one number — the accessors each rescan the buffer, while stats walks it once and reuses the running mean for the variance.

◆ stddev()

double promeki::WindowedStat::stddev ( ) const

Returns the population standard deviation, or 0.0 when empty.

Population (not sample) variance — the implementation divides by count rather than count-1 because the window is the entire dataset of interest, not a draw from a larger one. Returns 0.0 when fewer than two samples are present.

◆ toSerializedString()

String promeki::WindowedStat::toSerializedString ( ) const

Renders the window as the canonical "cap=N:[v1,v2,...,vK]" round-trip form.

Empty windows render as "cap=N:[]"; sample values are formatted with "%g" so trailing zeros are elided. This is the form parsed by fromString and is the string Variant::get<String>() returns for a TypeWindowedStat payload, so JSON snapshots survive a load / save cycle even though toString itself is no longer round-trippable.

◆ toString()

String promeki::WindowedStat::toString ( const ValueFormatter &  formatter = ValueFormatter()) const

Renders the window as a compact human-readable summary.

Format: "Avg: <a> StdDev: <s> Min: <mn> Max: <mx> WinSz: <n>" — single spaces between fields, no commas, so the line stays scannable in a fixed-width terminal. Each numeric field is rendered through formatter when supplied, so callers wanting unit-aware output (e.g. "4 ms" via Units::fromDurationNs) can inject the right scaler without WindowedStat needing to know the physical unit. When formatter is empty, every value is rendered via String::number.

Empty windows still emit a complete row with every field at zero so callers laying out tables don't have to special-case missing samples.

formatterOptional per-value renderer; when empty, defaults to numeric output.
Returns
The formatted summary line.
