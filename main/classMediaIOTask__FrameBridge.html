Detailed Description

MediaIO backend that publishes/consumes frames via FrameBridge.

A thin adapter that exposes FrameBridge through the standard MediaIO command interface. Both Input and Output modes are supported — Output constructs a bridge with the ring shape derived from config + pendingMediaDesc + pendingAudioDesc; Input connects by name and learns the shape from the handshake.

Config keys
Key Type Default Description
MediaConfig::FrameBridgeName String — Logical bridge name (required).
MediaConfig::FrameBridgeRingDepth int32 2 Ring-buffer depth.
MediaConfig::FrameBridgeMetadataReserveBytes int32 65536 Reserved metadata bytes per slot.
MediaConfig::FrameBridgeAudioHeadroomFraction double 0.20 Audio capacity headroom.
MediaConfig::FrameBridgeAccessMode int32 0600 POSIX file mode for shm + socket.
MediaConfig::FrameBridgeGroupName String "" Group for cross-user access.
MediaConfig::FrameBridgeSyncMode bool true Input-side sync (ACK every TICK).
MediaConfig::FrameBridgeWaitForConsumer bool true Output blocks writeFrame until a consumer attaches.
Stats
Publishes the output-side UUID as the SourceUUID string key on every frame metadata passed to downstream consumers, so logs and schedulers can correlate frames back to the producing bridge instance. Additionally sets Metadata::FrameBridgeTimeStamp to the publisher's queue timestamp so downstream stages can measure cross-process transport latency.
