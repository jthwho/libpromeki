Detailed Description

template<typename T>
class VariantQuery< T >

Compiled predicate that matches any VariantLookup-registered type against an expression.

Parses an expression once into an AST; evaluates it against any T for which VariantLookup<T> has been registered via PROMEKI_LOOKUP_REGISTER. The expression vocabulary mirrors VariantLookup<T>::resolve / ::format, so anything that can be rendered into a burn template or read by the key resolver can also drive a query.

Grammar
expr := or_expr
or_expr := and_expr ( '||' and_expr )*
and_expr := not_expr ( '&&' not_expr )*
not_expr := '!' not_expr | primary
primary := '(' expr ')' | has_form | comparison
has_form := 'has' '(' key ')'
comparison := operand ( relop operand )?
relop := '==' | '!=' | '<' | '<=' | '>' | '>=' | '~' | '~~'
operand := literal | key
literal := integer | float | string | regex | bool
integer := [-]? [0-9]+
float := [-]? [0-9]+ '.' [0-9]+ (exponent allowed)
string := '"' ... '"' (backslash escapes: \\ \" \n \t)
regex := '/' ... '/' (ECMAScript regex; same-char escape)
bool := 'true' | 'false'
key := ident ( '[' N ']' )? ( '.' key )?

Key references
All the forms accepted by VariantLookup<T>::resolve are valid keys. For Frame:
  • Metadata key (behind "Meta." prefix): "Meta.Title"
  • Frame scalar: "ImageCount"
  • Subscripted image/audio: "Image[0].Width", "Audio[0].Meta.Album"
  • Subscripted scalar: "VideoFormat[1]"

For Image and Audio the vocabulary is whatever keys their own VariantLookup registration exposes.

Operators
  • == / != — equality via Variant::operator==. When one side is a bare String literal and the other is a key, the literal is coerced to the key's declared type by consulting VariantLookup<T>::specFor so queries like Meta.Timecode == "01:00:00:00" and Meta.OutputPixelDesc == "RGBA8_sRGB" work out of the box. When no spec is declared, Variant's own cross-type conversion still handles common cases (e.g. Image[0].PixelDesc == "RGBA8_sRGB").
  • < / <= / > / >= — ordering. Tries numeric (both sides convertible to double), else Timecode (when one side is a Timecode), else String lexicographic. Returns false when neither side is ordered comparable.
  • ~ — regex match. RHS must be a regex literal /.../.
  • ~~ — substring match. RHS must be a string literal or a key whose value renders to a string.
  • has(Key) — true when the key resolves to a value on the instance.
  • && / || / ! — short-circuit logical ops.
Examples
auto [q, err] = VariantQuery<Frame>::parse("Meta.Timecode >= \"01:00:00:00\"");
if(err.isError()) throw ...;
for(const Frame::Ptr &f : frames) {
if(q.match(*f)) std::cout << "match\n";
}
VariantQuery<Image>::parse("Width > 1920 && PixelDesc == \"RGBA8_sRGB\"");
VariantQuery<Audio>::parse("Channels == 2 && SampleRate >= 48000");

Template Parameters
TThe target type. Must have VariantLookup<T> registrations for the keys the query references. Explicit template instantiations are provided for Frame, Image and Audio; other types can be instantiated by adding a template class VariantQuery<MyType>; line in the translation unit that owns MyType 's registrations.
