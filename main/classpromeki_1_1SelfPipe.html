Detailed Description

Nonblocking self-pipe used to wake a poll()-based event loop.

Wraps a POSIX pipe pair opened with O_CLOEXEC + O_NONBLOCK. The write end is safe to call from any thread or from an async-signal handler — write() on a pipe is both thread-safe and async-signal-safe, and EAGAIN is ignored since a saturated pipe already has a wake pending. The read end is drained by the loop thread after poll() reports it readable.

On non-POSIX platforms (Windows, Emscripten) the class compiles but all operations are no-ops and isValid returns false. That is enough to let code that uses SelfPipe to escape a blocked wait compile everywhere; the embedder supplies an equivalent wake mechanism on those platforms.

promeki::SelfPipe::SelfPipe ( )

Opens the pipe.

Prefers pipe2(O_CLOEXEC|O_NONBLOCK) on Linux; falls back to pipe() + fcntl on other POSIX systems that lack pipe2. A warning is logged on failure; isValid returns false.
