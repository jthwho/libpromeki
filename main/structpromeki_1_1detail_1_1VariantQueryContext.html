Detailed Description

Evaluation inputs injected into the AST by VariantQuery::match.

The AST itself is type-erased — every concrete node leans on std::function callbacks supplied through this context rather than referencing T directly — so the same parse result can be reused against any T for which a VariantLookup<T> has been registered.

std::function<const VariantSpec *(const String &)> promeki::detail::VariantQueryContext::specFor

Returns the declared VariantSpec for a key, or nullptr.

Used by the AST to coerce string literals on one side of a comparison to the spec-declared type of the key on the other side. VariantQuery::match wires this to VariantLookup<T>::specFor(key). May be null when the caller wants to skip spec-based coercion entirely.
