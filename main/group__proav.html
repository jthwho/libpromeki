Professional audio/video media objects and codecs.

Audio and image containers with format descriptors, file I/O factories, pixel format definitions, codec abstractions, and frame-level media handling for broadcast and post-production workflows.

Implements MediaPayload's VariantLookup dispatch hooks for a concrete leaf.

Place in the public section of a concrete MediaPayload subclass's class body. Expands to overrides of MediaPayload::variantLookupResolve / variantLookupAssign / variantLookupSpecFor that forward to VariantLookup<Self>::resolveDirect / assignDirect / specForDirect, so a caller holding a reference-to-base lands on Self's registry and the upward cascade fills in inherited keys. Using *Direct (not the public dispatch entry) avoids infinite recursion through the virtual.

◆ AudioCodecBackendRegistry

using promeki::AudioCodecBackendRegistry = typedef StringRegistry<"AudioCodecBackend">

Compile-time tag for the AudioCodec backend StringRegistry.

Chosen separately from the VideoCodec backend tag so the two namespaces never alias — an "Native" backend on the audio side is a different typed value from a "Native" video backend.

◆ AudioFileFactory

using promeki::AudioFileFactory = typedef FileFormatFactory<AudioFile>

Factory type for creating AudioFile instances based on file format.

AudioFileFactory is a typedef for FileFormatFactory<AudioFile>. Subclasses register themselves via PROMEKI_REGISTER_AUDIOFILE_FACTORY to handle specific audio file formats (e.g. WAV, FLAC).

◆ AudioStreamDescRegistry

using promeki::AudioStreamDescRegistry = typedef StringRegistry<"AudioStreamDesc">

Compile-time tag for the AudioStreamDesc registry.

Picks out the per-process StringRegistry that backs AudioStreamDesc identity. Stream names like "Main", "Commentary", "Music_FR" all live in this namespace and never collide with other StringRegistry consumers.

◆ VideoCodecBackendRegistry

using promeki::VideoCodecBackendRegistry = typedef StringRegistry<"VideoCodecBackend">

Compile-time tag for the VideoCodec backend StringRegistry.

Chosen separately from the AudioCodec backend tag so the two namespaces never alias — a Backend handle for "Nvidia" on the video side is a different typed value from a (hypothetical) audio "Nvidia" backend.
