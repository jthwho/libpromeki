Detailed Description

Sliding-window rate counter for live telemetry.

RateTracker accumulates frame and byte counts over a short rolling window and exposes them as smoothed rates. It is designed for the MediaIO "live telemetry" path (see MediaIOStats::BytesPerSecond and MediaIOStats::FramesPerSecond): cheap enough to call on every frame, queryable lazily by status displays like mediaplay --stats.

How it works

The tracker holds one current window plus a snapshot of the last completed window. record is lock-free and only does an atomic fetch_add on the two counters. Query functions (bytesPerSecond / framesPerSecond) take a small mutex and check whether the current window has aged past windowMs; if so they promote it to the "last" snapshot and reset the counters. The value returned is:

  • the finished last-window rate while the current window is still short (< half the window), so that a freshly-reset counter does not report zero;
  • the current-window rate once enough data has accumulated;
  • zero until at least one window has ever been observed.

The window is not resampled on every query — rotation happens only when the elapsed time crosses the full window boundary — so repeated queries inside a window see a stable value.

Thread safety

record is safe to call from any thread without coordination. Query functions are also safe but serialize on an internal mutex. reset must not race with in-flight record calls.
