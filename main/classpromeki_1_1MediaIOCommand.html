Detailed Description

Base class for MediaIO commands.

Commands are shared via SharedPtr<MediaIOCommand, false> with COW disabled — derived types are owned via the proxy path and never cloned.

Commands flow through the framework in three stages:

  1. Built by a public API method on MediaIO / MediaIOSource / MediaIOSink / MediaIOPortGroup. The Input fields are populated from the caller's arguments and from the MediaIO's pre-open / pending state. A MediaIORequest is constructed atop the command, arming the completion hook.
  2. Dispatched by MediaIO::submit(). The default strategy posts to a strand; concrete strategy classes (added in Phase 10) route the command differently. The strategy invokes executeCmd(...) on the backend, which populates the Output fields and returns an Error.
  3. Completed by MediaIO::completeCommand(). Cache writes happen first, then cache-derived signals fire, then the command's onComplete hook fires (resolving the MediaIORequest). This ordering is fixed so MediaIORequest::then() callbacks always observe up-to-date cached state.

◆ markCompleted()

void promeki::MediaIOCommand::markCompleted ( )

Marks the command completed and wakes every waiter.

Invoked by MediaIO::completeCommand() after cache writes and signal emission have run. Sets the _completed flag, broadcasts the wait condition so all waitForCompletion() callers unblock, and fires the registered continuation callback exactly once (marshalled through the captured EventLoop when one was supplied).

Idempotent: a second call is a no-op. Strategies may safely re-invoke it on a cancellation path even if the dispatch path already fired.

Called by MediaIORequest::then(). ◆ cancelled

Atomic<bool> promeki::MediaIOCommand::cancelled {false}

Cancellation flag — atomic for cross-thread polling.

Set by MediaIORequest::cancel(). Strategies read this just before dispatching the command — if set, the command short-circuits with Error::Cancelled instead of calling the backend's executeCmd. Late cancellations (after dispatch has begun) are ignored on non-blocking strategies; DedicatedThreadMediaIO uses the flag together with cancelBlockingWork() to interrupt blocking I/O.

◆ result

Error promeki::MediaIOCommand::result = Error::Ok

Outcome of executeCmd() — set by the framework.

The strategy class records the Error returned by the backend's executeCmd() invocation here before calling MediaIO::completeCommand(). Backends never write to this field directly — they return an Error from executeCmd() and the framework captures it. When the command never reaches the backend (cancelled before dispatch) the strategy sets this to Error::Cancelled.

◆ stats

MediaIOStats promeki::MediaIOCommand::stats

Telemetry — what happened on this command run.

Single MediaIOStats container that serves both per-command and instance-wide reporting:

  • For the standard commands (Open / Close / Read / Write / Seek / Params) this is per-command telemetry. The framework populates the timing keys (MediaIOStats::ExecuteDuration, MediaIOStats::QueueWaitDuration) automatically; backends may add per-command keys (MediaIOStats::BytesProcessed, backend-specific) from inside executeCmd(...).
  • For MediaIOCommandStats this is the instance-wide aggregate. The backend's executeCmd(MediaIOCommandStats &) populates the cumulative keys (MediaIOStats::FramesDropped, MediaIOStats::BytesPerSecond, etc.) and MediaIO::completeCommand overlays the framework-managed standard keys before resolution.

Callers retrieve the snapshot from the resolved MediaIORequest via .stats() — the same accessor regardless of which command type is underneath.

◆ urgent

bool promeki::MediaIOCommand::urgent = false

Hint for the strategy to prioritize this command.

When true the strategy should expedite the command — for example, the default SharedThreadMediaIO posts urgent commands to the front of its strand's queue so a stats poll does not block behind a deep backlog of real I/O. Strategies are free to ignore the hint when their executor has no notion of priority. Set by the public API method that builds the command (today only MediaIO::stats); backends never touch this.
