Detailed Description

Instance-wide stats query command.

Marker subclass with no extra fields — backends populate the cumulative aggregate keys directly into the inherited MediaIOCommand::stats container, and MediaIO::completeCommand overlays the framework-managed standard keys (rate trackers, drop / repeat / late counters, MediaIOStats::PendingOperations) before resolving the request.

Backends that want to expose backend-specific aggregate keys override executeCmd(MediaIOCommandStats &) and write into cmd.stats. The default implementation is a no-op (returns Error::Ok with whatever the framework has already populated).
