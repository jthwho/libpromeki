Detailed Description

Abstract clock interface.

A Clock is a reference-counted, polymorphic time source with a managed now that applies filtering, pause accounting, fixed offset and a monotonic clamp on top of the subclass's pure-virtual raw reading.

Construction

Derived clocks pass their ClockDomain, any fixed offset from that domain's epoch, their pause capability, and an optional ClockFilter pointer. The clock takes ownership of the filter and deletes it on destruction. Pass nullptr for no filtering.

Sharing

Clocks are natively reference-counted. Callers manage lifetime via Clock::Ptr . Clocks are not copyable — the internal clone helper asserts if invoked.

Implementations
  • WallClock — std::chrono::steady_clock, for local playback.
  • SyntheticClock — frame-count driven, for offline pipelines.
  • SDLAudioClock — derived from an SDL audio device's drain rate.
  • MediaIOClock — derived from a MediaIO's frame position.
Thread Safety
now / domain / offset / isPaused / isMonotonic are intended to be safe to call from any thread; subclass raw implementations must respect that contract. Mutators (setOffset, pause, resume, setFilter) require external synchronization unless the concrete subclass documents otherwise. The reference-counted Ptr makes lifetime safe to share across threads.

◆ now()

Result< MediaTimeStamp > promeki::Clock::now ( ) const

Returns the current time as a MediaTimeStamp.

Applies, in order: raw, then the filter (if any), then the paused-snapshot freeze (if paused), then the combined fixed + paused offsets, then a monotonic clamp against the last value returned. The returned MediaTimeStamp carries this clock's domain and the total offset that was in effect.

Returns
The timestamp, or an error propagated from raw().

◆ onPause()

virtual Error promeki::Clock::onPause ( bool  paused)

Called when pause state changes.

Subclasses whose underlying source actually stops on pause (ClockPauseMode::PausesRawStops) override this to start and stop the hardware. Called while the Clock's internal pause mutex is held, so implementations must not call back into the Clock's public API. Default returns Error::Ok.

◆ setPause()

Error promeki::Clock::setPause ( bool  paused)

Sets the pause state.

Calls onPause to let the subclass stop or resume any underlying resource. On pause, snapshots the filtered raw value; on resume, adds the accumulated raw delta to the paused-offset accumulator so that the reported time resumes seamlessly.

Parameters
pausedTarget state.
Returns
Error::NotSupported if !canPause(), otherwise forwards the result of onPause (which is typically Error::Ok).

◆ sleepUntil()

Error promeki::Clock::sleepUntil ( const MediaTimeStamp &  deadline) const

Blocks until now would return at or past deadline.

Verifies deadline is in this clock's domain and translates its value into the current raw timebase by adding the clock's current fixed + paused offsets, so a mid-wait offset change does what callers expect ("wake when @ref now reads X"). The stored MediaTimeStamp::offset is informational.

Parameters
deadlineWhen to wake, expressed in this clock's domain.
Returns
Error::Ok on successful wake, Error::ClockDomainMismatch when the deadline belongs to another domain, Error::ClockPaused when the clock is paused (either at entry or during the wait — the caller should retry after resume), or whatever error the subclass's sleepUntilNs produces.
