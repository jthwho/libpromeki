Detailed Description

Convenience wrapper that mounts the standard set of debug modules.

DebugServer is intentionally thin: it owns one HttpServer, one HttpApi attached to it, and exposes a one-call helper (installDefaultModules) that stitches together every installer in debugmodules.h under the canonical "/api" base prefix. Applications that want to cherry-pick which diagnostics they expose — or attach them to their own user-facing server — can skip this class and call the installer functions directly on any HttpApi.

Because the debug routes are registered through the embedded HttpApi, every endpoint shows up in the api explorer, the /_catalog JSON list, and the /_openapi document automatically — the debug API is itself self-describing.

The base prefix defaults to "/api", which lays out the URL space as follows:

* /                  → 302 redirect to /api/promeki/
* /api/               → built-in interactive API explorer
* /api/_catalog      → promeki-native API catalog
* /api/_openapi      → OpenAPI 3.1 doc covering the registered API
* /api/promeki/      → debug UI index.html
* /api/promeki/build → build info JSON
* /api/promeki/env   → environment snapshot
* /api/promeki/log   → logger control
* ...
*
Lifetime
Construct a DebugServer on whatever EventLoop should run its handlers — typically the application's main loop. Call listen with the address to bind, then leave it alive for the lifetime of the process. The destructor closes the listener and tears down all live connections.
Example
Application app(argc, argv);
DebugServer dbg;
dbg.installDefaultModules();
if(Error err = dbg.listen(8085); err.isError()) {
promekiWarn("debug server failed: %s", err.toString().cstr());
}
return app.exec();

◆ installDefaultModules()

void promeki::DebugServer::installDefaultModules ( )

Mounts the full promeki debug surface.

Calls HttpApi::installPromekiAPI to install every debug installer under <prefix>/promeki/, mounts the API's interactive explorer at <prefix>/ alongside /_catalog and /_openapi, and adds a 302 redirect from "/" to the debug UI at <prefix>/promeki/.

The root redirect is exclusive to DebugServer — applications that install the promeki API into their own server (via HttpApi::installPromekiAPI) keep control of "/" themselves.

Safe to call exactly once before listen.

◆ parseSpec()

static Result< SocketAddress > promeki::DebugServer::parseSpec ( const String &  spec)
static

Parses an environment-variable spec into a SocketAddress.

Accepts the forms used by PROMEKI_DEBUG_SERVER:

  • ":1234" → DefaultBindHost + port (loopback).
  • "host:port" → exactly that.
  • "[::1]:port" → IPv6 with brackets.

Returns the parsed address on success, or a default SocketAddress and Error::Invalid on parse failure.
