Detailed Description

Application-wide state for the promeki library.

Provides global application identity (name, UUID, command-line arguments) used by other parts of the library such as UUID v3/v5 generation.

All accessors are static and work whether or not an instance has been constructed. Optionally, an Application object can be created on the stack in main() to capture argc/argv and register as the current instance (similar to QApplication):

int main(int argc, char **argv) {
Application app(argc, argv);
Application::setAppName("myapp");
// ...
}

If no instance is created, the static accessors operate on default-constructed internal state.

Thread Safety
The static accessors are intended to be set once at startup (typically in main before worker threads spawn) and read thereafter from any thread. Mutating setters (setAppName, setAppUuid, setAppArguments) without external synchronization is unsafe once worker threads exist. mainThread / mainEventLoop are safe to call from any thread.

◆ exec()

int promeki::Application::exec ( )

Runs the main event loop until quit() is called.

Thin wrapper around mainEventLoop()->exec(). The Application's own EventLoop is constructed with the Application itself; any Subsystem objects (SDL, TUI, ...) installed before this call have registered their I/O sources on that loop, so exec() blocks until one of them or a posted quit() causes the loop to exit.

Returns
The exit code passed to quit (or the exit code passed to EventLoop::quit() directly).

◆ quit()

static void promeki::Application::quit ( int  exitCode = 0)
static

Requests the application to quit.

If a QuitRequestHandler has been installed via setQuitRequestHandler, it is invoked first. When the handler returns true, it has accepted responsibility for eventually driving the actual quit itself (typically after some async shutdown completes) and this call neither sets shouldQuit nor wakes the main EventLoop. When the handler returns false, or no handler is installed, the call proceeds with the default quit behaviour: the exit-code and shouldQuit flag are set and the main EventLoop is poked so any blocked EventLoop::exec unwinds.

Parameters
exitCodeThe exit code (retrievable via exitCode).

◆ refreshCrashHandler()

static void promeki::Application::refreshCrashHandler ( )
static

Re-snapshots the crash handler's install-time state.

CrashHandler takes a snapshot of various process state at install time (command line, environment, library options, registered MemSpaces, ...) so the signal handler can write a report without touching any non-signal-safe APIs. State that changes after install() — such as newly registered MemSpaces or updated LibraryOptions — will not appear in crash reports until the snapshot is refreshed.

This method refreshes the snapshot if (and only if) the crash handler is currently installed. If the crash handler was never installed or has been explicitly uninstalled, this is a no-op — it will not install the handler.

Typical use: call once during application startup after all subsystems have registered their custom MemSpaces.

int main(int argc, char **argv) {
Application app(argc, argv);
initMyGpuBackend(); // registers a "GPU" MemSpace
initMySharedMemPool(); // registers another MemSpace
Application::refreshCrashHandler();
// ... run event loop ...
}

◆ setQuitRequestHandler()

static void promeki::Application::setQuitRequestHandler ( QuitRequestHandler  handler)
static

Installs a handler invoked on every quit call.

Use this to defer the actual event-loop teardown until an async shutdown has completed — for example, running MediaPipeline::close with block=false and re-invoking quit from the pipeline's closedSignal. Pass an empty function to clear the handler.

Only one handler may be installed at a time; a second call replaces the first. The handler runs on the calling thread of the quit invocation (typically the signal-delivery thread), so it must be thread-safe — the canonical pattern is to post work to the main EventLoop from inside the handler rather than doing it inline.

Parameters
handlerThe handler (or an empty function to clear).

◆ startDebugServer() [1/2]

static Error promeki::Application::startDebugServer ( const SocketAddress &  address)
static

Starts the embedded debug HTTP server on address.

Constructs (if needed) the DebugServer, calls DebugServer::installDefaultModules so the standard URL space (/promeki/debug ...) is mounted, and binds to address. Calling this while the debug server is already listening fails with Error::AlreadyOpen — call stopDebugServer first to rebind.

The DebugServer's EventLoop affinity is inherited from the calling thread at the moment of the first startDebugServer call (typically the main thread, when invoked from within or just after Application construction).
