Detailed Description

Standalone video test pattern generator.

Generates video test patterns into Image objects and optionally burns a text overlay (including a timecode line) on top of the pattern. Supports dual-mode operation: create a new Image with the pattern, or render into an existing Image. Motion support via an offset parameter that advances per frame.

Performance

When the configured pattern is static (i.e. motionOffset == 0 and the pattern is not Noise), create() caches the rendered background on the first call and reuses it on subsequent calls with the same ImageDesc, pattern, and solid color. When burn-in is enabled the cached background is copied (via COW plane detach) and the text is drawn onto the copy, leaving the cache untouched so the next frame can reuse it again. When burn-in is disabled create() returns the cache by shallow copy with no pixel work at all.

For dynamic patterns (Noise, or when motionOffset != 0) the background is rendered fresh each call.

This class is not thread-safe. External synchronization is required for concurrent access.

Burn-in

Burn-in is a separate pass: callers render the background via create(), then draw a text overlay via applyBurn() once the text content has been determined. This lets the caller assemble upstream context (e.g. a Frame with metadata) before deciding what to burn in. The overlay text may contain '\n ' to span multiple lines; each line is centered horizontally within the bounding box and the whole block is positioned according to burnPosition(). Burn-in uses the FastFont glyph cache so repeated draws of the same characters at the same size/color are a memcpy per scanline with no per-pixel alpha blending — see Font Rendering. The image must be in a paintable pixel format (PixelDesc::hasPaintEngine() == true); applyBurn() returns Error::NotSupported on non-paintable formats.

Example
VideoTestPattern gen;
gen.setPattern(VideoPattern::ColorBars);
// Static pattern, no burn: the first create() renders, subsequent
// calls return the cached image via shallow copy.
ImageDesc desc(1920, 1080, PixelDesc::RGB8_sRGB);
Image a = gen.create(desc); // renders colorbars
Image b = gen.create(desc); // returns cache
// Add a burn-in line. A font filename may optionally be set;
// without it the library's bundled default font is used.
gen.setBurnFontSize(48);
a.ensureExclusive();
gen.applyBurn(a, "01:00:00:00\nmediaplay");

Error VideoTestPattern::applyBurn ( Image &  img,
const String &  burnText 
) const

Draws a text overlay onto an existing image.

Splits burnText on '\n ' and renders each line stacked under the burn font, centered horizontally within the bounding box and positioned according to burnPosition. When burnEnabled is false or burnText is empty this is a no-op and returns Error::Ok. The image must already be detached (call Image::ensureExclusive before invoking).

Parameters
imgTarget image. Must be valid and in a paintable pixel format (PixelDesc::hasPaintEngine() == true). Mutated in place.
burnTextText to draw. May contain '\n ' for multi-line.
Return values
Error::OkOn success or no-op.
Error::InvalidArgumentimg is invalid.
Error::NotSupportedThe image's pixel format has no paint engine.
Error::FontUnavailableThe burn font could not be loaded.

Creates a new Image and renders the pattern into it.

For static patterns (not Noise) called with motionOffset == 0, the background is rendered once and cached; subsequent calls with the same desc return a shallow copy of the cache.

The currentTimecode is only used by the AvSync pattern to choose between the marker (white, tc.frame()==0) and non-marker (black) cached frames. Burn-in is a separate pass — see applyBurn.

◆ setBurnDrawBackground()

Enables/disables the background rectangle drawn behind the burn text.

When enabled (default), a filled rect in the configured background color is drawn behind the text for legibility. When disabled, the FastFont character cells are drawn directly over the pattern — they still include the background color inside each cell, but without surrounding padding.

◆ setBurnEnabled()

Enables or disables text burn-in.

When false, applyBurn() is a no-op regardless of the text passed in. The flag exists as a convenience gate for callers that drive a VideoTestPattern from a config and want a single place to turn the overlay on or off.
