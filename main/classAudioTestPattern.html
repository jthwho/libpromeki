Detailed Description

Multi-channel audio test pattern generator.

Generates audio test patterns with per-channel pattern assignment. Every channel is driven by its own AudioPattern value pulled from the configured EnumList. If the channel-mode list is shorter than the stream's channel count, any remaining channels are silenced; if the list is longer, the extra entries are ignored.

Supported per-channel patterns:

  • Tone — continuous sine at toneFrequency.
  • Silence — constant zero.
  • LTC — Linear Timecode audio for the current frame timecode.
  • AvSync — one-shot tone burst on tc.frame() == 0, silent otherwise. Pairs with the picture AvSync marker for A/V drift detection.
  • SrcProbe — 997 Hz reference sine. The prime-relative frequency reveals sample-rate-conversion artifacts on the decode side.
  • ChannelId — channel-unique sine at channelIdBaseFreq() + ch * channelIdStepFreq(), so a downstream consumer that sees channels out of order can identify each one by frequency.
  • Sweep — linear frequency sweep.
  • Polarity — positive half-sine impulse for polarity verification.
  • SteppedTone — cycles through discrete frequency steps.
  • Blits — EBU BLITS identification / polarity / silence cycle.
  • EbuLineup — 1 kHz at −18 dBFS, 3 s on / 2 s off.
  • Dialnorm — pink noise at a calibrated loudness.
  • Iec60958 — biphase-mark channel-status block in PCM.

Call configure after changing any pattern parameter and before calling create or render. The class is not thread-safe; external synchronization is required for concurrent access.

Example
AudioTestPattern gen(AudioDesc(48000, 4));
// LTC on ch0, AvSync click on ch1, SRC probe on ch2, silence on ch3.
EnumList modes = EnumList::forType<AudioPattern>();
modes.append(AudioPattern::LTC);
modes.append(AudioPattern::AvSync);
modes.append(AudioPattern::SrcProbe);
modes.append(AudioPattern::Silence);
gen.setChannelModes(modes);
gen.setToneFrequency(1000.0);
gen.setToneLevel(AudioLevel::fromDbfs(-20.0));
gen.setLtcLevel(AudioLevel::fromDbfs(-20.0));
gen.configure();
Audio audio = gen.create(samplesPerFrame, currentTimecode);

◆ noiseBufferSeconds()

double AudioTestPattern::noiseBufferSeconds ( ) const
inline

Returns the noise buffer length in seconds.

The white/pink noise channels share a pre-generated sample buffer of this duration, cached in configure. The default is ten seconds — long enough that a casual listener can't pick out the loop repetition, and cheap enough at typical sample rates (≈1.9 MB at 48 kHz float). The buffer is generated with a raised-cosine crossfade at the wrap boundary so the loop is seamless; the longer the buffer the less often the (already-inaudible) transition fires.
