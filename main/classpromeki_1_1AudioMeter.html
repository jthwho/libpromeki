Detailed Description

Pluggable per-block metering callback.

AudioMeter is the extension point through which AudioBuffer surfaces every sample it sees on the float-domain processing path to an observer. The buffer calls process exactly once per push that takes the via-float route, with frames frames of channels-channel interleaved float samples already in normalized [-1, 1] range and after channel remap and gain have been applied.

Implementations can compute whatever meter shape the application needs — peak, RMS, EBU R 128 LU, true-peak, K-weighted loudness, etc. — and stash the result wherever readers expect to find it. The library ships AudioPeakRmsMeter as a reasonable default.

Thread Safety
process is invoked from the push thread. Readout methods defined by the concrete subclass are responsible for being safe to call concurrently from any other thread; the default AudioPeakRmsMeter implementation uses lock-free atomics for exactly this reason.
