Detailed Description

MediaIOTask writer that plays frames through SDL (legacy path).

Deprecated:
Use SDLPlayerTask / createSDLPlayer instead. SDLPlayerOldTask is the original FramePacer-based implementation, retained only so the new FrameSync-based sink can be validated side-by-side. It will be removed once FrameSync is proven in the field; see devplan/proav_framesync.md Phase 4. New code must not depend on it.

SDLPlayerOldTask is the MediaIO analog of SDLDisplayNode: it is a write-only sink backend that consumes Frame objects via MediaIO::writeFrame() and displays their video through an application-provided SDLVideoWidget and (optionally) plays their audio through an application-provided SDLAudioOutput.

Creation

Unlike registered backends, SDLPlayerOldTask cannot be created via MediaIO::create() because it needs raw pointers to the video widget and audio output — values that can't be carried in a MediaConfig. Use the free factory function createSDLPlayerOld() to construct a ready-to-use MediaIO that adopts a properly configured SDLPlayerOldTask.

Compressed input — automatic decode

SDLPlayerOldTask accepts compressed video formats (JPEG, JPEG XS, and anything else with a registered ImageCodec) as input and decodes them transparently before the frame is handed to the widget. The decode runs inside executeCmd(Write) on the MediaIO strand worker thread via Image::convert, which dispatches to the codec registry by name — the task does not hard-code any codec. The decode target is PixelDesc::RGBA8_sRGB so the decoded image can be handed to SDLVideoWidget directly, regardless of which codec was involved.

Running the decode on the strand thread (rather than the main thread) keeps the UI responsive for heavy codecs: the main thread only ever sees an already-decoded image ready for texture upload. This means a receiver like mediaplay -i foo.sdp -o SDL just works for any format the codec layer can decode, with no CSC stage needed in front of the SDL sink.

If a decode fails (missing codec, malformed bitstream) the offending frame is counted as dropped and the task continues processing subsequent frames.

Uncompressed input with a pixel layout SDL cannot upload directly (e.g. 10-bit planar YUV) is handled by the widget's own CSC fallback — the task does not pre-convert those.

Open-time setup

For a writer, MediaIO delivers the caller-provided MediaDesc and (optionally) AudioDesc to the task in MediaIOCommandOpen. SDLPlayerOldTask reads the frame rate from the MediaDesc and configures and opens the SDLAudioOutput using either the explicit pendingAudioDesc or the first entry in pendingMediaDesc.audioList(). Callers must therefore set the media description before opening:

MediaIO *player = createSDLPlayerOld(&videoWidget, &audioOutput);
player->setExpectedDesc(sourceMediaIO->mediaDesc());
player->setExpectedAudioDesc(sourceMediaIO->audioDesc());
player->open(MediaIO::Sink);

Teardown ordering

The main thread's render callable captures a pointer to the task. The caller must therefore close() the MediaIO and let the main event loop drain (for example by quitting the SdlSubsystem before delete) before destroying the owning MediaIO. Destroying the MediaIO while a render callable is still queued on the main thread's event loop is undefined behavior — mirroring the same constraint that applies to SDLDisplayNode.

Constructs a player task.

Neither pointer is owned. video may be nullptr if the caller has no interest in displaying video (audio-only playback). audio may be nullptr if the caller does not want audio, or if the source will never carry audio.

The player is always paced to the source's frame rate. When useAudioClock is true and an audio output is available, timing is derived from the audio device's consumption rate; otherwise the system's monotonic wall clock is used.

Parameters

audioAudio output for playback, or nullptr.
