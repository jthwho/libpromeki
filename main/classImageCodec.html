Detailed Description

Abstract base class for image codecs.

ImageCodec provides an interface for encoding and decoding images. Concrete subclasses implement specific algorithms (JPEG, PNG, etc.).

Codec instances are configured once (quality, subsampling, etc.) then encode or decode multiple frames without reconfiguration.

Subclasses are registered via PROMEKI_REGISTER_IMAGE_CODEC and looked up by name at runtime.

This class is not thread-safe. Each thread should use its own codec instance.

Example
ImageCodec *codec = ImageCodec::createCodec("jpeg");
if(codec) {
Image compressed = codec->encode(sourceImage);
delete codec;
}

◆ configure()

virtual void ImageCodec::configure ( const MediaConfig &  config)
virtual

Applies caller-supplied options to this codec instance.

Lets a caller (typically Image::convert) hand a MediaConfig to a freshly-constructed codec without having to know which keys the concrete subclass cares about. Each subclass overrides this to read its own well-known MediaConfig keys (e.g. JpegQuality, JpegSubsampling) and updates its internal state. Keys the codec doesn't recognize are ignored, so the same MediaConfig can be reused across pipeline stages without filtering.

The default implementation is a no-op so codecs that have no configurable knobs don't need to override.

Calling configure() between encode/decode operations is allowed; the codec is expected to honor the new settings on subsequent calls.

Parameters
configCaller-supplied configuration database. May be empty.
Example
ImageCodec *codec = ImageCodec::createCodec("jpeg");
MediaConfig cfg;
cfg.set(MediaConfig::JpegQuality, 95);
codec->configure(cfg);
Image jpeg = codec->encode(rgb);
delete codec;
