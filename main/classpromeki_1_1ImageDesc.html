Detailed Description

Describes the format and layout of a single image.

ImageDesc encapsulates image dimensions (Size2Du32), pixel description, line padding and alignment, interlace mode, and associated metadata. It is used by Image and MediaDesc to define the properties of image data.

Example
ImageDesc desc(1920, 1080, PixelFormat::RGBA8_sRGB);
size_t stride = desc.pixelFormat().lineStride(0, desc);
int planes = desc.planeCount();

Compares structural image fields, ignoring metadata.

Covers size, line padding / alignment, scan mode, and pixel description — every field that influences the byte layout or colour interpretation of the image. Metadata (colour tags, creator fields, user data) is deliberately excluded so negotiation between stages can compare "are these the same shape" without being derailed by cosmetic metadata differences.

◆ fromSdp()

static ImageDesc promeki::ImageDesc::fromSdp ( const SdpMediaDescription &  md)
static

Derives an ImageDesc from an SDP media description.

Interprets the RTP payload encoding name in the a=rtpmap attribute and (when applicable) the a=fmtp parameters to build a video ImageDesc. Supported encodings:

  • jxsv (RFC 9134 JPEG XS) — geometry comes from width= / height= in the fmtp line; sampling and depth pick one of the JPEG_XS_* PixelFormat entries. Defaults to JPEG_XS_YUV10_422_Rec709 when fmtp is incomplete.
  • raw (RFC 4175 uncompressed) — geometry and pixel layout come from the fmtp line (sampling, depth, width, height). Not yet implemented; this overload returns an invalid ImageDesc for now.
  • JPEG (RFC 2435 MJPEG) — geometry is carried in the RFC 2435 packet header rather than SDP, so this overload returns an invalid ImageDesc. Callers that need MJPEG should read the geometry off the first packet.

Any other encoding (or a non-video md) yields an invalid ImageDesc. The interpretation is RTP-specific — these encoding names come from the IANA RTP payload registry and their respective RFCs — and lives on ImageDesc because ImageDesc is the target type and the rest of proav already depends on the network layer.

Parameters
mdThe SDP media description to interpret.
Returns
A populated ImageDesc, or an invalid ImageDesc on any failure.

Resolves a JPEG PixelFormat from ST 2110-20 colorimetry and RANGE parameters plus subsampling.

Used by the deferred JPEG geometry path in the RTP reader to pick the correct PixelFormat variant when colorimetry/RANGE are available from the SDP fmtp line. When colorimetry is empty, defaults to Rec.601 full range (JFIF standard).

isRgbTrue for RGB (overrides subsampling).

◆ toSdp()

SdpMediaDescription promeki::ImageDesc::toSdp ( uint8_t  payloadType) const

Builds an SDP media description from this ImageDesc.

The inverse of fromSdp. Populates the returned SdpMediaDescription with m=video, an a=rtpmap line, and (where applicable) an a=fmtp line carrying the format-specific parameters:

  • JPEG — rtpmap JPEG/90000, fmtp with ST 2110-20 colorimetry and RANGE derived from the PixelFormat's ColorModel and component range.
  • JPEG XS — rtpmap jxsv/90000, fmtp with sampling, depth, width, height, colorimetry, and RANGE per RFC 9134.
  • raw — rtpmap raw/90000, fmtp with sampling, depth, width, height, colorimetry, and RANGE per RFC 4175 / ST 2110-20.

Returns an empty SdpMediaDescription if the ImageDesc is invalid or the PixelFormat is not supported.

Parameters
payloadTypeRTP payload type (0-127).
Returns
A populated SdpMediaDescription, or an empty one on failure.
