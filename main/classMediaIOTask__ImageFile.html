Detailed Description

MediaIOTask backend for image files and image sequences.

Wraps the ImageFile / ImageFileIO subsystem to provide read and write access to image files through the MediaIO interface. Supported single-image formats include DPX, Cineon, TGA, SGI, PNM, PNG, and raw YUV variants.

Single-image mode

When the filename resolves to a plain file (no mask placeholder, no .imgseq extension), the backend treats it as a single-frame source or sink:

  • Readers load the file on open() and deliver the same cached frame on every subsequent read. With the backend's default step of 0, reads continue indefinitely; with step != 0, a second read returns EndOfFile.
  • Writers save exactly one file per writeFrame() call; each call overwrites the target file.
  • canSeek is false, frameCount is 1 (reader) or 0 (writer).
Sequence mode

When the filename contains a mask placeholder — either hash-style ("shot_####.dpx", "shot_#.dpx") or printf-style ("shot_%04d.dpx", "shot_%d.dpx") — or refers to an .imgseq JSON sidecar file, the backend treats it as an image sequence:

  • Readers scan the containing directory once on open() to detect the head and tail frame numbers (unless the sidecar supplies them), then lazily load each requested frame on read. canSeek is true and frameCount is the length of the detected range.
  • Writers synthesize the output filename for each writeFrame() call from the mask, starting at the configured head (default 1).
  • Step, reverse playback and seekToFrame() all work as expected.

The format of each individual image in the sequence is taken from the mask's suffix extension (e.g. ".dpx" maps to ImageFile::DPX).

The .imgseq sidecar

An .imgseq file is a small JSON document describing one sequence in its containing directory. See the ImgSeq class for the full schema. The pattern inside the sidecar can use either mask syntax and is resolved relative to the sidecar's directory. The sidecar can also supply an optional frame rate, video size, pixel descriptor, and sequence-level metadata. Multiple sidecars may coexist in a single directory, each describing a different sequence.

Frame rate

Still images have no intrinsic temporal rate, so the backend reports a frame rate chosen by a priority chain:

  1. Writer: cmd.pendingMediaDesc.frameRate() if valid.
  2. ImgSeq sidecar frameRate (reader only).
  3. MediaConfig::FrameRate (always populated — the backend's default config pre-seeds it with DefaultFrameRate).

The resolved source is recorded in the returned metadata under Metadata::FrameRateSource as one of "file" (from a sidecar or writer-supplied MediaDesc) or "config" (from the MediaConfig::FrameRate entry, whether it came from the backend default or a caller override). Callers that care about whether a real frame rate was available can inspect that metadata entry.

Builds a MediaIO::FormatDesc for a single ImageFileIO backend.

Driven by io 's own extensions(), canLoad(), canSave(), and mediaIoName() accessors — no hard-coded per-format table. Called by ImageFileIO::registerImageFileIO to piggy-back a MediaIO registration onto every ImageFileIO backend registration, which keeps the two registries in sync without suffering from static-init ordering across translation units.
