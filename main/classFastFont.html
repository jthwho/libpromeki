Detailed Description

High-performance cached font renderer.

FastFont is optimized for repeated rendering of the same characters at the same size and color. Each glyph is rasterized once with FreeType, then pre-rendered into a small Image in the target pixel format using the configured foreground and background colors. The cached glyph covers the full character cell (advance width by line height), so subsequent draws are a memcpy per scanline with no alpha blending at render time.

When to use
Use FastFont for high-frame-rate overlays such as timecodes, HUD text, or any scenario where the same characters are drawn repeatedly across many frames. It is the fastest rendering path available.
Tradeoffs
  • Glyph cells are opaque: the background color fills the entire cell area, so FastFont is not suitable for transparent text overlays where the underlying image must show through.
  • Higher memory usage due to the per-glyph image cache.
  • Any change to font properties (filename, size, colors, paint engine pixel format) invalidates the entire cache.
Example
Image img(1920, 1080, PixelDesc::RGB8_sRGB);
FastFont ff(img.createPaintEngine());
ff.setFontFilename("/path/to/font.ttf");
ff.setFontSize(48);
ff.setForegroundColor(Color::White);
ff.setBackgroundColor(Color::Black);
ff.drawText("01:02:03:04", 100, 200);

void FastFont::onStateChanged ( )

Called when any configuration property changes.

Subclasses override this to invalidate caches or mark internal state as dirty. The default implementation does nothing.

Note
This is not called when setPaintEngine() is invoked with an engine that has the same pixel format pointer as the current one.
