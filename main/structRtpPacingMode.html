Detailed Description

Well-known Enum type for RTP sender pacing mode.

Selects how the RTP sink stages space packets out over time. Drives the MediaConfig::RtpPacingMode config key and the equivalent runtime path inside MediaIOTask_Rtp.

  • Auto — pick the best available mechanism at open time. On Linux this resolves to KernelFq; on other platforms it falls back to Userspace. This is the default — callers only need to set an explicit mode when they want to override the platform default (e.g. None for loopback / LAN tests, TxTime for ST 2110-21 deployments).
  • None — burst all packets at once. Appropriate only for loopback / LAN tests or when the downstream network is guaranteed to absorb the burst.
  • Userspace — pace by sleeping between sends (the RtpSession::sendPacketsPaced() path). Works everywhere without kernel configuration but ties up the worker thread during the pacing window.
  • KernelFq — push the rate to SO_MAX_PACING_RATE and let the fq qdisc space the packets with zero per-packet CPU cost.
  • TxTime — per-packet SCM_TXTIME deadlines via the ETF qdisc. Only enabled when the transport and kernel both support it; falls back to KernelFq otherwise. Used for ST 2110-21-grade pacing.
