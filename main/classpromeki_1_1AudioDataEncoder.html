Detailed Description

Sample-domain binary data encoder for PCM audio payloads.

Audio analogue of ImageDataEncoder. Stamps 64-bit opaque payloads into a chosen channel of a PcmAudioPayload as a Manchester-encoded 76-bit codeword (4 sync bits, 64 payload bits, 8 CRC-8/AUTOSAR bits), where every bit occupies a fixed integer number of audio samples. The wire format is sample-accurate when the audio path does not resample, and survives ordinary sample-rate conversion thanks to Manchester's mid-bit transition and the integrate-and-compare demodulator on the AudioDataDecoder side.

Wire format
Each Item produces an N -sample run laid out as follows, where S = samplesPerBit, H = S/2, and the trailing pad fills any additional samples in the Item up to its sampleCount: 
*   |  4 sync bits  |       64 payload bits        |  8 CRC bits  | pad |
*   |  1010         | MSB ........................ | MSB ... LSB  |     |
*   each bit = H samples of (+A or -A) followed by H samples of
*   the opposite sign.  bit '1' = +A, then -A; bit '0' = -A, then +A.
*
  • Sync nibble: 0b1010 (transmitted MSB-first).
  • Payload: 64-bit value, transmitted MSB-first (bit 63 first).
  • CRC: CRC-8/AUTOSAR over the 8 payload bytes interpreted big-endian — byte 0 is bits 56..63 of the payload, byte 7 is bits 0..7. Transmitted MSB-first.
  • Trailing pad: samples in the Item beyond the 76*S codeword are written as silence (0.0f converted to the format's neutral value — e.g. 128 for unsigned 8-bit).

Bits cells are constant amplitude in each half-bit (the encoder does not band-limit the transition); this keeps DC zero, gives Manchester's strong mid-bit timing edge, and lets the encoder use a single memcpy per half-bit on planar formats. The Manchester fundamental frequency is sampleRate/(2H) — at the default samplesPerBit=8 that is 6 kHz at 48 kHz, comfortably below the passband edge of any reasonable sample-rate converter.

Channel layout
Items target one channel each. The encoder handles both interleaved and planar PCM transparently using AudioDesc::channelBufferOffset and AudioDesc::bytesPerSampleStride. The same channel index means "channel @c N of the multi-channel stream" regardless of planar / interleaved layout.
Per-format value mapping
The encoder builds two single-channel "primer" buffers at construction (+A and -A in the target sample format, H samples each) plus a one-sample silence primer for the pad. Floating-point formats use the amplitude directly; integer formats use AudioFormat::floatToSamples to perform the round to the format's quantisation grid. After construction the hot path is a fixed sequence of memcpy / strided byte copy calls — 152 half-bits per Item per channel, plus the pad fill.
Lifetime and reuse
Construct one encoder per (AudioDesc, samplesPerBit, amplitude) triple and reuse it across many payloads. The encoder owns its primer buffers internally; encode is reentrant on a single instance only with respect to itself (concurrent encode calls on the same instance must be externally synchronized). Use isValid to detect a construction failure (descriptor invalid, compressed format, samplesPerBit out of range, etc.).
Example
AudioDataEncoder enc(payload->desc()); // defaults: 8 SPB, 0.1 amp
if(!enc.isValid()) return Error::Invalid;
AudioDataEncoder::Item items[] = {
{ 0, payload->sampleCount(), 0, frameId }, // ch 0: rolling frame ID
{ 0, payload->sampleCount(), 1, channelId }, // ch 1: channel marker
};
Error err = enc.encode(*payload, items);

Stamps every Item's payload into the supplied payload.

The payload's audio descriptor must compare equal to the one supplied at construction. Each Item must satisfy:

  • channel < desc().channels()
  • sampleCount ≥ packetSamples()
  • firstSample + sampleCount ≤ inout.sampleCount()

Items that fail any of these are rejected with Error::OutOfRange (geometry) or Error::InvalidArgument (channel out of range).

Parameters
inoutPCM payload to write into. Must be allocated; the data is modified in place.
itemsList of Items to stamp.
Returns
Error::Ok on success.

◆ DefaultSamplesPerBit

constexpr uint32_t promeki::AudioDataEncoder::DefaultSamplesPerBit = 8
staticconstexpr

Default samples-per-bit (8).

76 × 8 = 608 samples per codeword. At 48 kHz the Manchester fundamental sits at 6 kHz, well inside the flat passband of any reasonable SRC. Fits in one 60p audio frame at 48 kHz (800 samples) or 44.1 kHz (735 samples) with room to spare.

◆ MaxSamplesPerBit

Maximum samples-per-bit (64).

Larger values are physically valid but produce codewords longer than typical per-frame audio windows. Cap exists to keep the per-Item upper bound predictable.

◆ MinSamplesPerBit

constexpr uint32_t promeki::AudioDataEncoder::MinSamplesPerBit = 4
staticconstexpr

Minimum samples-per-bit (4).

Below 4 the Manchester fundamental moves above the Nyquist of common destination rates (e.g. 24 kHz fundamental at samplesPerBit=2 doesn't survive a 48 k→44.1 k SRC). Must be even.

Sync nibble bit pattern 1010 (MSB-first).

Manchester encodes this as half-bit run-length (H, 2H, 2H, 2H, H) so the decoder gets three independent same-bit-width measurements bounded on both sides by encoder transitions.
