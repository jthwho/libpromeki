333 // -- Full-range uncompressed YCbCr --
334 //
335 // The library-wide YCbCr naming convention is that
336 // the unsuffixed form (e.g. YUV8_422_Rec709) is
337 // limited-range (16..235 Y, 16..240 Cb/Cr) to match
338 // broadcast / SDI / ST 2110 defaults. The explicit
339 // "_Full" suffix opts in to full-range 0..255 Y and
340 // 0..255 Cb/Cr, which is what JPEG / JFIF uses and
341 // what most consumer video decoders (ffplay,
342 // browsers, libjpeg-turbo) expect. These
343 // PixelDescs exist both as general-purpose
344 // full-range YCbCr storage and as encode-source
345 // intermediates for the full-range JPEG variants
346 // below.

352 // -- Full complement of JPEG YCbCr variants (matrix × range) --
353 //
354 // Following the library-wide YCbCr convention: the
355 // unsuffixed JPEG names (existing
356 // JPEG_YUV8_422_Rec709 / JPEG_YUV8_420_Rec709) are
357 // limited-range to stay consistent with the
358 // uncompressed YCbCr defaults. Full-range variants
359 // take the explicit "_Full" suffix. All eight
360 // combinations (matrix × range × subsampling) are
361 // provided; pick the one that matches what the
362 // downstream decoder expects to interpret:
363 //
364 // - Rec.601 + full range (strict JFIF) → ffplay,
365 // browsers, libjpeg-turbo, most consumer apps
366 // - Rec.709 + full range → modern cameras with
367 // an ICC profile; good for high-quality still
368 // storage that a colour-managed viewer honours
369 // - Rec.709 + limited range (existing default) →
370 // broadcast / SDI JPEG pipelines (ST 2110 JPEG XS
371 // style)
372 // - Rec.601 + limited range → legacy broadcast
373 JPEG_YUV8_422_Rec601 = 147,
374 JPEG_YUV8_420_Rec601 = 148,
