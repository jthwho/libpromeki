Detailed Description

template<typename A, typename B>
class Pair< A, B >

Typed pair container wrapping std::pair.

Provides a Qt-inspired API over std::pair with consistent naming conventions matching the rest of libpromeki. Simple value type — no PROMEKI_SHARED_FINAL.

Supports structured bindings via std::tuple_size/std::tuple_element specializations.

Constructs a pair from two values, perfect-forwarded.

Accepts any combination of lvalue and rvalue arguments and forwards them into the underlying std::pair, so move-only element types like std::unique_ptr work seamlessly. SFINAE-guarded to only participate in overload resolution when both A and B are actually constructible from the supplied argument types — this keeps unrelated single-arg constructors like the std::pair conversion overload from being shadowed by accidental two-arg matches.
