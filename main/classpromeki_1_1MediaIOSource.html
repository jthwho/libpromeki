Detailed Description

Read-side port of a MediaIO.

A MediaIOSource produces frames out of a MediaIO instance. It carries the per-port read API — readFrame, the per-port read queue, format negotiation (proposeOutput), and the per-port frameReady signal. Step / current-frame / seek state lives on the source's MediaIOPortGroup, so multi-source groups stay in lockstep by definition; the methods on this class are convenience forwarders to the group.

Sources are created by the backend during open and parented to their MediaIOPortGroup, which is in turn parented to the owning MediaIO.

Default name
If the name argument is empty the constructor synthesizes "src{index}" (e.g. "src0", "src1") so every source has a stable, human-readable identifier even when the backend has nothing specific to call it.
Port groups
All sources in a group share the group's clock, step, current-frame, and seek state. The multi-source "atomic distribute one frame per source per @ref MediaIOCommandRead" path is reserved for a future phase — today MediaIOCommandRead carries a single Frame::Ptr targeted at one source within the group, and sources advance their queues one read at a time. Sources in single-port groups behave independently.

◆ readFrame()

MediaIORequest promeki::MediaIOSource::readFrame ( )

Reads the next frame from the source.

Vends a request bound to the next prefetched read from the source's MediaIOReadCache. Always returns immediately with a valid request — the underlying cmd may already have completed (cache hit) or may still be in flight (cache miss). Callers retrieve the typed payload via req.commandAs<MediaIOCommandRead>() once the request resolves:

MediaIORequest req = source->readFrame();
Error err = req.wait();
if (err.isOk()) {
Frame::Ptr frame = req.commandAs<MediaIOCommandRead>()->frame;
// ... use frame ...
}

EOF semantics
Once a read returns Error::EndOfFile, the group latches end-of-stream and stops issuing further prefetch reads. Subsequent calls keep returning Error::EndOfFile (without going down to the backend) until the user calls MediaIOPortGroup::seekToFrame or MediaIO::close to reset the state.
Pre-open / closed
Returns an immediately-resolved Error::NotOpen request when the owning MediaIO is not open. Returns Error::Invalid when the source is detached from its MediaIO or group.
Cancellation
Already-vended requests resolve with Error::Cancelled when cancelPending or MediaIORequest::cancel is invoked before the underlying cmd dispatches.

◆ frameReadySignal

Signal< > promeki::MediaIOSource::frameReadySignal

Emitted on the transition "next read is ready.".

Signal:

The contract is "if you call @ref readFrame right now, the returned request is already resolved." Concretely the signal fires on every transition from "head of the cache not ready" to "head of the cache ready" — this includes the first read that finishes after a fresh prefetch as well as a second read landing while the first is still unconsumed. After the consumer pops via readFrame the cache re-evaluates the new head and re-fires the signal if that one is already complete too.

EOF and error completions count: a synthetic EOS pushed during close, or a real backend error, surfaces through the same edge as a real frame delivery so the consumer can observe the outcome by simply calling readFrame and waiting on the request.
