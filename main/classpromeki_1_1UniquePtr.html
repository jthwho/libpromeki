Detailed Description

template<typename T>
class promeki::UniquePtr< T >

A move-only smart pointer with exclusive ownership semantics.

This class provides exclusive-ownership lifetime management for a heap-allocated object. Unlike SharedPtr, there is no reference counting, no copy-on-write, and no proxy wrapper for non-native types. Ownership can only be transferred via move operations.

Typical use case:

// Create a new object in-place
auto ptr = UniquePtr<MyClass>::create(arg1, arg2);
// Or take ownership of an existing raw pointer
auto ptr2 = UniquePtr<MyClass>::takeOwnership(existingRawPtr);
// Transfer ownership with std::move
UniquePtr<MyClass> other = std::move(ptr);
// ptr is now null
// Access the owned object
other->doSomething();

Thread Safety:

  • A UniquePtr is not thread-safe. It is assumed that exclusive ownership is maintained by a single thread at a time. Transferring ownership between threads requires external synchronization by the caller.

Limitations:

  • No custom deleters; the owned object is always destroyed with delete.
  • When upcasting to a base class (either directly or via move), the base class must have a virtual destructor for correct destruction to occur, just as with std::unique_ptr.
