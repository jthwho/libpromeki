Detailed Description

Value-typed envelope for everything MediaPipeline can announce to a subscriber.

A PipelineEvent unifies the existing per-signal stage / pipeline notifications and the new stats tick into one stream that callers (a demo UI, a REST relay, an in-process listener) can subscribe to via MediaPipeline::subscribe. Each event carries:

  • a Kind that describes what happened,
  • an optional stageName (empty for pipeline-level events),
  • either a Variant payload (primitive payloads such as a state name or log message) or a JsonObject jsonPayload (richer payloads such as a MediaPipelineStats or MediaPipelineConfig snapshot — these types do not round-trip through Variant directly so they are stored as already- serialized JSON),
  • free-form metadata,
  • a timestamp seeded by MediaPipeline::publish when the event is dispatched.

The class is a plain Shareable value (not an ObjectBase): it follows the project's data-object policy with PROMEKI_SHARED_FINAL so callers may share via Ptr when convenient and copy by value otherwise.

Per-kind payload contract
  • Kind::StateChanged — payload holds a String naming the new MediaPipeline::State (e.g. "Running").
  • Kind::StageState — payload holds a String naming the stage transition ("Opened", "Started", "Stopped", "Closed"); stageName identifies the stage.
  • Kind::StageError — payload holds a String message; metadata carries the error name / code via "code".
  • Kind::StatsUpdated — jsonPayload holds the result of MediaPipelineStats::toJson; payload is invalid.
  • Kind::PlanResolved — jsonPayload holds the result of MediaPipelineConfig::toJson; payload is invalid.
  • Kind::Log — payload holds the log message String; metadata carries "level", "source", "line", "threadName".
Example
PipelineEvent ev;
ev.setKind(PipelineEvent::Kind::StateChanged);
ev.setPayload(Variant(String("Running")));
JsonObject json = ev.toJson();
Error err;
PipelineEvent round = PipelineEvent::fromJson(json, &err);
assert(err.isOk());
assert(round.kind() == PipelineEvent::Kind::StateChanged);

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized. PipelineEvent::Ptr uses an atomic refcount and is safe to share across threads — events are typically published by MediaPipeline to subscribers on different threads via shared pointer.

◆ toJson()

JsonObject promeki::PipelineEvent::toJson ( ) const

Serializes the event to a stable JSON envelope.

Shape:

{
"kind": "StatsUpdated",
"stage": "tpg1", // omitted when empty
"ts": 12345.678, // monotonic seconds
"metadata": { ... }, // omitted when empty
"payload": ... // primitive: scalar; complex: object
}

Primitive payloads are emitted as JSON scalars. Complex payloads (StatsUpdated / PlanResolved) are emitted as the nested JSON object held by jsonPayload.
