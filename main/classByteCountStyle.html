Detailed Description

Well-known Enum type for human-readable byte-count formatting.

Selects the unit family used by String::fromByteCount when formatting an allocation size or similar byte-valued quantity:

  • Metric — powers of 1000 with SI suffixes: B, KB, MB, GB, TB, PB, EB.
  • Binary — powers of 1024 with IEC suffixes: B, KiB, MiB, GiB, TiB, PiB, EiB.

Inherits from TypedEnum so function signatures can take const ByteCountStyle & and get compile-time type checking — other enum kinds (e.g. VideoPattern) will fail to compile when passed in the same slot. Runtime compatibility with Variant and any API that takes a plain Enum is preserved via implicit derived-to-base slicing.

Example
String s = String::fromByteCount(1048576, 3, ByteCountStyle::Binary);
// → "1 MiB"
// Round-trip through Variant still works:
Variant v = ByteCountStyle::Binary; // Enum slice
Enum e = v.get<Enum>(); // "ByteCountStyle::Binary"
ByteCountStyle back{e.value()}; // back to typed
