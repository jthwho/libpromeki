◆ BridgeFunc

using MediaIO::FormatDesc::BridgeFunc = std::function< bool(const MediaDesc &from, const MediaDesc &to, Config *outConfig, int *outCost)>

Optional bridge declaration for transform backends.

When set, this backend declares that an instance of itself can convert from one MediaDesc shape to another. The pipeline planner uses this to automatically insert bridging stages (CSC, decoder, frame-rate sync, etc.) when a route's source and sink are not directly format-compatible.

The callback returns true if the bridge is applicable, in which case it must:

  • Populate outConfig with the MediaConfig the planner should use when instantiating the bridge stage (e.g. set OutputPixelFormat on a CSC bridge).
  • Set outCost to the unitless quality cost of the conversion (lower = higher quality). The planner uses costs to pick among multiple valid bridges.

Cost bands (unitless):

  • 0 identity / no-op
  • 1–10 metadata-only transform
  • 10–100 lossless precision-preserving
  • 100–1000 bounded-error lossy
  • 1000–10000 heavily lossy (re-encode)
  • > 10000 last resort

Backends that are not transforms (sources, sinks, pure passthroughs) leave this null.

using MediaIO::FormatDesc::CanHandlePathFunc = std::function<bool(const String &path)>

Optional path-based probe.

Returns true if this backend can handle the given filesystem path (e.g. a V4L2 backend recognises "/dev/video0"). Checked by createForFileRead after URL scheme dispatch but before extension or content probes, so device nodes that are not regular files are routed to the right backend.

◆ ConfigSpecFunc

using MediaIO::FormatDesc::ConfigSpecFunc = std::function<Config::SpecMap ()>

Returns the config specs for this backend.

The returned map lists every MediaConfig::ID the backend understands, each paired with a VariantSpec that carries the backend-specific default value, accepted types, ranges, and description. Backends can narrow or widen the global spec for each ID.

MediaIO::defaultConfig builds a Config from the specs' default values. MediaIO::configSpecs returns the raw map for introspection.

◆ DefaultMetadataFunc

using MediaIO::FormatDesc::DefaultMetadataFunc = std::function<Metadata ()>

Returns the metadata schema this backend honors.

The returned Metadata lists every Metadata::ID the backend understands, pre-populated with empty / default values. It is the metadata counterpart to ConfigSpecFunc — callers that want to know which metadata keys are honored (for --help dumps, GUI editors, etc.) can walk the returned container without having to hard-code a per-backend list. Backends that do not consume any container-level metadata may leave this callable null — MediaIO::defaultMetadata then returns an empty Metadata.

◆ EnumerateFunc

using MediaIO::FormatDesc::EnumerateFunc = std::function<StringList()>

Optional callback that lists available device instances.

Returns the locator strings (suitable for use as MediaConfig::Filename) for each available instance of this backend. File-based backends generally leave this null; device backends provide an implementation that scans the system.

◆ UrlToConfigFunc

using MediaIO::FormatDesc::UrlToConfigFunc = std::function< Error(const Url &url, Config *outConfig)>

Optional URL → Config translator.

Called when createFromUrl (or the transparent URL path in createForFileRead / createForFileWrite) dispatches to this backend via its registered schemes. The backend should populate outConfig from url's authority and path components only — query parameters are applied automatically by the factory via applyQueryToConfig against this backend's configSpecs, so the callback body stays short. MediaConfig::Type is set by the factory before the callback runs, so the backend does not need to populate it.

Case sensitivity
Query keys are matched case-sensitively against configSpecs (see applyQueryToConfig). Scheme and host follow RFC 3986 — scheme is normalized to lowercase by Url::fromString, host is preserved verbatim.

A non-OK return causes the factory to fail the open so the caller learns about malformed URLs via the normal error path rather than a silently-misconfigured MediaIO.
