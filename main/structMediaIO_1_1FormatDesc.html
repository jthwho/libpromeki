◆ DefaultMetadataFunc

using MediaIO::FormatDesc::DefaultMetadataFunc = std::function<Metadata ()>

Returns the metadata schema this backend honors.

The returned Metadata lists every Metadata::ID the backend understands, pre-populated with empty / default values. It is the metadata counterpart to DefaultConfigFunc — callers that want to know which metadata keys are honored (for --help dumps, GUI editors, etc.) can walk the returned container without having to hard-code a per-backend list. Backends that do not consume any container-level metadata may leave this callable null — MediaIO::defaultMetadata then returns an empty Metadata.

◆ EnumerateFunc

using MediaIO::FormatDesc::EnumerateFunc = std::function<StringList()>

Optional callback that lists available device instances.

Returns the locator strings (suitable for use as MediaConfig::Filename) for each available instance of this backend. File-based backends generally leave this null; device backends provide an implementation that scans the system.
