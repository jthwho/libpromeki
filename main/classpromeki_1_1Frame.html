Detailed Description

A media frame containing payloads and metadata.

Aggregates one or more MediaPayload entries — uncompressed or compressed video (VideoPayload) and audio (AudioPayload) — together with a Metadata container, into a single unit that represents one frame of media content.

All essence lives in the payloadList. Use videoPayloads / audioPayloads to filter by kind. Compressed bitstream access units travel as CompressedVideoPayload or CompressedAudioPayload entries in the same list.

Thread Safety
Distinct instances may be used concurrently — Frame is a value-with-COW container (PROMEKI_SHARED_FINAL) and copies share underlying payloads via atomic refcount. A single instance is conditionally thread-safe: const accessors are safe; mutators (addPayload, setMetadata, ...) require external synchronization.

◆ configUpdate()

const MediaConfig & promeki::Frame::configUpdate ( ) const
inline

Returns a const reference to the config update delta.

When non-empty, the config update is applied by the receiving MediaIO before the frame is processed. This gives frame-level synchronisation for dynamic parameter changes (bitrate, quality, etc.) — the update travels with the write command through the strand queue, so a flush can never separate a config change from its frame.

◆ dump()

StringList promeki::Frame::dump ( const String &  indent = String()) const

Returns a human-readable multi-line dump of this frame's full contents.

Emits the scalar-key block registered with VariantLookup<Frame> (VideoCount, AudioCount, PayloadCount, VideoFormat), the frame's metadata via Metadata::dump, the configUpdate delta when non-empty, and then a subdump of each payload in payloadList indented by two spaces. Each payload section is headed with the kind name and per-kind index ("Video[0]:", "Audio[0]:", "Subtitle[0]:", …) and its body is the concrete leaf's full VariantLookup dump.

No leading "Frame:" header is emitted — callers that need one (e.g. pmdf-inspect printing a frame index) prepend their own title and an additional indent so the output is uniform regardless of whether one frame or many are being printed.

Parameters
indentLeading whitespace to prefix every line.
Returns
A StringList (one entry per line).

◆ isSafeCutPoint()

bool promeki::Frame::isSafeCutPoint ( CutPointScope  scope = CutPointAudioVideo) const

Returns true when stopping the stream before this frame leaves the chosen essence streams intact.

Defers to MediaPayload::isSafeCutPoint for every video payload in payloadList and every audio payload. A scope that excludes a side of the essence skips that side entirely — useful for sinks that only accept one kind of essence and should not be held hostage to a mid-GOP frame that will never be written to them.

An empty essence list for the selected scope is treated as trivially safe (nothing to truncate). Null entries are skipped rather than rejected — the pipeline builds lists that always carry valid pointers.

Parameters
scopeWhich essence streams to consider.
Returns
true when the cut is safe for every payload in scope.

◆ mediaDesc()

MediaDesc promeki::Frame::mediaDesc ( ) const

Assembles a MediaDesc describing this Frame.

Returns a MediaDesc populated from the frame's own state: the frame rate is taken from Metadata::FrameRate (if set), the video descriptors are built from each VideoPayload in payloadList, the audio descriptors from each AudioPayload, and the frame's metadata is copied across.

The returned MediaDesc is only valid if both a frame rate is present in metadata and the frame carries at least one video or audio payload.

◆ videoPayloads()

VideoPayload::PtrList promeki::Frame::videoPayloads ( ) const

Returns the Video-kind entries from payloadList as typed pointers.

Walks payloadList once and collects every MediaPayload whose kind is MediaPayloadKind::Video, returning them as a fresh VideoPayload::PtrList with shared ownership (no clone). Null payload entries are skipped.

The convenience consumers want during migration: read once, get only the video side, still see UncompressedVideoPayload and CompressedVideoPayload polymorphically via MediaPayload::as.

Returns
A fresh VideoPayload::PtrList of all video-kind payloads in payload-list order.
