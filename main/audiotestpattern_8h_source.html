425 // AvSync tone burst cache: built lazily per requested

433 // Cached noise buffers. One-dimensional arrays of
434 // samples that the per-channel noise dispatch indexes
435 // with a per-channel offset so different WhiteNoise /
436 // PinkNoise channels stay decorrelated while sharing
437 // the same underlying samples. Generated at most once
438 // per configure() call; empty until a noise channel
439 // requests them.
440 List<float> _whiteNoiseBuffer;

442
443 // Continuous-sweep cursor/phase for the Chirp
444 // generator. The cursor is the sample position inside
445 // the current sweep period (wraps at
446 // _chirpDurationSec * sampleRate). The phase is the
447 // running sin-argument; we accumulate it sample-by-
448 // sample instead of recomputing it from the closed-
449 // form integral so the waveform stays sample-exact
450 // continuous across both chunk boundaries and period
451 // wraps (the closed form resets to 0 at each period
452 // wrap, which causes an audible click).
453 mutable double _chirpSampleCursor = 0.0;
454 mutable double _chirpPhase = 0.0;
455
456 // Per-tone phase accumulators for the DualTone
457 // generator. Tracking these across create() calls is
458 // the only way to avoid a once-per-chunk phase-reset
459 // click; a 30 fps pipeline restarts the pattern 30
460 // times a second otherwise, which is plainly audible.
461 mutable double _dualTonePhase1 = 0.0;
462 mutable double _dualTonePhase2 = 0.0;
463
464 // Cumulative sample cursor for the cached noise
465 // buffers. Advanced once per create() call so
466 // successive frames read from successive regions of
467 // the buffer instead of re-reading the same slice —
468 // otherwise the noise would modulate at the frame
469 // rate and sound like a tone, not noise. All noise
470 // channels share this cursor, with a per-channel
471 // offset layered on top so multiple noise channels
472 // on the same stream stay decorrelated.
473 mutable size_t _noiseSampleCursor = 0;
