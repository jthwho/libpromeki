481 // Continuous-sweep cursor/phase for the Chirp
482 // generator. The cursor is the sample position inside
483 // the current sweep period (wraps at
484 // _chirpDurationSec * sampleRate). The phase is the
485 // running sin-argument; we accumulate it sample-by-
486 // sample instead of recomputing it from the closed-
487 // form integral so the waveform stays sample-exact
488 // continuous across both chunk boundaries and period
489 // wraps (the closed form resets to 0 at each period
490 // wrap, which causes an audible click).
491 mutable double _chirpSampleCursor = 0.0;
492 mutable double _chirpPhase = 0.0;
493
494 // Per-tone phase accumulators for the DualTone
495 // generator. Tracking these across create() calls is
496 // the only way to avoid a once-per-chunk phase-reset
497 // click; a 30 fps pipeline restarts the pattern 30
498 // times a second otherwise, which is plainly audible.
499 mutable double _dualTonePhase1 = 0.0;
500 mutable double _dualTonePhase2 = 0.0;
501
502 // Cumulative sample cursor for the cached noise
503 // buffers. Advanced once per create() call so
504 // successive frames read from successive regions of
505 // the buffer instead of re-reading the same slice —
506 // otherwise the noise would modulate at the frame
507 // rate and sound like a tone, not noise. All noise
508 // channels share this cursor, with a per-channel
509 // offset layered on top so multiple noise channels
510 // on the same stream stay decorrelated.
