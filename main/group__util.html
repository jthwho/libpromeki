Detailed Description

General-purpose utilities, error handling, and diagnostics.

Variant type, Error reporting, UUID generation, FourCC codes, Metadata key-value store, environment access, command-line parsing, logging, memory pools, platform detection, terminal I/O, and more.

See Debugging and Diagnostics for build types, per-module debug logging, and crash handling.

static constexpr ID Name = ID::literal(#Name); \
[[maybe_unused]] static inline const ID PROMEKI_CONCAT(_promeki_spec_reg_, Name) = declareID(#Name, __VA_ARGS__)

Declares a well-known VariantDatabase key as a compile-time constant.

Expands, inside a class derived from VariantDatabase, to two declarations that together give the key both a constexpr ID and a registered VariantSpec. Specifically:

  1. static constexpr ID <Name> initialized from the pure FNV-1a hash of "<Name>" via StringRegistry::Item::literal — usable in switch labels, static_assert, template parameters, and other constant expression contexts.
  2. A sibling static inline const ID whose initializer calls VariantDatabase::declareID with the same name to register the spec (and the name, for reverse lookup) at static-init time. The strict-hash path guarantees the two IDs agree; a hash collision with another well-known key will abort the process before main() runs.

The name appears once in the macro call, so there is no risk of the literal hash and the registered name drifting apart.

Example
class Metadata : public VariantDatabase<"Metadata"> {
public:
PROMEKI_DECLARE_ID(Title,
VariantSpec().setType(Variant::TypeString)
.setDefault(String())
.setDescription("Title of the media."));
// Title.id() is now a constant expression:
static_assert(Title.id() == ID::literal("Title").id());
};

Declares an promeki::Enum type with a table placed in rodata.

Expands inside a class body (typically one deriving from promeki::TypedEnum) to three declarations that together give the type a zero-heap, compile-time-validated registration:

  1. static constexpr Enum::Entry _promeki_enum_entries_[] — the (name, value) table, placed in rodata.
  2. static inline Enum::Definition _promeki_enum_def_ — a single Definition struct pointing at the table. The registry stamps its typeIndex at static-init time; nothing else is ever written.
  3. static inline const Enum::Type Type — the handle consumed by promeki::TypedEnum's constructors and by user code.

Three static_asserts verify at compile time that:

  • the entry table is non-empty,
  • no two entries share a name or an integer value,
  • the default value is present in the table.
Parameters
TypeNameString literal — the human-readable type name.
DefaultInteger default value, which must appear in the table.
...Brace-initialized Enum::Entry rows — { "Name", integer } comma-separated.
Example
class VideoPattern : public TypedEnum<VideoPattern> {
public:
PROMEKI_REGISTER_ENUM_TYPE("VideoPattern", 0,
{ "ColorBars", 0 },
{ "ColorBars75", 1 },
{ "Ramp", 2 });
using TypedEnum<VideoPattern>::TypedEnum;
static const VideoPattern ColorBars;
};

Pre-release stage of a libpromeki build.

Drives the optional pre-release suffix in the VERSION file (MAJOR.MINOR.PATCH-alpha1, -beta3, -rc2 …). ◆ formatBuildInfo()

String promeki::formatBuildInfo ( const String &  fmt,
const BuildInfo *  bi = nullptr 
)

Expand {token}-style placeholders against a BuildInfo.

Lightweight named-token formatter for composing build-info strings without forcing every call site to handcraft printf format strings. Use this when the shape of the output depends on context (e.g. a banner versus a one-line status line); for the standard banners, call buildInfoString() / buildPlatformString() etc., which are themselves implemented as preset templates over this function.

Recognized tokens
  • {name} — project name (e.g. "libpromeki").
  • {version} — bare MAJOR.MINOR.PATCH (no stage, no build).
  • {major}, {minor}, {patch} — integer version components.
  • {build} — CI build number, or 0 for local builds.
  • {stage} — "alpha", "beta", "rc", or "release".
  • {stageNum} — stage number (>=1 for alpha/beta/rc; 0 for release).
  • {extra} — auto-composed pre-release suffix: "-alpha1", "-beta3", "-rc2", or empty for release. Does not include the CI build number.
  • {date}, {time}, {type}, {hostname} — build environment.
  • {repoident} — full git commit hash.
  • {ref} — git ref label (branch / tag / describe / short hash); see BuildInfo::ref.
  • {ident} — opaque per-build identity (see verifyBuildIdent()).

Unknown tokens are emitted verbatim (e.g. "{foo}") so typos are visible in the output rather than silently dropped. Use "{{" and "}}" to emit literal braces.

Parameters
fmtFormat string with {token} placeholders.
biBuildInfo to format against. If nullptr, the library's compiled-in BuildInfo (getBuildInfo()) is used.
Returns
The expanded String.

◆ promekiResolveVariantPath()

Variant promeki::promekiResolveVariantPath ( const Variant &  root,
const String &  path,
Error *  err = nullptr 
)

Resolves a dotted/indexed path against a Variant tree.

Walks root following the same key grammar used by VariantLookup (‘segment ( ’.' segment )*, where segment := name ( '[' index ']' )?`), descending through nested VariantMap and VariantList alternatives as needed.

Examples
  • "foo" against a VariantMap → returns the value for key "foo".
  • "foo.bar" against a VariantMap → looks up foo, descends if it holds a VariantMap, then looks up bar.
  • "foo[0]" → looks up foo, descends if it holds a VariantList, returns element 0.
  • "[0]" against a VariantList — the leading segment is empty so the walker treats the whole root as the list and returns element 0.
Errors
  • Error::IdNotFound — a key was missing from a VariantMap.
  • Error::OutOfRange — a list index was beyond the VariantList size.
  • Error::ParseFailed — the path itself was malformed.
  • Error::Invalid — descended into a Variant that wasn't VariantMap / VariantList when the path still had segments to walk.
Parameters
rootThe Variant to walk. Typically holds a VariantMap or VariantList.
pathThe dotted/indexed path to follow.
errOptional error output.
Returns
The resolved Variant, or an invalid Variant on failure.

◆ uniquePointerCast()

template<typename Derived , typename Base >
UniquePtr< Derived > promeki::uniquePointerCast ( UniquePtr< Base > &&  up)

Dynamic downcast across a polymorphic UniquePtr hierarchy.

Attempts a dynamic_cast from Base to Derived. On success, ownership is transferred from up into the returned UniquePtr and up is left null. On failure (including when up is null), an empty UniquePtr is returned and up is left unchanged — so the caller retains ownership of the original base object.

Verify that the library's runtime build identity matches the caller's compile-time identity (staleness check).

What this is for
Detects that the calling binary was compiled against a different libpromeki build than the one currently loaded at runtime. An application captures PROMEKI_BUILD_IDENT (a #define from <promeki/buildident.h> regenerated on every library build) into one of its translation units at its compile time, then passes that captured value here. The function compares it byte-for-byte against getBuildInfo()->ident, which the library compiled into its own binary. Any divergence means the .so was rebuilt without rebuilding the caller — i.e., the caller is stale.
What this is not for
This is only a stale-binary detector. Do not use it for version compatibility, feature gating, or any human-visible reporting. For normal version / build queries, use getBuildInfo() and the buildInfoString() / buildPlatformString() / buildFeatureString() helpers — those are the supported path for inspecting library build details. The ident format is opaque and may change at any time.
Parameters
expectedThe PROMEKI_BUILD_IDENT value captured at the caller's compile time (typically supplied via the PROMEKI_VERIFY_BUILD_IDENT() macro).
errOptional Error to populate on mismatch (Error::BuildIdentMismatch).
Returns
true if the idents match, false otherwise. Mismatch is also logged via promekiErr() with both strings so the failure is diagnosable from logs alone.

Fatal variant of verifyBuildIdent().

Calls std::abort() (after logging both ident strings via promekiErr()) if the library's runtime identity does not match the caller's compile-time identity. Drop one call into your application's main() — typically via the PROMEKI_VERIFY_BUILD_IDENT_OR_ABORT() macro — to make a stale binary fail loudly at startup rather than silently executing against changed library code.

Same scope rules as verifyBuildIdent(): use only for staleness detection. For normal build-info queries, use the BuildInfo framework.

Walks a key path through a VariantSpec tree.

Used by VariantLookup::Registrar::variantTree to surface the declared spec of a sub-path through the variantTree binding's outer spec. Each segment of path advances the cursor:

  • A bracketed [N] step descends into cur->elementSpec().
  • A name step (e.g. foo) descends into cur->valueSpec().
  • Both, in segments shaped name[N], applies the name descent first then the index descent — matches the runtime resolver.

Returns nullptr when the cursor cannot descend (no element/value spec declared at that depth) and sets err to Error::IdNotFound. An empty path returns cur unchanged.

Parameters
curStarting spec; may be nullptr (returns nullptr).
pathDotted/indexed sub-path (no leading dot).
errOptional error output.
Returns
The spec for the leaf, or nullptr on failure.
