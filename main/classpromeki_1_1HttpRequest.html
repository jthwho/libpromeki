Detailed Description

A parsed HTTP request.

HttpRequest is a Shareable data object: a value-with-COW container that holds the method, URL, headers, body, and any route-matched path parameters. It is what HttpServer hands to a registered handler, and what HttpClient sends.

The body is held as a Buffer, with text and JSON convenience accessors. Because the URL is a full Url, all scheme/host/path/query/fragment knowledge is delegated there; path and queryValue are convenience forwards onto the URL the request was constructed with.

Path parameters are populated by the HttpRouter when a pattern containing "{name}" or "{name:*}" placeholders matches the request path. Handlers consume them via pathParam.

Thread Safety
Distinct instances may be used concurrently — copies share the underlying body Buffer via atomic refcount. A single instance is conditionally thread-safe: const accessors are safe, mutators (setUrl, setBody, headers().set, ...) require external synchronization.
Example
void serveItem(const HttpRequest &req, HttpResponse &res) {
String id = req.pathParam("id");
bool wantJson = req.header("Accept").contains("json");
// ...
}

String promeki::HttpRequest::bodyAsString ( ) const

Returns the body interpreted as a UTF-8 String.

No conversion is performed — the bytes are taken at face value. Suitable for JSON, plain text, or any other Latin-1/ASCII-compatible body; binary callers should use body directly.
