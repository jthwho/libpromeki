Detailed Description

Well-known Enum type for the role a single audio channel plays.

Each entry names a position in a multi-channel layout — the "what does channel N carry?" answer that lets routers, downmixers, and meters speak about specific channels independently of their physical index. Roles are deliberately pre-coordinated with the standard SMPTE / Dolby / DTS / WAVEFORMATEXTENSIBLE naming so an AudioChannelMap can interoperate with foreign tooling.

The integer values are the library's own and are not the WAVEFORMATEXTENSIBLE dwChannelMask bit positions; map between them at the wire-format boundary if needed.

  • Unused — reserved / unassigned channel. Used as the default for newly constructed maps so callers must opt in to declaring a role.
  • Mono — single-channel program audio (1.0 layout).
  • FrontLeft / FrontRight — front-pair stereo (2.0 / L/R).
  • FrontCenter — center channel (3.0+ layouts).
  • LFE — low-frequency effects ("subwoofer").
  • BackLeft / BackRight — surround pair behind the listener (5.1 SMPTE order, 7.1 rears).
  • BackCenter — single rear surround (6.1).
  • SideLeft / SideRight — side surround pair (7.1 sides).
  • FrontLeftOfCenter / FrontRightOfCenter — wide front pair used in some film mixes between the L/R and center.
  • TopFrontLeft / TopFrontRight / TopFrontCenter — overhead front (Atmos / Auro-3D heights).
  • TopBackLeft / TopBackRight / TopBackCenter — overhead rear.
  • TopCenter — single top-of-room speaker (less common).
  • AmbisonicW / AmbisonicX / AmbisonicY / AmbisonicZ — first-order ambisonic (FOA) channels in ACN order: W is the omnidirectional component, XYZ are the first-order spherical harmonics.
  • Aux0 … Aux7 — generic auxiliary channels for non-positional or program-specific data (commentary, descriptive audio, embedded SMPTE 337M data carriers, …).

Default value is Unused.
