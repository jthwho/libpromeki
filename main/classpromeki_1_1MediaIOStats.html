Detailed Description

Telemetry container shared by per-command and instance-wide reporting.

Single VariantDatabase type used in two contexts:

  • As MediaIOCommand::stats on every dispatched command, where backends and the framework populate the per-command keys (ExecuteDuration, QueueWaitDuration, BytesProcessed, plus any backend-specific keys the command chose to surface).
  • As the instance-wide aggregate populated when a MediaIOCommandStats command resolves: backends fill in cumulative counters (FramesDropped, BytesPerSecond, etc.) into the same container, so a single accessor (MediaIORequest::stats) covers both cases without forcing callers to think about which kind of report they are reading.

Standard keys are declared as static members; backends are free to add their own backend-specific keys for data not covered by the standard set. Has its own StringRegistry so stats keys do not collide with config or param keys.

Spec-aware VariantDatabase::setFromJson with WindowedStat detection.

Per-command WindowedStat entries (ReadExecuteDuration, WriteBytesProcessed, etc.) are dynamic — there is no spec registered for them, so the inherited VariantDatabase::setFromJson cannot coerce the JSON-encoded string form back to its native Variant type. This shadow override recognises the canonical "cap=N:[...]" WindowedStat form and reconstructs the typed value before storing it. Every other key falls through to the base behaviour, including spec-driven coercion for the standard cumulative-aggregate keys.

◆ toString()

String promeki::MediaIOStats::toString ( ) const

Renders the standard telemetry keys as a compact log line.

Formats whichever standard keys are present into a single space-separated line suitable for dumping to a terminal or logger. Keys are emitted in a fixed order so periodic telemetry output stays scannable. Cheap counters that are still zero (and the empty LastErrorMessage) are elided so the line stays quiet under normal operation.

Returns
A single-line summary, or an empty String if the container holds no standard keys.
