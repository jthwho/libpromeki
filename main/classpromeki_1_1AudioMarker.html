Detailed Description

One sample-region annotation on an audio payload.

AudioMarker locates a region of samples within a carrying PcmAudioPayload and classifies it with an AudioMarkerType. It is intentionally a small plain-value type: no shared pointer variant, no internal allocation, copyable and comparable. Markers normally live inside an AudioMarkerList stamped on a payload's Metadata::AudioMarkers — applications rarely construct one in isolation.

Region semantics

offset is the zero-based sample index where the region starts; length is the number of samples it covers. Both values are in the units of the payload's AudioPayload::sampleCount, i.e. one unit per frame regardless of channel count. A region may cover [offset, offset + length) sample indices; the half-open interval mirrors the rest of the library (e.g. AudioPayload::sampleCount).

A boundary-only marker (e.g. AudioMarkerType::Discontinuity) sets length == 0 — the marker locates a point on the timeline without claiming any samples.

Thread Safety

Plain value type — copy and compare freely between threads. Concurrent mutation of one instance must be externally synchronized.
