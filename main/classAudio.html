Detailed Description

Object to hold some number of audio samples.

This object is meant to hold some number of audio samples described by an AudioDesc. The sample data is stored in a shared Buffer. When shared ownership is needed, use Audio::Ptr.

Compressed audio
An Audio object can also carry a compressed bitstream such as AAC, Opus, or MP3. A compressed Audio uses an AudioDesc whose codecFourCC() is non-zero (and whose dataType() is typically Invalid since the encoded stream has no single bit depth). The raw encoded bytes live in the object's single buffer. Use isCompressed() to test, compressedSize() to get the byte count, and Audio::fromBuffer() to adopt an existing Buffer::Ptr without copying.
Example
AudioDesc desc(48000, 2, AudioDesc::Float32);
Audio audio(desc, 1024); // 1024 samples
// Shared ownership for pipeline passing
Audio::Ptr shared = Audio::Ptr::create(desc, 1024);

Zero-copy factory: adopts buffer as the audio's data.

Used for both compressed bitstreams and PCM where the caller already has the bytes in a Buffer::Ptr (e.g. from File::readBulk). For compressed audio, desc must have a non-zero codecFourCC(); for PCM audio, the buffer size must be a whole multiple of the PCM frame size (channels × bytesPerSample).
