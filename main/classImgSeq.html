Detailed Description

Describes a numbered image sequence on disk.

ImgSeq captures everything an image-sequence reader needs to know about a sequence:

  • The filename pattern (a NumName — e.g. "shot_####.dpx")
  • The head and tail frame numbers (inclusive range)
  • Optional FrameRate, Size2Du32 "videoSize" hint, and PixelDesc for headerless formats
  • Optional Metadata passthrough carried alongside each frame

The class is also the in-memory representation of the .imgseq JSON sidecar format. An .imgseq file is a small JSON document sitting next to the actual image files in the same directory, and is used to describe one specific sequence's intended configuration (including things the image files themselves cannot carry, such as a frame rate). Multiple .imgseq files may sit in the same directory, each describing a different sequence.

JSON format
{
"type": "imgseq",
"name": "shot_####.dpx",
"head": 1,
"tail": 100,
"frameRate": "24/1",
"videoSize": "1920x1080",
"pixelDesc": "YUV_10_422_DPX",
"metadata": { "Title": "Shot 01" }
}

All fields except type are optional: the bare minimum for a valid sidecar is a name field, everything else is filled in from the directory scan or defaulted.

The "type" field must be present and equal to "imgseq" — this tag identifies the file as an ImgSeq sidecar regardless of its extension. A directory may contain multiple .imgseq files for different sequences.

Separation from MediaIOTask_ImageFile

ImgSeq is a standalone data class with no dependency on the MediaIO framework. Code that wants to describe or persist a sequence can use it directly; the image-file MediaIO task uses it as a convenient container when reading .imgseq files and when a user chooses to write one out after producing a sequence.

When loaded from an .imgseq sidecar, this is the value of the "audioFile" JSON field. Relative paths are resolved against the sidecar's (or sequence's) directory; absolute paths are used as-is. Empty means no sidecar audio was specified in the sidecar file.

◆ detectRange()

Error ImgSeq::detectRange ( const FilePath &  dir)

Detects the head and tail from the containing directory.

Scans dir for files matching the current pattern and updates head/tail to the minimum/maximum frame numbers observed. Files whose NumName is not in the same sequence as this pattern (different prefix, suffix, or padding) are ignored. The pattern must already be set — call setName() before calling this.

Parameters
dirThe directory to scan.
Returns
Error::Ok on success (including zero matches), or Error::Invalid if the pattern is invalid.
