Detailed Description

JPEG XS VideoEncoder built on JpegXsImageCodec.

Adapts the existing intra-only JpegXsImageCodec into the stateful VideoEncoder push-frame / pull-packet contract. Holding a JpegXsImageCodec instance across calls is what lets future revisions reuse the SVT-JPEG-XS encoder context across frames without paying the expensive setup cost on every submitFrame.

Registered against VideoCodec::JPEG_XS. Every emitted packet is flagged MediaPacket::Keyframe because every JPEG XS access unit is independently decodable.

◆ configure()

void JpegXsVideoEncoder::configure ( const MediaConfig &  config)
overridevirtual

Applies encoder parameters from a MediaConfig.

Each backend reads the well-known keys it understands (e.g. BitrateKbps, VideoRcMode, GopLength, VideoPreset) and silently ignores the rest, so the same MediaConfig can be reused across pipeline stages without filtering. Calling configure() while an encoder session is active is allowed; the backend applies what it can at runtime and may defer the rest to the next IDR. The default implementation is a no-op.

Signals end-of-stream to the encoder and asks it to emit remaining packets.

After flush(), the caller drains with receivePacket until it sees a packet with the MediaPacket::EndOfStream flag or a null Ptr. After the flush completes, the encoder may be reused for a fresh stream by calling reset followed by configure; or it may be destroyed.

Error JpegXsVideoEncoder::submitFrame ( const Image &  frame,
const MediaTimeStamp &  pts = MediaTimeStamp() 
)
overridevirtual

Submits one uncompressed frame for encoding.

The encoder takes a logical copy of what it needs from frame before returning; the caller may reuse or release the frame afterward. The pts is recorded in the encoder's internal queue so that the matching output packet can carry it back out. Submitting does not guarantee a packet will be available immediately — B-frames and look-ahead both introduce output delay.

Parameters
frameThe source image in one of the supported uncompressed pixel formats.
ptsPresentation timestamp for this frame. May be invalid if the pipeline does not care about presentation ordering.
Returns
Error::Ok on success, or an error code describing why the frame was rejected.
