Detailed Description

JPEG XS VideoEncoder built on SVT-JPEG-XS.

Encodes and decodes JPEG XS (ISO/IEC 21122), a modern low-complexity intra-only codec designed for visually-lossless contribution and transport. JPEG XS targets the broadcast / ST 2110 / RFC 9134 RTP workflow: sub-frame latency, constant bitrate, and native support for 10- and 12-bit YCbCr.

Supported input formats:

  • YUV8_422_Planar_Rec709, YUV10_422_Planar_LE_Rec709, YUV12_422_Planar_LE_Rec709 (Rec.709 limited range)
  • YUV8_420_Planar_Rec709, YUV10_420_Planar_LE_Rec709, YUV12_420_Planar_LE_Rec709 (Rec.709 limited range)
  • RGB8_Planar_sRGB (sRGB full range)

JPEG XS is constant-bitrate rather than quality-targeted: configure the encoder with a bits-per-pixel budget via MediaConfig::JpegXsBpp. Typical visually-lossless broadcast values are 2-6 bpp; the codec default is 3 bpp (≈ 8:1 compression for 8-bit 4:2:2).

Registered against VideoCodec::JPEG_XS. Every emitted packet is flagged MediaPacket::Keyframe because every JPEG XS access unit is independently decodable.

void JpegXsVideoEncoder::configure ( const MediaConfig &  config)
overridevirtual

Applies encoder parameters from a MediaConfig.

Each backend reads the well-known keys it understands (BitrateKbps, VideoRcMode, GopLength, VideoPreset, …) and silently ignores the rest. Calling configure() while a session is active is allowed; the backend applies what it can at runtime and defers the rest to the next IDR. The default implementation is a no-op.
