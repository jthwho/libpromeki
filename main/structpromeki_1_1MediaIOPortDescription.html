Detailed Description

Snapshot of a single MediaIOPort's identity and format landscape.

One per source / sink on a multi-port MediaIO. Carried in MediaIODescription's sources / sinks lists. The per-port preferred / producible / acceptable format fields drive the pipeline planner's bridge-insertion decisions.

◆ acceptableFormats

MediaDesc::List promeki::MediaIOPortDescription::acceptableFormats

Format shapes a sink port can accept directly.

Sink-side counterpart to producibleFormats. Empty for Source ports and for sinks whose backend has nothing to advertise. The pipeline planner uses this together with the upstream producibleFormats to decide whether direct delivery is possible or a bridge stage is required.

Refers into MediaIODescription::portGroups; lets planners identify ports that share timing, frame count, and seek state.

◆ producibleFormats

MediaDesc::List promeki::MediaIOPortDescription::producibleFormats

Format shapes a source port can produce directly.

Source-side counterpart to acceptableFormats. Empty for Sink ports and for sources whose backend has nothing to advertise. The pipeline planner consults this when matching upstream outputs to downstream inputs and deciding where to splice in a CSC / decoder bridge.
