Detailed Description

Resolves a partial MediaPipelineConfig into one whose every route is directly format-compatible by splicing in the bridging stages declared via MediaIOFactory::bridge.

The planner runs offline (no signals, no frame motion). It walks every route, queries the source's produced MediaDesc and the sink's accepted MediaDesc, and — when they differ — chooses the cheapest bridge (or short bridge chain) that closes the gap.

How sources are queried

For every source stage (no upstream in the route graph) the planner needs to know the MediaDesc that stage will produce. It tries the following sources in order:

  1. The cached MediaIO::mediaDesc on a stage that the caller already opened.
  2. The preferredFormat field of MediaIO::describe.
  3. The pre-open hint set via MediaIO::setExpectedDesc.
  4. As a last resort, the planner opens the source briefly in MediaIO::Source mode to read its mediaDesc, then closes it. Backends with side-effecting opens (RTP, V4L2 capture) pay the cost of one open / close cycle during planning. The probe uses only the stage's registered backend and path — it does not forward the caller's expectedDesc / expectedAudioDesc / expectedMetadata pre-open hints on the injected MediaIO, nor any per-stage metadata from the input MediaPipelineConfig. Backends whose opened mediaDesc depends on those hints should implement MediaIO::describe (strategy 2) or set expectedDesc (strategy 3) so the planner never reaches the probe path.
Bridge solving

For each gapped route the planner asks every registered backend with a MediaIOFactory::bridge callback whether it can convert from the source desc to the sink's preferred desc. The cheapest applicable bridge wins. When no single bridge fits, the planner tries the codec-transitive two-hop pattern (VideoDecoder + VideoEncoder) for compressed→compressed transitions. Deeper chains are out of scope for v1; a future revision will replace this with a generic Dijkstra search bounded by Policy::maxBridgeDepth.

Limitations in v1
  • Simultaneous frame-rate and pixel-format gaps in a single hop are not solved. CSC rejects frame-rate mismatches and FrameSync rejects pixel-format mismatches, so a 30 fps RGBA8 source feeding a 24 fps NV12 sink must be resolved by the caller inserting an explicit intermediate stage (typically a CSC ahead of FrameSync or vice-versa). The planner reports Error::NotSupported with a diagnostic when it hits this case.
  • The codec-transitive two-hop path only applies when both ends are compressed. A compressed source feeding an uncompressed sink needs a single VideoDecoder hop, which is handled by the normal single-hop path.
Determinism

Iteration order over the registered bridges is deterministic (matches MediaIOFactory::registeredFactories), so given the same input config and registry state the planner always returns the same resolved config. The resolved config round-trips through JSON and DataStream the same as any hand-authored config.

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized. The planner is typically used briefly at pipeline-build time on the caller's thread. The static MediaIO registry it queries is itself thread-safe.

◆ InjectedStages

using promeki::MediaPipelinePlanner::InjectedStages = ::promeki::Map<String, MediaIO *>

Map of stage name to externally-built MediaIO.

Some backends (the SDL player, a future hardware- device backend) cannot be instantiated through the registry because they need application-supplied resources (window handles, audio device pointers). Callers register them with MediaPipeline::injectStage; the planner needs the same map so it can call MediaIO::describe and MediaIO::proposeInput on the live instance instead of failing to build a stand-in from the registry.

Quality preference for bridge selection.

Modifies the planner's cost function when there are multiple valid bridge chains. All modes still respect Policy::excludedBridges and Policy::maxBridgeDepth.

Enumerator
Highest 

Lowest unmodified cost wins (default).

Balanced 

Slight penalty for high-CPU bridges.

Fastest 

Heavy penalty for high-CPU bridges; prefers cheap CPU paths.

ZeroCopyOnly 

Reject any bridge with cost > 100; fail if no zero-copy chain exists.

For every stage name in the input config, the planner uses the matching MediaIO* from injected if present (querying it for describe / proposeInput just like a registry-built stage). When a name is absent the planner falls back to building from the registry as before. This is how injected SDL / device handles take part in the autoplan pass.

Resolves in into a fully-direct out.

On success out has every route's source MediaDesc directly accepted by the route's sink — the same shape contract that MediaPipeline::build assumes but never enforces. Inserted bridge stages get generated names of the form "<from>__bridge<N>__<to>" so they are stable across re-plans.

On failure out is left empty. When diagnostic is non-null the planner writes a multi-line, human- readable description of the first unsolvable route.

Parameters

outReceives the resolved config.
policyOptional planner policy.

Returns
Error::Ok on success, otherwise the first underlying error (Error::NotSupported for unbridgeable routes, Error::Invalid for a malformed input config).
