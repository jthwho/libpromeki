Detailed Description

First-class audio sample-format descriptor.

Uses the TypeRegistry pattern: a lightweight inline wrapper around an immutable Data record, identified by an integer ID. Well-known formats are provided as ID constants; user-defined formats can be registered at runtime via registerType and registerData — exactly parallel to PixelFormat on the pixel side.

AudioFormat describes the format of an audio stream (sample type, bit depth, endianness, planar vs interleaved, optional compressed codec identity) independent of sample rate and channel count. The tuple format + sampleRate + channels is bundled by AudioDesc.

PCM vs compressed

PCM formats (e.g. PCMI_S16LE, PCMP_Float32LE) populate every field meaningfully and expose sample conversion helpers via samplesToFloat / floatToSamples. Compressed formats (e.g. Opus, AAC) set isCompressed to true and carry an audioCodec identifying the codec family; their sample-level fields are zero / default because a compressed bitstream has no single bit depth or endianness.

Naming convention

C++ identifiers and string names for PCM use a short prefix followed by the sample type and byte order:

  • PCMI_ — interleaved PCM (all channels stored sample-by-sample).
  • PCMP_ — planar PCM (each channel stored in its own plane).

After the prefix, the name spells out the sample type (S8, U8, S16LE, S32BE, Float32LE, …). Compressed formats use the codec's canonical name as the identifier (Opus, AAC, FLAC, MP3, AC3).

Example usage
AudioFormat fmt(AudioFormat::PCMI_S16LE);
assert(fmt.bytesPerSample() == 2);
assert(fmt.isSigned());
assert(!fmt.isCompressed());
AudioFormat opus(AudioFormat::Opus);
assert(opus.isCompressed());
assert(opus.audioCodec().id() == AudioCodec::Opus);
