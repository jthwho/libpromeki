Detailed Description

MediaIO sink backend that decodes and validates a media stream.

The inverse of MediaIOTask_TPG: where the TPG produces test-pattern frames, the inspector consumes frames and runs a configurable set of checks on each one. The set of checks is driven by MediaConfig::InspectorTests — an EnumList of InspectorTest values. The default lists every known test, so a default-configured inspector runs the full suite; override the list to run a subset. Currently supported tests:

  • ImageData — pulls the two 64-bit payloads written by ImageDataEncoder out of the top of every frame, recovering the rolling frame number, stream ID, and BCD timecode.
  • Ltc — runs the selected audio channel through LtcDecoder and reports any timecodes recovered.
  • TcSync — with both decoders running, computes the per-frame offset between the picture timecode and the audio LTC, in audio samples and in fractional frames. This is an instantaneous measurement; a constant offset across frames is healthy (audio and video are locked), while any movement is flagged as a discontinuity.
  • Continuity — tracks frame number, stream ID, picture timecode, and LTC timecode from one frame to the next, and flags any unexpected jump as a InspectorDiscontinuity record on the per-frame event.
  • Timestamp — checks that the per-essence MediaTimeStamp is valid on every frame, records the frame-to-frame delta (min / max / avg) for video and audio, and reports the actual observed FPS.
  • AudioSamples — tracks per-frame audio sample count (min / max / avg) and derives the measured audio sample rate by dividing cumulative samples by the elapsed audio MediaTimeStamp span.

Dependency relationships are auto-resolved at open time: enabling TcSync implicitly turns on ImageData and Ltc; enabling Continuity implicitly turns on ImageData. The user does not have to specify the upstream decoders explicitly.

Per-frame event delivery
The inspector exposes its results in three complementary ways:
  1. Per-frame callback — set via setEventCallback. Invoked once per frame from the worker thread; the callback receives a fully-populated InspectorEvent and must be thread-safe.
  2. Accumulator snapshot — snapshot returns a thread-safe copy of the running totals plus the most recent event. Useful for "polled" consumers and for tests.
  3. Periodic log line — at the cadence configured by MediaConfig::InspectorLogIntervalSec (default 1.0 s of wall time), the inspector emits a one-line summary via promekiInfo. Set the interval to 0 to disable.
Construction pattern
Because setEventCallback must be called before open(), and the standard MediaIO::create() factory hides the underlying task, callers that want a callback should construct the task directly and adopt it via MediaIO::adoptTask:
auto *insp = new MediaIOTask_Inspector();
insp->setEventCallback([](const InspectorEvent &e) {
// ... runs on the worker thread
});
MediaIO *io = new MediaIO();
MediaIO::Config cfg = MediaIO::defaultConfig("Inspector");
// Default is every test; narrow the list to run a subset:
EnumList tests = EnumList::forType<InspectorTest>();
tests.append(InspectorTest::TcSync);
tests.append(InspectorTest::Continuity);
cfg.set(MediaConfig::InspectorTests, tests);
io->setConfig(cfg);
io->adoptTask(insp);
io->open(MediaIO::Sink);
// Pump frames in:
io->writeFrame(frame);

Callers that don't need the callback can use the standard factory path: MediaIO::create("Inspector", ...) and poll snapshot() via the typed task pointer (or just rely on the periodic log).

Frame disposition
The inspector is a sink: by default frames are dropped after the checks run (MediaConfig::InspectorDropFrames defaults to true). This is what makes it usable as the terminal stage of a pipeline. Disabling the drop is currently a no-op since there is no downstream — the option exists for future tee-style wrappers.
