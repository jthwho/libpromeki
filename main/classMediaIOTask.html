Detailed Description

Backend interface for media I/O.

MediaIOTask is the abstract base class for all MediaIO backends (TPG, ImageFile, AudioFile, etc.). It is only driven by MediaIO — backends should never be used directly. All interaction is through commands processed on a worker thread.

Backends override the private executeCmd() virtuals. All inputs and outputs flow through the command struct, so the task does not need to maintain any state visible to MediaIO — only its own internal resources (file handles, generators, etc.).

Friendship

MediaIO is friended so it can call the private virtuals. In C++, derived classes can override private virtual functions even though they cannot call them through the base. This NVI-style pattern enforces the contract that backends are only invoked through MediaIO.

Lifetime

The task instance is owned by MediaIO. MediaIO calls close() before destroying the task. Task destructors should release their own internal resources but should NOT attempt to close() — that is MediaIO's responsibility.

Open-failure cleanup contract

If executeCmd(MediaIOCommandOpen &) returns an error, MediaIO will automatically dispatch executeCmd(MediaIOCommandClose &) on the same task instance to give it a chance to release any resources it allocated before the failure. Backends MUST handle being closed from a failed-open state without crashing — typically by checking whether each resource is valid before releasing it. The same applies to a normal Close after a successful open.

Parameterized commands

Backends can expose operations beyond the standard set by overriding executeCmd(MediaIOCommandParams &). Dispatch on cmd.name and read cmd.params; populate cmd.result for the caller. Return Error::NotSupported for unrecognized names.

Applies the standard Output* overrides from config to input, returning the resulting MediaDesc.

Every transform backend (CSC, FrameSync, SRC, VideoEncoder, VideoDecoder) consumes the same family of MediaConfig keys to express "given this input shape, produce this output":

  • MediaConfig::OutputPixelDesc — replaces the ImageDesc::pixelDesc of every video image.
  • MediaConfig::OutputFrameRate — replaces MediaDesc::frameRate.
  • MediaConfig::OutputAudioRate — replaces AudioDesc::sampleRate on every audio entry.
  • MediaConfig::OutputAudioChannels — replaces AudioDesc::channels on every audio entry.
  • MediaConfig::OutputAudioDataType — replaces AudioDesc::dataType on every audio entry.

Keys at their default (invalid PixelDesc, invalid FrameRate, zero audio rate / channels, invalid Enum) mean "inherit from input" — the corresponding field passes through unchanged.

The pipeline planner uses this helper to compute what shape a configured transform will produce without instantiating it. Backend authors call it from MediaIOTask::proposeOutput / MediaIOTask::describe so every transform answers the planner uniformly.

The MediaDesc the transform will produce.

◆ defaultUncompressedPixelDesc()

static PixelDesc MediaIOTask::defaultUncompressedPixelDesc ( const PixelDesc &  source)
static

Returns a canonical uncompressed PixelDesc that stays in the same family as source.

Transform backends that refuse compressed or paint-engine-less input (Burn, FrameSync, ...) use this to ask the pipeline planner for a same-family uncompressed substitute in proposeInput. The planner's VideoDecoder / CSC bridges then close the gap. The helper keeps the mapping in one place so the fallback does not drift between backends.

  • YCbCr sources → PixelDesc::YUV8_422_Rec709
  • RGB / other → PixelDesc::RGBA8_sRGB

◆ stampWorkBegin()

void MediaIOTask::stampWorkBegin ( )

Stamps the work-begin benchmark on the active frame.

Call this at the point where the task begins its real per-frame processing work (after any pacing or throttling). Paired with stampWorkEnd(), this brackets the actual processing cost so the framework can report it separately from end-to-end latency.

The infrastructure sets the active benchmark pointer before calling executeCmd() and clears it after, so this works for both reads and writes with no arguments needed. No-op when benchmarking is disabled or the task is unattached.
