Detailed Description

template<typename T>
class promeki::VariantLookup< T >::Registrar

Fluent builder returned by registrar.

All methods side-effect into the per-T static registry and return a fresh Registrar so registrations can be chained. The instance itself is stateless; it exists only so users can write the PROMEKI_LOOKUP_REGISTER block as a single expression.

Note
Registration methods are marked const because Registrar carries no per-instance state — the const here denotes "no @c Registrar members are modified", not "no side effects". Each call takes a write lock on the per-T static registry and inserts the handler there.

Registers a prefixed database binding.

Looking up "Prefix.DbKey" calls get on the instance then resolves DbKey through the returned VariantDatabase. The remainder is treated opaquely: it is handed to VariantDatabase<DbName>::ID::find without further parsing. A null get result yields Error::IdNotFound; an unregistered key also yields Error::IdNotFound. Spec lookup delegates to VariantDatabase<DbName>::specFor(rest), so declared types for keys like Meta.Timecode surface uniformly.

Registers an indexed child where the child target is constructed by value on demand.

Companion to indexedChild for types that can be cheaply materialised from the parent but aren't stored anywhere long-term — the textbook case is a lightweight proxy like BufferView::Entry that only holds a parent pointer and an index. get returns a std::optional<U> by value; resolve borrows the temporary long enough to run VariantLookup<U>::resolve on it.

Read-only: the by-value shape rules out assign — there is no stable target to hold a mutation. Consumers that need to mutate must use the pointer-returning indexedChild overload.

Template Parameters
UThe child type. Must be registered with its own VariantLookup<U>.
Parameters
nameSegment name (e.g. "Buffer").
getComputes the child on demand; return std::nullopt to signal out-of-range. The dump walker stops at the first nullopt.

Declares T's C++ base for lookup cascade.

When a lookup on T misses every handler, the dispatcher upcasts instance to Base and delegates to VariantLookup<Base>::resolveDirect (or the assign / specFor peer) so the derived class only registers what it adds. Composes transitively — the Base may itself declare inheritsFrom<SuperBase>().

Registers a variant-tree binding under name.

Lets a type expose a free-form Variant — typically holding a VariantList or VariantMap — and have lookups descend into the tree using the standard dotted / indexed key syntax:

  • "Name" — terminal, returns the whole Variant.
  • "Name.sub" — descends if the Variant holds a VariantMap.
  • "Name[N]" — descends if the Variant holds a VariantList.
  • "Name[N].sub" — combines the above.

Spec discovery follows the same shape: the spec parameter is the spec for the top-level Variant, and trailing path fragments walk the spec's VariantSpec::elementSpec / VariantSpec::valueSpec chain so specFor("Name.sub") returns the inner spec when one was declared.

Parameters
nameSegment name.
getFunction returning the Variant at Name on the instance. Returning an invalid Variant for terminal lookups yields std::nullopt with Error::IdNotFound.
specOptional VariantSpec describing the top-level Variant's shape. When set with element / value sub-specs, the cascade surfaces them via specFor.
Note
Read-only. Mutating a sub-path through a variantTree binding is not yet supported because it requires splicing a new Variant back into the registered tree. Callers that need write access should expose a top-level scalar setter that takes a whole new Variant.
