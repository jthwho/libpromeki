Detailed Description

Provides information about a file system entry.

Wraps std::filesystem to query file metadata such as existence, type, size, permissions, and path components. The file status is lazily cached and can be force-refreshed.

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized — the lazy status cache is mutated by read-only-looking accessors.
Example
FileInfo info("/path/to/video.mxf");
bool exists = info.exists();
size_t bytes = info.size();
String name = info.fileName(); // "video.mxf"
String ext = info.extension(); // "mxf"

◆ lastModified()

DateTime promeki::FileInfo::lastModified ( ) const
inline

Returns the file's last-modified wall-clock time.

Returns a default-constructed DateTime (epoch value) when the path is missing or last_write_time fails — callers that need to distinguish "epoch" from "missing" should FileInfo::exists first.

Used by HTTP responses (ETag / Last-Modified) and anywhere a wall-clock view of file mtime is needed.

Returns
The last-modified time, or epoch on failure.
