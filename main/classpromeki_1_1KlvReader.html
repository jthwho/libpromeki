Detailed Description

Binary-stream reader for KlvFrame framing.

Reads KLV frames from any IODevice. Typical use is over a LocalSocket, TcpSocket, or a BufferIODevice for in-memory testing.

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized. Thread-safety of the underlying IODevice also applies.

Reads a complete frame (Key + Length + Value) in one call.

The value payload is allocated into out.value. If the frame's length exceeds maxValueBytes, returns Error::TooLarge without consuming the value (leaving the stream at an indeterminate position — typically the caller closes the connection).

maxValueBytesMaximum accepted value size.
Returns
