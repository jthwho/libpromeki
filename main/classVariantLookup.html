Detailed Description

template<typename T>
class VariantLookup< T >

Type-parameterised registry of named accessors on T.

Template Parameters
TThe target type this lookup resolves against.

Generalises the pseudo-metadata lookup that Frame, Image and Audio all implement by hand. Each well-known key registers a small std::function against one of four handler kinds, and a two-line resolve / assign dispatch pair replaces the resolvePseudoKey + resolveKey + subscript-parsing boilerplate per type.

Grammar
key := segment ( '.' segment )*
segment := name ( '[' index ']' )?
List
Dynamic array container wrapping std::vector.

Handler kinds
  • scalar — terminal "Name" producing a Variant.
  • indexedScalar — terminal "Name[N]" producing a Variant.
  • child — composition "Name.rest" delegating to VariantLookup<U>::resolve on a borrowed const U&.
  • indexedChild — composition "Name[N].rest" delegating to VariantLookup<U>::resolve on a borrowed const U&.
  • database<Name> — composition "Prefix.DbKey" resolving DbKey through a borrowed VariantDatabase.

Each handler optionally accepts a setter / mutable accessor. Omitting it marks the path read-only, causing assign to fail with Error::ReadOnly. Terminal handlers also accept an optional VariantSpec pointer describing the value's declared type; the spec surfaces through specFor for callers that need to coerce string literals to the key's native type (query compilers, introspectors, configuration tools).

Lookup keys
Internally every registered name is hashed to a uint64_t via 64-bit FNV-1a and stored in hash-keyed maps so resolve / assign / specFor avoid per-call string comparisons. A per-T reverse map preserves names for introspection. The Key nested type is a lightweight handle around the same uint64_t ID with a constexpr Key::literal factory, so callers that already know which terminal they want can skip the name round-trip with a typed overload.
Registration
Handlers register at static-init time via the PROMEKI_LOOKUP_REGISTER macro and the fluent Registrar builder:
PROMEKI_LOOKUP_REGISTER(Image)
.scalar("Width", [](const Image &i){ return Variant(i.desc().width()); })
.scalar("Height", [](const Image &i){ return Variant(i.desc().height()); })
.database<"Metadata">("Meta",
[](const Image &i) -> const VariantDatabase<"Metadata">* {
return &i.desc().metadata();
});

Error codes
  • Error::Ok — resolution / assignment succeeded.
  • Error::IdNotFound — no handler matched the leading segment.
  • Error::ParseFailed — malformed key (empty name, unterminated bracket, trailing dot, etc.).
  • Error::OutOfRange — subscripted child / indexed scalar got an out-of-range index.
  • Error::ReadOnly — assign hit a handler without a setter or a read-only traversal.
  • Error::ConversionFailed — setter rejected the supplied value.
Note
Registrations are thread-safe but are expected at static-init time; the lookup path uses a shared read lock.

Substitutes {Key[:spec]} placeholders against instance.

Walks tmpl character by character. Each balanced {…} token is split on the first ':' into a key name and an optional format spec; the key is resolved via resolve and the returned Variant is rendered via Variant::format. Literal braces are escaped as "{{" and "}}", matching the std::format convention.

When resolve returns std::nullopt, the optional resolver callback is consulted with (keyName, spec). A returned std::optional<String> holding a value is substituted as-is. A returned std::nullopt (or an absent resolver) leaves the key unresolved: the output gets the literal text "[UNKNOWN KEY: <name>]" and err is set to Error::IdNotFound.
