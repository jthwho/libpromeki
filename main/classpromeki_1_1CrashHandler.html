Detailed Description

Installs signal handlers for fatal crash signals.

When installed, catches SIGSEGV, SIGABRT, SIGBUS, SIGFPE, and SIGILL, and writes a crash report to stderr and a log file before re-raising the signal for core dump generation.

The signal handler is designed to be as signal-safe as possible: all header output uses raw write() calls, the crash log path is pre-built at install time (no heap allocation until the stack trace stage), and thread enumeration on Linux uses direct syscalls against /proc/self/task.

The crash report includes:

  • Signal name and number
  • PID and timestamp
  • Crashing thread TID and name
  • List of all threads with their TIDs and names
  • Stack trace via backtrace, with C++ symbols demangled by abi::__cxa_demangle (falls back to raw frames via backtrace_symbols_fd if allocation fails)
  • Path to the saved crash log file

The stack-trace stage is not strictly async-signal-safe (it uses backtrace_symbols and abi::__cxa_demangle, both of which allocate), but it runs after all critical header and thread information has already been written via signal-safe primitives. A failure during demangling costs us a prettier trace — nothing more.

After the stack trace, a best-effort promekiLogSync() flushes any buffered logger output.

Controlled by LibraryOptions::CrashHandler (default true) and wired into Application automatically.

Example
// Automatic via Application (default):
Application app(argc, argv);
// crash handlers are now active
// Manual control:
CrashHandler::install();
// ...
CrashHandler::uninstall();

◆ install()

static void promeki::CrashHandler::install ( )
static

Installs crash signal handlers.

Registers handlers for SIGSEGV, SIGABRT, SIGBUS, SIGFPE, and SIGILL. Pre-builds the crash log file path from the current Application::appName and process ID.

If LibraryOptions::CoreDumps is true, also raises RLIMIT_CORE to the hard limit.

Snapshots the following process state into static buffers so the signal handler can write a report without touching any non-signal-safe APIs:

  • Command line (Application::arguments)
  • Hostname, working directory, uname info
  • Environment (if LibraryOptions::CaptureEnvironment)
  • LibraryOptions values
  • The set of registered MemSpace entries (metadata only — counter values are read live at crash time via atomic loads)

State that changes after install() — such as MemSpace instances registered by a subsystem later in startup — will not appear in crash reports until the snapshot is refreshed. Use Application::refreshCrashHandler to re-snapshot.

Safe to call multiple times — subsequent calls update the pre-built state but do not stack handlers.

◆ writeTrace()

static void promeki::CrashHandler::writeTrace ( const char *  reason = nullptr)
static

Writes a diagnostic trace report to a unique file.

Produces the same report a crash would produce — process identity, build info, OS info, memory stats, thread list, stack trace, memory map — but without re-raising a signal. Intended for programmatic diagnostic snapshots from inside the application (e.g. when detecting a soft failure or an inconsistent state).

The trace is written to: 

* <crashLogDir>/promeki-trace-<appname>-<pid>-<seqno>.log
*

where seqno is an internal atomic counter that makes each call's filename unique within the process.

May be called before install(); if so, some of the snapshotted fields (host, working directory, uname data) will be missing from the report but the trace itself is still written. Calling install() once on startup ensures a complete report.

This function is not async-signal-safe — it allocates via String and calls abi::__cxa_demangle in the stack trace path. Do not call it from a signal handler.

Parameters
reasonOptional human-readable reason for the trace. If non-null, appears in the trace header.
