Detailed Description

RFC 4648 Base64 encode / decode helpers.

Standard Base64 alphabet ("+/", padding with '='): the encoder always pads, the decoder accepts both padded and unpadded input. All overloads operate on raw byte ranges and Buffer / String containers — no allocation surprises.

Decodes a Base64 string into a Buffer.

Whitespace inside text is ignored so callers can feed PEM-style line-wrapped data directly. Padding is optional.
