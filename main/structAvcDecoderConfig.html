Detailed Description

AVCDecoderConfigurationRecord — the payload of an avcC box inside an H.264 MP4 / QuickTime sample description.

Defined by ISO/IEC 14496-15 §5.3.3.1.2. Carries the out-of-band parameter sets (SPS, PPS) and profile / level identifiers that an H.264 decoder needs to initialize before it can decode the VCL NAL units stored in the MP4 mdat.

Typical producer flow (writer side)
// On the first IDR access unit, extract parameter sets and
// build the avcC record to place inside the stsd entry.
AvcDecoderConfig cfg;
H264Bitstream::fromAnnexB(idrAccessUnit, cfg); // not here; see fromAnnexB
Buffer::Ptr avccPayload;
cfg.serialize(avccPayload);
qtWriter.setTrackAvcC(trackId, avccPayload);

Extended-profile fields not modeled
The record has additional fields (chroma_format, bit_depth_luma_minus8, bit_depth_chroma_minus8, SPS-Ext NALs) that appear only for profile_idc values 100, 110, 122, or 144. This implementation does not read or write them; they default to the standard 8-bit 4:2:0 interpretation when absent. A follow-up can add parsing for high-profile workflows when the codecs need them.

AvcDecoderConfig &  out 
)
static

Populate an AvcDecoderConfig from an Annex-B access unit.

Walks the input for H.264 SPS (NAL type 7) and PPS (NAL type 8) units, appending each to sps / pps. Extracts avcProfileIndication / profileCompatibility / avcLevelIndication from bytes 1-3 of the first SPS found. Parameter sets are stored as freshly allocated Buffer::Ptr copies so the result outlives au.

Parameters
auAn Annex-B access unit that contains at least one SPS and one PPS — typically the access unit of the first IDR frame emitted by the encoder.
outReceives the populated configuration. On failure, out is left partially populated (do not use).
Returns
Error::Ok on success, Error::CorruptData if the input contains no start codes, or Error::InvalidArgument if the input has no SPS.
