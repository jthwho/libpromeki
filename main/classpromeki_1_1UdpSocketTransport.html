Detailed Description

UDP socket implementation of PacketTransport.

UdpSocketTransport owns a kernel UdpSocket and delegates every PacketTransport operation to it. This is the default transport used by RtpSession and the RTP MediaIO backends; it supports kernel pacing via SO_MAX_PACING_RATE, batch send via sendmmsg(), and transmit-time scheduling via SO_TXTIME on platforms that implement those options.

Configuration

Configuration is applied at open() time based on members set before the call. The bind address defaults to 0.0.0.0:0. Optional knobs include multicast TTL, multicast outgoing interface, DSCP marking, and IPv6 mode.

Example
UdpSocketTransport transport;
transport.setLocalAddress(SocketAddress::any(0));
transport.setDscp(46);
Error err = transport.open();
PacketTransport::Datagram d;
d.data = payload.data();
d.size = payload.size();
d.dest = SocketAddress(Ipv4Address(239, 0, 0, 1), 5004);
PacketTransport::DatagramList batch;
batch.pushToBack(d);
transport.sendPackets(batch);

Sends a batch of packets.

Backends implement this as sendmmsg() on Linux sockets, as a ring-burst on DPDK, or as a queue enqueue on loopback. The call returns the number of datagrams the transport accepted before erroring; a fully successful call returns datagrams.size().
