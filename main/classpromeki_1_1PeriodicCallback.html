Detailed Description

Calls a function at a fixed interval when serviced.

PeriodicCallback is a passive timer: it does not spawn threads or use system timers. The caller drives it by calling service() from whatever loop or thread it already runs on. When the configured interval has elapsed since the last invocation, service() calls the stored function and resets the clock.

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent calls to service / setters on a single instance must be externally synchronized — typically by confining the instance to one driver thread.
Example
PeriodicCallback pc(1.0, []{ promekiDebug("tick"); });
while(running) {
doWork();
pc.service();
}
