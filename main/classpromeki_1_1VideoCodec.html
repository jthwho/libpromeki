Detailed Description

First-class identifier for a video codec family.

Uses the TypeRegistry pattern: a lightweight inline wrapper around an immutable Data record, identified by an integer ID. Well-known codecs are provided as named enum constants; applications and third-party backends can register additional codecs at runtime via registerType and registerData.

Multi-backend support

Every codec family can have any number of concrete backends (NVIDIA NVENC/NVDEC, libjpeg-turbo, SVT-JPEG-XS, vendored native implementations, application-supplied overrides, …). Backends are identified by a typed Backend handle rather than naked strings, registered in a process-wide VideoCodecBackendRegistry via registerBackend / lookupBackend, and attached to specific (codec, backend) pairs by the concrete VideoEncoder / VideoDecoder factory registration.

The wrapper can optionally carry a pinned Backend:

  • VideoCodec(VideoCodec::H264) — unpinned; createEncoder picks the highest-weight available backend (further overridden by MediaConfig::CodecBackend when that key is present).
  • VideoCodec(VideoCodec::H264, nvidiaBackend) — pinned specifically to the NVIDIA backend; createEncoder fails with Error::IdNotFound if the NVIDIA backend is not registered for H264.
String form

Serialized codec values travel as "<CodecName>" or "<CodecName>:<BackendName>" — e.g. "H264" for unpinned, or "H264:Nvidia" to pin the NVIDIA backend. fromString parses both forms into a fully resolved wrapper and returns Error::IdNotFound when the codec or backend is unknown. toString emits the canonical form.

Capability introspection

Data exposes rich codec-family metadata — inter- vs intra-frame coding, lossless capability, bit-depth support, HDR metadata support, rate-control modes, and more — to let pipeline planners reason about codec suitability before instantiating any session. Per-backend capabilities (specific input PixelFormats, bitrate ceilings, …) live on the backend record and are queried via encoderSupportedInputs / decoderSupportedOutputs.

Example
// Look up by ID and create a session:
VideoCodec codec(VideoCodec::H264);
auto result = codec.createEncoder();
if(isOk(result)) { VideoEncoder *enc = value(result); … }
// Look up by string name, with optional backend pin:
auto parsed = VideoCodec::fromString("H264:Nvidia");
if(isOk(parsed)) { VideoCodec pinned = value(parsed); … }
// Enumerate every registered codec:
for(auto id : VideoCodec::registeredIDs()) {
VideoCodec vc(id);
printf("%s — %s\n", vc.name().cstr(), vc.description().cstr());
}

Thread Safety
Fully thread-safe. The VideoCodec handle wraps an integer ID and is safe to share by value across threads. Registrations are expected at static-init time and the registry is internally synchronized; thereafter lookup is lock-free.

  1. The wrapper's pinned backend (backend), when valid.
  2. MediaConfig::CodecBackend on config, when set and parses as a registered backend.
  3. The highest-weight registered encoder backend for this codec.
