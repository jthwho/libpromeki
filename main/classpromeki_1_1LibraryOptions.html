Detailed Description

Library-wide option database populated from environment variables.

Provides a central registry of configuration knobs that affect the behaviour of the promeki library itself. Each option is declared with a VariantSpec that captures its type, default, and description.

On startup the Application constructor calls loadFromEnvironment which scans the process environment for variables matching PROMEKI_OPT_<Name> and merges them — type-coerced via VariantSpec::parseString — into the singleton instance().

Options can also be set programmatically at any time:

Thread Safety
Inherits VariantDatabase — distinct instances may be used concurrently. The singleton instance is intended to be configured once at startup (typically in main before worker threads start) and read thereafter from any thread. Concurrent mutation requires external synchronization.
Example
// Override an option before Application construction
LibraryOptions::instance().set(LibraryOptions::CoreDumps, true);
// Or via the environment:
// export PROMEKI_OPT_CoreDumps=true

◆ loadFromEnvironment()

void promeki::LibraryOptions::loadFromEnvironment ( )

Populates options from PROMEKI_OPT_* environment variables.

Scans the process environment for variables whose name begins with PROMEKI_OPT_, strips the prefix, and attempts to resolve the remainder as a registered option name. The value string is type-coerced via the option's VariantSpec and merged into this database. Unrecognised names are logged as warnings and skipped.

◆ IpcDir

constexpr ID promeki::LibraryOptions::IpcDir
staticconstexpr

String — override for the path returned by Dir::ipc (empty = platform default, which is /dev/shm/promeki on Linux — a tmpfs location well suited to both shared-memory objects and named AF_UNIX sockets — and Dir::temp elsewhere).

The IPC directory is where cross-process primitives (shared memory regions, local-socket files, lock files) live. Set this option to pin the path for every IPC-consuming feature in the library in one place. Common use cases:

  • Move IPC traffic off a read-only or restricted /dev/shm onto a writable partition.
  • Isolate test runs by pointing at a per-test directory.
  • Align with a site's /run layout for cross-user IPC: create a group-owned directory at deployment time and point every process at it.

Set via code:

LibraryOptions::instance().set(
LibraryOptions::IpcDir,
String("/run/promeki/ipc"));

◆ TempDir

constexpr ID promeki::LibraryOptions::TempDir
staticconstexpr

String — override for the path returned by Dir::temp (empty = use the OS default, which is std::filesystem::temp_directory_path — typically /tmp on Linux, TEMP% on Windows).

Set this option to pin every Dir::temp call — and every consumer downstream of it (crash logs, scratch JSON, ad-hoc test output, etc.) — to a dedicated directory. The common use case is moving temp traffic off a tmpfs-backed /tmp onto a disk-backed mount so large scratch files don't consume RAM. Set via code:

LibraryOptions::instance().set(
LibraryOptions::TempDir,
String("/mnt/data/tmp/promeki"));
