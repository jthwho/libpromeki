Detailed Description

RFC 6455 WebSocket endpoint, client- or server-side.

WebSocket implements the framing and connection-management half of the WebSocket protocol on top of TcpSocket (or SslSocket for wss://). The HTTP upgrade half is shared with the HTTP stack: client sockets perform the upgrade inside connectToUrl; server sockets are produced by HttpServer::routeWebSocket once the HTTP/1.1 101 Switching Protocols handshake has completed.

Thread Safety
Inherits ObjectBase — thread-affine. Every WebSocket lives on exactly one EventLoop — the loop that's EventLoop::current at construction time. All frame parsing, signal emission, and write pumping happens on that loop's thread. Cross-thread callers should connect to the textMessageReceivedSignal / binaryMessageReceivedSignal signals; the standard signal-dispatch machinery moves the call to the receiver's loop.
Usage (client)
WebSocket *ws = new WebSocket();
ws->connectedSignal.connect([ws]() {
ws->sendTextMessage("hello");
});
ws->textMessageReceivedSignal.connect([](String msg) {
promekiInfo("recv: %s", msg.cstr());
});
ws->connectToUrl("ws://localhost:9000/echo");

Limitations
v1 implements text + binary messages, ping / pong, and the close handshake. Per-message-deflate and other extensions are not negotiated. Subprotocols can be set on the client and are echoed verbatim by the server route helper.

◆ connectToUrl()

Error promeki::WebSocket::connectToUrl ( const String &  url)

Connects to a ws:// or wss:// URL.

Resolves the host, opens an underlying socket (TcpSocket for ws, SslSocket for wss), and drives the HTTP upgrade. The call itself returns immediately; connectedSignal fires when the upgrade completes successfully and errorOccurredSignal fires on failure.

Parameters
urlTarget URL.
Returns
Error::Ok if the connect was initiated successfully (note: not "connected"), Error::Invalid for a malformed URL, Error::HostNotFound for DNS failure, Error::AlreadyOpen if a connection is already in progress.

◆ setSslContext()

void promeki::WebSocket::setSslContext ( SslContext::Ptr  ctx)
inline

Attaches an SslContext used for wss:// connects.

Without one, a default-constructed SslContext (no CA chain → effectively no peer verification) is created lazily. Mirrors HttpClient.

Always available regardless of PROMEKI_ENABLE_TLS; in a TLS-disabled build the context is stored but its operations are inert (see SslContext) and an actual wss connect is later rejected with Error::NotImplemented.
