Detailed Description

Pattern-based dispatcher for HTTP requests.

Models Go's http.ServeMux: route patterns map to handlers, with exact matches preferred over parameterized matches and longer literal segments preferred over shorter ones. Method-aware: a pattern registered for HttpMethod::Get does not respond to POST, and the default 405 handler fills in the Allow header automatically based on the methods that did match the path.

Pattern syntax
  • "/items" — literal path, exact match.
  • "/items/{id}" — single-segment placeholder; matches "/items/42" but not "/items/42/edit".
  • "/files/{path:*}" — greedy tail placeholder; matches the rest of the path (zero or more segments).
  • Trailing slash on a literal pattern is significant: "/api/" only matches "/api/" exactly, not "/api".

Path parameters captured during matching are stuffed into the HttpRequest::pathParams map (the request is mutated in-place inside dispatch); handlers read them via HttpRequest::pathParam.

Middleware
Middleware registered via use runs before the matched handler in registration order. Each middleware receives a next callable it must invoke (synchronously) to continue the chain — short-circuit by simply returning without calling next.
Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently. A single router is intended to be configured at startup and then used read-only by HttpServer's owning EventLoop thread; concurrent route mutation alongside dispatch requires external synchronization.
Example
HttpRouter mux;
mux.use([](const HttpRequest &req, HttpResponse &res, auto next) {
promekiInfo("%s %s", req.method().wireName().cstr(), req.path().cstr());
next();
});
mux.route("/api/items", HttpMethod::Get, [](const auto &, auto &res) {
res.setJson(allItems());
});
mux.route("/api/items/{id}", HttpMethod::Get, [](const auto &req, auto &res) {
auto id = req.pathParam("id");
// ...
});
