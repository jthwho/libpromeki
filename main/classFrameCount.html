Detailed Description

Count of frames, optionally empty, unknown, or infinite.

Stores a single int64_t with four states encoded in the storage value:

State Storage value Meaning
Unknown -1 Count is not known (NaN-like).
Infinity -2 Unbounded / open-ended count.
Empty 0 Zero frames (a valid count).
Valid >0 Concrete positive frame count.

FrameCount is a Simple data object: plain value, no heap, no SharedPtr. It pairs with FrameNumber to form MediaDuration (a starting frame plus a length).

Sentinel arithmetic
  • Any operation involving Unknown yields Unknown (NaN-like poisoning).
  • Infinity + finite = Infinity, Infinity + Infinity = Infinity, Infinity - Infinity = Unknown.
  • Finite - Infinity = Unknown (no negative counts).
  • Finite arithmetic that would produce a negative result also poisons to Unknown.
Comparisons
  • operator== compares storage values directly, so two Unknown counts compare equal and two Infinity counts compare equal.
  • Ordering (<, <=, >, >=) returns false when either side is Unknown (NaN-like); among the remaining states Empty < finite valid < Infinity.
String format
  • Unknown → empty string
  • Infinity → "inf"
  • otherwise → decimal count followed by 'f' (e.g. "50f")
Example
FrameCount c(50);
assert(c.isValid() && c.isFinite());
assert(c.toString() == "50f");
FrameCount inf = FrameCount::infinity();
assert(inf.isInfinite());
FrameCount u = FrameCount::unknown();
assert(u.isUnknown());
FrameCount sum = c + inf; // Infinity
FrameCount bad = inf - inf; // Unknown

Parses a FrameCount from a string.

Accepted forms (leading/trailing whitespace ignored, case insensitive where relevant):

  • empty string → Unknown
  • "unknown" / "unk" / "?" → Unknown
  • "inf" / "infinity" / "∞" → Infinity
  • decimal integer >= 0, with an optional trailing 'f' (e.g. "0", "0f", "50", "50f")

◆ operator<=()

bool FrameCount::operator<= ( const FrameCount &  other) const
inline

Less-than-or-equal ordering (NaN-like).

Returns false if either operand is Unknown — so Unknown <= Unknown is false even though Unknown == Unknown is true. This preserves the NaN-like invariant that no ordering comparison against Unknown ever yields true.
