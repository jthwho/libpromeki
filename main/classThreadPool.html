Detailed Description

General-purpose thread pool for submitting callable tasks.

Tasks are submitted via submit() which returns a Future for the result. The pool manages a set of worker threads that pull tasks from an internal

By default, worker threads are spawned lazily — the first submit() that finds all existing workers busy spawns a new thread, up to the configured maximum. Pass lazy=false to the constructor (or call setThreadCount()) to pre-spawn all threads immediately.

Example
ThreadPool pool(4); // up to 4 worker threads, spawned on demand
Future<int> result = pool.submit([]() {
return expensiveComputation();
});
int value = result.get(); // blocks until ready

void ThreadPool::setNamePrefix ( const String &  prefix)

Sets the name prefix for worker threads.

Each worker is named prefix0, prefix1, etc. The name is applied to the OS thread (visible in debuggers, htop, ps -L). Must be called before threads are spawned to take effect on all workers.

Resizes the thread pool.

Shuts down all existing threads, then respawns up to count threads. When lazy is true (default), no threads are spawned until tasks arrive. A count of 0 means subsequent tasks run inline.

Parameters
countThe new maximum thread count.
