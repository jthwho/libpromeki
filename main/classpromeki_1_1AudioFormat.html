Detailed Description

First-class audio sample-format descriptor.

Uses the TypeRegistry pattern: a lightweight inline wrapper around an immutable Data record, identified by an integer ID. Well-known formats are provided as ID constants; user-defined formats can be registered at runtime via registerType and registerData — exactly parallel to PixelFormat on the pixel side.

AudioFormat describes the format of an audio stream (sample type, bit depth, endianness, planar vs interleaved, optional compressed codec identity) independent of sample rate and channel count. The tuple format + sampleRate + channels is bundled by AudioDesc.

PCM vs compressed

PCM formats (e.g. PCMI_S16LE, PCMP_Float32LE) populate every field meaningfully and expose sample conversion helpers via samplesToFloat / floatToSamples. Compressed formats (e.g. Opus, AAC) set isCompressed to true and carry an audioCodec identifying the codec family; their sample-level fields are zero / default because a compressed bitstream has no single bit depth or endianness.

Naming convention

C++ identifiers and string names for PCM use a short prefix followed by the sample type and byte order:

  • PCMI_ — interleaved PCM (all channels stored sample-by-sample).
  • PCMP_ — planar PCM (each channel stored in its own plane).

After the prefix, the name spells out the sample type (S8, U8, S16LE, S32BE, Float32LE, …). Compressed formats use the codec's canonical name as the identifier (Opus, AAC, FLAC, MP3, AC3).

Example usage
AudioFormat fmt(AudioFormat::PCMI_S16LE);
assert(fmt.bytesPerSample() == 2);
assert(fmt.isSigned());
assert(!fmt.isCompressed());

Converts samples from this format to dst.

Uses the registered direct converter when one exists for the (this, dst) pair, otherwise falls back to a via-float trip through samplesToFloat / floatToSamples using the supplied scratch buffer (must hold samples float values). When the fastpath hits, scratch is unused and may be nullptr.

Both buffers must be sized for samples in their respective formats (interleaved: total samples; planar: per-channel run).

Parameters
dstDestination format.
outDestination bytes.
inSource bytes.
samplesNumber of samples to convert.
scratchFloat scratch buffer (samples floats); ignored on the direct path.
Returns
Error::Ok on success, Error::NotSupported when no direct converter exists and at least one side is compressed (no via-float possible), Error::InvalidArgument when scratch is null but the via-float fallback is required.

◆ convertTo() [2/2]

Error promeki::AudioFormat::convertTo ( const AudioFormat &  dst,

size_t  samplesPerChannel,
size_t  channels,
float *  scratch = nullptr 
) const

Channel-aware overload that bridges planar / interleaved layouts.

The samplesPerChannel parameter is the per-channel frame count (stereo at 1 frame = 2 floats total). When src and dst differ only in isPlanar(), the converter transposes the float-domain data accordingly:

  • planar → interleaved: per-channel runs [ch0[0..N), ch1[0..N), ...] are folded into L,R,L,R,... order.
  • interleaved → planar: vice versa.

The format-level byte conversions (samplesToFloat / floatToSamples) themselves are layout-agnostic — they walk a contiguous run of values — so the transpose step is the only thing the planar IDs need beyond their interleaved siblings.

Parameters
dstDestination format.
outDestination bytes.
inSource bytes.
samplesPerChannelPer-channel frame count.
channelsChannel count (≥ 1).
scratchFloat scratch buffer of at least samplesPerChannel × channels floats. May be nullptr only when a direct converter handles the (src,dst) pair and no transpose is required.
Returns
Error::Ok on success.

◆ isBitAccurate()

static bool promeki::AudioFormat::isBitAccurate ( ID  src,
ID  dst 
)
static

Returns true if a registered (src, dst) path is bit-accurate.

A converter is bit-accurate when the destination sample pattern preserves every input bit, allowing trivial reversible repositioning (endian byte swap, signed↔unsigned sign flip, lossless upcast with zero-fill of unused lower bits). Bit-accurate paths are the only ones safe for non-PCM data carried in audio payloads.

Returns false if no direct converter is registered for the pair, or if the registered converter is not flagged bit-accurate.
