Detailed Description

Command to replace the Clock bound to a port group.

Built by MediaIOPortGroup::setClock and dispatched through MediaIO::submit. The backend's executeCmd(MediaIOCommandSetClock &) decides whether it can honor the new clock as its timing reference — typically a sender uses the supplied clock to pace outbound frames against an upstream device clock (capture card, AES67, PTP grandmaster).

On Error::Ok the framework swaps the clock pointer on group inside MediaIO::completeCommand; on any error the existing clock is left in place, so group->clock() always reflects what is actually in effect.

Default base implementation returns Error::NotSupported. A null clock means "restore the backend's default clock behavior" — backends that override this command interpret a null input as an explicit unbind.
