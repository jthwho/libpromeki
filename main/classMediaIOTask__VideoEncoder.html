Detailed Description

MediaIOTask backend that encodes uncompressed Frames into compressed packets.

MediaIOTask_VideoEncoder wires a VideoEncoder session into the MediaIO command/signal model. It accepts a Frame on writeFrame(), feeds image[0] into the encoder via VideoEncoder::submitFrame, and drains VideoEncoder::receivePacket producing one output Frame per emitted MediaPacket. Audio tracks on the source Frame are forwarded alongside each output packet so downstream stages still see them on the same PTS.

The registered backend name is "VideoEncoder"; callers pick a concrete codec via the MediaConfig::VideoCodec key (e.g. "H264", "HEVC") — the task looks up the matching factory through VideoEncoder::createEncoder.

Mode support

Only MediaIO::InputAndOutput is supported — the backend has no independent source or sink of its own.

First-cut limitations
  • Only the first video image in a Frame is encoded; additional images are dropped with a one-shot warning. Stereoscopic / multi-camera encoding is a follow-up.
  • The concrete VideoEncoder must operate in a 1-in / 1-out regime (no B-frames, no look-ahead) so every writeFrame produces its corresponding packet(s) synchronously. The NvencVideoEncoder v1 already satisfies this. B-frame support needs an EOS protocol between the pipeline and this task and is deferred.
  • No SPS/PPS/VPS extraction — parameter sets are concatenated with the first IDR in a single MediaPacket, which is what the NVENC backend naturally emits. Splitting them into their own MediaPacket::ParameterSet packets (using the BufferView slicing we just landed) comes when the RTP H.264 / MP4 sinks need it.

Example
MediaIO::Config cfg;
cfg.set(MediaConfig::Type, "VideoEncoder");
cfg.set(MediaConfig::VideoCodec, "H264");
cfg.set(MediaConfig::BitrateKbps, 8000);
cfg.set(MediaConfig::VideoPreset, VideoEncoderPreset::LowLatency);
MediaIO *enc = MediaIO::create(cfg);
enc->setMediaDesc(upstreamDesc);
enc->open(MediaIO::InputAndOutput);
enc->writeFrame(nv12Frame);
Frame::Ptr encoded;
enc->readFrame(encoded); // encoded->packetList()[0] is the H.264 packet.
enc->close();
