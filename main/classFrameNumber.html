Detailed Description

Absolute frame index along a media timeline.

Stores a single int64_t. Non-negative values identify a specific frame (zero-based). The sentinel value -1 means "unknown" — for example, a decoder that has not yet produced a frame, or metadata that was never populated. Any other negative storage value is canonicalised to Unknown on construction.

FrameNumber is a Simple data object: plain value, no heap, no SharedPtr. It carries no frame-rate context and therefore cannot convert to Timecode on its own — use Timecode::fromFrameNumber when a rate is known.

Sentinel semantics
  • Any arithmetic operation involving an Unknown operand yields Unknown (NaN-like poisoning).
  • Two Unknown values compare equal; ordering uses the raw int64_t value, so Unknown (-1) sorts below every valid frame number. This makes FrameNumber usable in sorted containers without surprises.
Example
FrameNumber n = 42;
assert(n.isValid());
n += 3; // 45
FrameNumber u; // default = Unknown
assert(u.isUnknown());
FrameNumber x = n + u; // poisons to Unknown
assert(x.isUnknown());
String s = n.toString(); // "45"
Error err;
FrameNumber p = FrameNumber::fromString(" 100 ", &err);
assert(err.isOk() && p.value() == 100);

Parses a FrameNumber from a string.

Accepted forms (leading/trailing whitespace ignored, case insensitive where relevant):

  • "unknown" / "unk" / "?" → Unknown
  • decimal integer >= 0 → the corresponding frame
