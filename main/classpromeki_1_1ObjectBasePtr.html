Detailed Description

template<typename T = ObjectBase>
class promeki::ObjectBasePtr< T >

Object that holds a pointer to an ObjectBase-derived object.

Registers itself with the tracked object so that when the object is destroyed the internal pointer is atomically cleared, giving you a safe, non-dangling reference. The template parameter T selects the concrete type returned by data(); it defaults to ObjectBase so untyped usage ObjectBasePtr<> still works.

Template Parameters
TThe ObjectBase-derived type being tracked.

Class template argument deduction is supported via a deduction guide, so ObjectBasePtr{obj} deduces ObjectBasePtr<Foo> from a Foo*.

An ObjectBasePtr<Derived> converts implicitly to ObjectBasePtr<Base> where Derived inherits from Base, mirroring raw pointer conversions.

Note
Thread safety: The internal pointer is stored as a std::atomic, and the ObjectBase pointer map is protected by a Mutex. This allows an ObjectBasePtr to be safely invalidated from a different thread than the one holding it, as happens during cross-thread object destruction. However, the ObjectBasePtr itself is not designed for concurrent read/write from multiple threads without external synchronization.
