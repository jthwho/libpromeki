Detailed Description

Uniform Resource Locator / Identifier.

Parses and builds URIs per the common subset of RFC 3986. Two serialized forms round-trip through this class:

  • Authority form: scheme://[userinfo@]host[:port][/path][?query][fragment]
  • Opaque form: scheme:opaque[?query][fragment]

The query component is exposed as an ordered Map of key → value pairs (percent-decoded). The raw query string is not preserved across a round-trip when the caller edits individual query values; parsing is destructive to ordering within duplicate keys, which matches how virtually every HTTP application actually uses query strings.

Case sensitivity
Per RFC 3986:
  • Scheme is case-insensitive; fromString lowercases it so callers always see the canonical form ("pmfb", never "PMFB").
  • Host is case-insensitive in the spec but preserved verbatim here because libpromeki schemes like pmfb:// use the host field as a logical name where case may matter to the backend.
  • Path, query, and fragment are case-sensitive — both keys and values in query() are preserved exactly as written. Consumers that match query keys against an identifier registry (see MediaIO::applyQueryToConfig) therefore require an exact-case match; this is a deliberate forgiveness tradeoff that prevents near-miss spellings from resolving to the wrong key.
Example
Url u = Url::fromString("pmfb://studio-a?FrameBridgeRingDepth=4");
u.scheme(); // "pmfb"
u.host(); // "studio-a"
u.queryValue("FrameBridgeRingDepth"); // "4"
u.queryValue("framebridgeringdepth"); // "" (case-sensitive miss)
String out = u.toString(); // round-trip

Whether the serialized form has an authority component.

True for URIs that start with scheme://, false for opaque URIs like foo@e.nosp@m.xamp.nosp@m.le.co.nosp@m.m. Set automatically by fromString; call setHasAuthority to flip between forms when building a URL by hand.

Percent-encodes a string per RFC 3986.

Unreserved characters (ALPHA / DIGIT / - / . / _ / ~) are passed through; every other byte of the UTF-8 encoding is emitted as XX. Characters listed in safe are additionally passed through — callers pass context-specific safe sets (e.g. "/" for paths, ":@/" for query values that may embed colons).
