Detailed Description

Lightweight sub-bucketed log2 histogram for hot-path instrumentation.

Designed to capture latency / interval / size distributions on the fly without per-sample allocation, system calls, or locking. Each sample is binned into a power-of-two octave plus one of SubBucketsPerOctave linear sub-buckets within the octave, so percentiles can be approximated to within roughly 100% / SubBucketsPerOctave relative error — about 6% with the default of 16 sub-buckets per octave. That precision is enough to spot pacing wobble at the 5–10% level (the typical regime where RTP receivers start dropping or hitching) while keeping the memory cost cheap (1024 buckets = 8 KiB per histogram).

Running min, max, count, and sum are tracked separately so the exact mean is also available without a second pass over the buckets.

Threading

Single-writer, multiple-reader. addSample is intentionally not internally synchronised — the intended usage is "one writer thread per Histogram instance" (e.g. one Histogram per RTP stream owned by its TX or RX worker thread). Readers calling toString or percentile see a consistent enough snapshot for diagnostics: small races on the bucket counts manifest as off-by-one errors that are well below the noise floor for the intended use. Wrap externally if true multi-writer semantics are needed.

Bucket layout

The address space is laid out as OctaveCount octaves of SubBucketsPerOctave linear sub-buckets each. Octave O covers values in the half-open range [2^O, 2^(O+1)) and subdivides that range into SubBucketsPerOctave equal-width sub-buckets, so the bucket width within an octave is 2^O / SubBucketsPerOctave. This is the same scheme HDR Histogram uses, just at a fixed precision. 64 octaves cover 0 through 2^64-1, which is around 580 years in nanoseconds — more dynamic range than any practical timing or size measurement needs. Percentile estimation walks the cumulative bucket counts and returns the midpoint of the matched sub-bucket. The result is then clamped to [min, max] so it never reports a value outside the observed range.

Example
Histogram h;
h.setName("rtp-tx-frame-interval");
h.setUnit("us");
for (int i = 0; i < 1000; ++i) {
TimeStamp t0 = TimeStamp::now();
doWork();
h.addSample((TimeStamp::now() - t0).microseconds());
}
promekiInfo("%s", h.toString().cstr());

int64_t Histogram::percentile ( double  p) const

Approximates a percentile via bucket walk.

Walks the cumulative bucket counts and returns the midpoint of the bucket containing the requested percentile. Maximum error is one bucket — i.e. the returned value is within a factor of two of the true percentile, regardless of dataset size.
