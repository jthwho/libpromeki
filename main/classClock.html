Detailed Description

Abstract clock interface.

A Clock provides a time source (nowNs) and a blocking wait primitive (sleepUntilNs), paired with enough self-description for callers to reason about its accuracy:

  • domain returns the ClockDomain that owns this clock. Timestamps from two clocks in the same domain are comparable.
  • resolutionNs tells callers the smallest meaningful time step the clock can distinguish.
  • jitter returns an asymmetric error envelope callers should assume when filtering clock-derived signals.
  • rateRatio exposes the clock's measured rate relative to its nominal rate. 1.0 for locked clocks; audio clocks report their actual drain rate relative to the rate the stream was configured for. This is the primary drift-correction signal for media-rate converters built on top of the clock.

All times are in nanoseconds from the clock's own epoch. The epoch is opaque to callers — only deltas between nowNs readings and deadline values matter.

Implementations
  • WallClock — std::chrono::steady_clock, for local playback.
  • SyntheticClock — frame-count driven, for offline / test / "pristine rate" pipelines.
  • SDLAudioClock — derives time from an SDL audio device's consumed-byte counter.
  • PTP / hardware clocks (future).

◆ rateRatio()

virtual double Clock::rateRatio ( ) const
inlinevirtual

Ratio of actual tick rate to nominal tick rate.

For clocks that are their own nominal reference (WallClock, PTP, SyntheticClock) the ratio is 1.0. For clocks derived from an external rate source (audio device drain, a remote stream's packet rate), the ratio reports the actual rate measured against that reference, which callers can use as a drift-correction signal (e.g. feeding an audio resampler ratio).

The default implementation returns 1.0.

Reimplemented in SDLAudioClock, and SyntheticClock.

virtual int64_t Clock::resolutionNs ( ) const
pure virtual

Smallest meaningful time step in nanoseconds.

The resolution tells callers what precision to expect from nowNs. A 48 kHz audio clock has ~20833 ns resolution (one sample period); a steady_clock wall clock is near 1 ns; a PTP hardware clock is also ~1 ns.

Returns
Resolution in nanoseconds (>= 1).

Implemented in WallClock, SDLAudioClock, and SyntheticClock.

◆ sleepUntilNs()

virtual void Clock::sleepUntilNs ( int64_t  targetNs)
pure virtual

Blocks until the clock reaches targetNs.

Implementation determines the waiting mechanism: an OS sleep for wall clocks, polling a queue depth for audio clocks, waiting on a hardware timer for PTP.

If targetNs is in the past, the call returns immediately. A SyntheticClock implements this as a no-op — its notion of "now" is driven by explicit advance calls, not by sleeping.

Parameters
targetNsTarget time in nanoseconds from the clock's epoch.

Implemented in WallClock, SDLAudioClock, and SyntheticClock.
