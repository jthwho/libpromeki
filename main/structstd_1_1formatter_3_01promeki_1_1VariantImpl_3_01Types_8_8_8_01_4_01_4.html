Detailed Description

template<typename... Types>
struct std::formatter< promeki::VariantImpl< Types... > >

std::formatter partial specialization for promeki::VariantImpl.

Routes the held value through VariantImpl::get<String>(), which already knows how to convert every variant alternative to a String (numbers via String::number, library types via their toString, collections like StringList via join, etc.). This means promeki::Variant — and any other VariantImpl instantiation — is usable as a promeki::String::format argument out of the box, regardless of what type it currently holds.

Standard string format specifiers (width, fill, alignment) are inherited from std::formatter<std::string_view>.

Variant v;
v.set(42); // holds int
String s = String::format("v = {}", v); // "v = 42"
v.set(promeki::UUID::generate()); // now holds UUID
s = String::format("v = {}", v); // "v = <uuid>"
