Detailed Description

MediaIOTask writer that plays frames through SDL.

SDLPlayerTask is the MediaIO analog of SDLDisplayNode: it is a write-only sink backend that consumes Frame objects via MediaIO::writeFrame() and displays their video through an application-provided SDLVideoWidget and (optionally) plays their audio through an application-provided SDLAudioOutput.

Creation

Unlike registered backends, SDLPlayerTask cannot be created via MediaIO::create() because it needs raw pointers to the video widget and audio output — values that can't be carried in a MediaConfig. Use the free factory function createSDLPlayer() to construct a ready-to-use MediaIO that adopts a properly configured SDLPlayerTask.

Pacing

The task has two playback modes selected via the paced constructor argument:

  • Paced (default): the task enforces the source's frame rate. When the frame contains audio and the caller supplied an SDLAudioOutput, the audio stream is used as the playback clock — each writeFrame() pushes the frame's audio to the output and then blocks until the queued audio drains below a one-frame threshold. If there is no audio, the task falls back to a wall-clock sleep based on the configured frame rate.
  • Fast ("as fast as possible"): audio is dropped entirely (the SDLAudioOutput is never opened), no pacing is applied, and the task simply hands each frame to the main thread for rendering. Because only the most recently stashed image is painted, intermediate frames are naturally dropped when the main thread cannot keep up.

In both modes the video widget always sees only the most recent image — the render callback posted to the main thread picks up whatever image is currently stashed, so any older image still waiting when a newer one arrives is counted as a dropped frame.

Compressed input — automatic decode

SDLPlayerTask accepts compressed video formats (JPEG, JPEG XS, and anything else with a registered ImageCodec) as input and decodes them transparently before the frame is handed to the widget. The decode runs inside executeCmd(Write) on the MediaIO strand worker thread via Image::convert, which dispatches to the codec registry by name — the task does not hard-code any codec. The decode target is PixelDesc::RGBA8_sRGB so the decoded image can be handed to SDLVideoWidget directly, regardless of which codec was involved.

Running the decode on the strand thread (rather than the main thread) keeps the UI responsive for heavy codecs: the main thread only ever sees an already-decoded image ready for texture upload. This means a receiver like mediaplay -i foo.sdp -o SDL just works for any format the codec layer can decode, with no Converter stage needed in front of the SDL sink.

If a decode fails (missing codec, malformed bitstream) the offending frame is counted as dropped and the task continues processing subsequent frames.

Uncompressed input with a pixel layout SDL cannot upload directly (e.g. 10-bit planar YUV) is handled by the widget's own CSC fallback — the task does not pre-convert those.

Open-time setup

For a writer, MediaIO delivers the caller-provided MediaDesc and (optionally) AudioDesc to the task in MediaIOCommandOpen. SDLPlayerTask reads the frame rate from the MediaDesc and, in paced mode, configures and opens the SDLAudioOutput using either the explicit pendingAudioDesc or the first entry in pendingMediaDesc.audioList(). Callers must therefore set the media description before opening:

MediaIO *player = createSDLPlayer(&videoWidget, &audioOutput);
player->setMediaDesc(sourceMediaIO->mediaDesc());
player->setAudioDesc(sourceMediaIO->audioDesc());
player->open(MediaIO::Input);

Teardown ordering

The main thread's render callable captures a pointer to the task. The caller must therefore close() the MediaIO and let the main event loop drain (for example by quitting the SDLApplication before delete) before destroying the owning MediaIO. Destroying the MediaIO while a render callable is still queued on the main thread's event loop is undefined behavior — mirroring the same constraint that applies to SDLDisplayNode.
