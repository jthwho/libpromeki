Detailed Description

A media frame containing images, audio, compressed packets, and metadata.

Aggregates one or more uncompressed image planes, one or more audio tracks, one or more compressed packets, and a metadata container into a single unit that represents a frame of media content.

The packetList companion to imageList / audioList holds encoded bitstream access units — the output of a VideoEncoder, the input to a VideoDecoder, or the on-disk samples read from a muxed container. A single Frame may carry uncompressed essence (images / audio) and compressed packets simultaneously — for example, while a pipeline is mid-encode the encoder task's output can be spliced back onto the same Frame that carried the source image for that PTS.

Example
MediaDesc mdesc;
Frame::Ptr frame = Frame::Ptr::create(mdesc);
frame->setTimecode(Timecode(Timecode::NDF24, 1, 0, 0, 0));
Image img = frame->image(0);
// After encoding, attach a compressed packet alongside the image:
frame.modify()->packetList().pushToBack(encoded);

Substitutes {Key[:spec]} placeholders against this frame.

Built on top of Metadata::format with a custom resolver that adds three layers of introspection on top of the bare metadata lookup:

  1. Frame-level pseudo keys (prefixed with "@" so they cannot collide with metadata keys):
    • @ImageCount, @AudioCount, @PacketCount — list sizes (uint64).
    • @HasBenchmark — bool.
    • @VideoFormat / @VideoFormat[N] — VideoFormat for image N (default 0). Combines the frame's Metadata::FrameRate with the image's raster and scan mode and honours VideoFormat format specs (e.g. :smpte). Out-of-range N falls through to the user resolver / unknown-key path.
  2. Subscripted descent into images and audio — any placeholder of the form "Image[N].<sub>" or "Audio[N].<sub>" is dispatched to the corresponding Image::resolveTemplateKey or Audio::resolveTemplateKey, so the full vocabulary of those classes (their pseudo keys and their own metadata) is available with no extra plumbing.
  3. The user-supplied resolver is consulted last for anything else.
Example
Frame::Ptr frame = Frame::Ptr::create();
frame.modify()->imageList().pushToBack(Image::Ptr::create(1920, 1080,
PixelDesc::RGBA8_sRGB));
frame.modify()->metadata().set(Metadata::Timecode,
Timecode(Timecode::NDF24, 1, 0, 0, 0));
String s = frame->makeString(
"[{Timecode:smpte}] {@ImageCount}img {Image[0].@Size}");
// "[01:00:00:00] 1img 1920x1080"

◆ mediaDesc()

MediaDesc Frame::mediaDesc ( ) const

Assembles a MediaDesc describing this Frame.

Returns a MediaDesc populated from the frame's own state: the frame rate is taken from Metadata::FrameRate (if set), the image list is built from the Image::desc of each Image in imageList, the audio list is built from the Audio::desc of each Audio in audioList, and the frame's metadata is copied across.

The returned MediaDesc is only valid if both a frame rate is present in metadata and the frame carries at least one image or audio entry.

std::optional< String > Frame::resolveTemplateKey ( const String &  key,
const String &  spec 
) const

Resolves a single template key against this frame's structure.

Used by makeString. Returns a value when the key names something this frame can describe and std::nullopt otherwise. The recognised forms mirror makeString:

  • "@<Pseudo>" — frame-level introspection (see makeString for the list).
  • "Image[N].<sub>" — delegates to Image::resolveTemplateKey on the image at index N. When N is out of range or the image is null, returns std::nullopt.
  • "Audio[N].<sub>" — delegates to Audio::resolveTemplateKey on the audio at index N.
  • Any registered metadata key — looked up in the frame's metadata and rendered with Variant::format.
Parameters
keyThe placeholder key (no braces, no colon).
specThe format spec (may be empty).
