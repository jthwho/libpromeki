Detailed Description

A media frame containing images, audio, and metadata.

Aggregates one or more Image entries (uncompressed or compressed), one or more Audio tracks, and a metadata container into a single unit that represents a frame of media content.

Compressed bitstream access units are not stored as a separate list on the Frame — they travel with their owning essence via Image::packet and Audio::packet. A compressed Image carries its encoded MediaPacket directly; that's the canonical location a downstream VideoDecoder reads from, and the canonical location a VideoEncoder writes to.

◆ configUpdate()

const MediaConfig & Frame::configUpdate ( ) const
inline

Returns a const reference to the config update delta.

When non-empty, the config update is applied by the receiving MediaIOTask before the frame is processed. This gives frame-level synchronisation for dynamic parameter changes (bitrate, quality, etc.) — the update travels with the write command through the strand queue, so a flush can never separate a config change from its frame.

◆ dump()

StringList Frame::dump ( const String &  indent = String()) const

Returns a human-readable multi-line dump of this frame's full contents.

Emits the scalar-key block registered with VariantLookup<Frame> (ImageCount, AudioCount, HasBenchmark), the frame's metadata via Metadata::dump, the configUpdate delta when non-empty, and then Image::dump / Audio::dump for every entry indented by two spaces.

No leading "Frame:" header is emitted — callers that need one (e.g. pmdf-inspect printing a frame index) prepend their own title and an additional indent so the output is uniform regardless of whether one frame or many are being printed.

indentLeading whitespace to prefix every line.
Returns
A StringList (one entry per line).

◆ mediaDesc()

MediaDesc Frame::mediaDesc ( ) const

Assembles a MediaDesc describing this Frame.

Returns a MediaDesc populated from the frame's own state: the frame rate is taken from Metadata::FrameRate (if set), the image list is built from the Image::desc of each Image in imageList, the audio list is built from the Audio::desc of each Audio in audioList, and the frame's metadata is copied across.

The returned MediaDesc is only valid if both a frame rate is present in metadata and the frame carries at least one image or audio entry.

Composes the image's raster and scan mode with the frame rate carried in metadata under Metadata::FrameRate. Returns a default (invalid) VideoFormat when index is out of range or when the frame rate metadata is missing or not convertible to a FrameRate.
