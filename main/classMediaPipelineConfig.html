Detailed Description

Declarative description of a complete media pipeline.

A MediaPipelineConfig is the shareable data object that MediaPipeline consumes at build() time. It lists the MediaIO instances (stages) to instantiate, the frameReady → writeFrame edges between them (routes), and pipeline-wide Metadata.

Two serialization paths round-trip the same in-memory shape:

  • toJson / fromJson — stable, human-editable form used by preset files and GUI editors.
  • operator<< / operator>> on DataStream — compact binary form suitable for IPC and saved state.

validate performs both structural checks (route endpoints exist, no cycles, no orphans) and per-stage VariantSpec checks against the backend's registered config specs.

◆ validate()

Error MediaPipelineConfig::validate ( ) const

Validates the config for use by MediaPipeline::build.

Checks performed:

  • Every stage has a non-empty Stage::name and the names are unique.
  • Every stage has a non-empty Stage::type or Stage::path.
  • Every stage has a valid Stage::mode.
  • Every route's from and to reference existing stage names.
  • The route graph is acyclic.
  • Every stage participates in at least one route (except single-stage configs).
  • Each stage's MediaConfig keys are recognized (via MediaIO::unknownConfigKeys) and each set value passes its registered VariantSpec.

Findings are logged at warning level; the return value is Error::Invalid or Error::InvalidArgument on the first structural problem encountered, or Error::Ok if the config is well-formed.

Returns
Error::Ok when the config is valid.
