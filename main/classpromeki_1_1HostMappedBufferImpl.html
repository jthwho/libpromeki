Detailed Description

Abstract intermediate base for backends backed by host memory.

Concrete subclasses (HostBufferImpl, HostSecureBufferImpl, WrappedHostBufferImpl, CudaHostBufferImpl) supply the alloc / release strategy in their constructor / destructor; everything else — host pointer accessors, mapAcquire(Host) refcount bump, fill, copyFromHost — is provided here.

Implicit Host mapping
Backends that derive from this class are host-resident for their entire lifetime. The constructor seeds the MemDomain::Host refcount to 1 so existing call sites that simply dereference data() without an explicit mapAcquire keep working. Additional mapAcquire(Host) calls increment the count; mapRelease(Host) decrements. The host backing is owned by the subclass (lifetime tied to the BufferImpl instance) so the refcount has no teardown effect — it is bookkeeping only.
Non-host domains
mapAcquire / mapRelease for any domain other than Host resolve with Error::NotSupported by default. Subclasses that support additional domains (e.g. CudaHostBufferImpl exposing the pinned region for direct CUDA-device DMA) override the relevant branch.

Increments the per-domain mapping refcount and ensures the buffer is accessible to domain.

Backends that already hold an open mapping to domain may resolve the returned request inline (via BufferRequest::resolved); backends that need to stage a copy or perform a DMA dispatch the work to a worker / strand and return the in-flight handle.

Backends that cannot map to domain return BufferRequest::resolved with Error::NotSupported.

Implements promeki::BufferImpl.

Decrements the per-domain mapping refcount.

When the count for domain reaches zero the backend performs any required write-back (when the acquire was Write or ReadWrite) and tears down the mapping. Calling release on a domain that was never acquired returns Error::Invalid.
