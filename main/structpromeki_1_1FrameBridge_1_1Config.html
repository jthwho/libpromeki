Output-side configuration.

Only openOutput consumes this struct. Input-side openers learn the effective config from the shm header and the ACPT handshake.

bool promeki::FrameBridge::Config::waitForConsumer = true

Block writeFrame until a consumer is connected.

When true (the default) the first writeFrame call after opening (and any later call made while no consumers are attached) blocks, servicing accepts and pruning dead peers, until at least one consumer is connected. This matches the common "producer stalls until downstream is ready" pattern and prevents silently dropping frames before any consumer connects. close aborts the wait and causes writeFrame to return Error::NotOpen.

Set to false when the publisher is the source of truth for pacing (for example a live capture that must run whether anyone is listening or not); writeFrame then returns immediately and becomes a silent no-op when no consumers are attached.
