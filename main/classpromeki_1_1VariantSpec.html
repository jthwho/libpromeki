Detailed Description

Describes the constraints and default value for a VariantDatabase ID.

VariantSpec captures the accepted type(s), default value, numeric range, enum type, and human-readable description for a single VariantDatabase key. Every ID declared via VariantDatabase::declareID is associated with a VariantSpec, enabling:

  • Introspection — callers can enumerate all keys a backend supports along with their types, ranges, and descriptions.
  • Validation — VariantDatabase::set can optionally warn or reject values that violate the spec.
  • Self-documenting defaults — the spec carries the default value, eliminating the need for separate default-config factories.

Specs are built using a fluent builder pattern:

Example
VariantSpec spec = VariantSpec()
.setType(Variant::TypeS32)
.setDefault(85)
.setRange(1, 100)
.setDescription("JPEG quality 1-100");
bool ok = spec.validate(Variant(50)); // true
bool bad = spec.validate(Variant(200)); // false (out of range)
Variant def = spec; // Variant(85) via operator Variant()

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance — including any combination of the fluent setters with validate() / defaultValue() — must be externally synchronized. In typical usage a spec is built once at registration time and read freely thereafter.

◆ coerce()

Variant promeki::VariantSpec::coerce ( const Variant &  val,
Error *  err = nullptr 
) const

Recursively coerces val to the spec's native shape.

Designed for the JSON deserialisation path where every leaf arrives as a JSON primitive (often String) but the spec wants the rich native type (Color, Size2D, Enum, etc.). Walks the value:

  • TypeString and the spec doesn't accept String natively → parseString to the spec's native type.
  • TypeVariantList with an elementSpec set → recurse into every element.
  • TypeVariantMap with a valueSpec set → recurse into every value.
  • Anything else → returned as-is.

Returns the coerced Variant on success. On failure (parseString rejected a string, recursive coercion failed) returns an invalid Variant and sets err to the underlying parse / conversion code.

Parameters
valThe value to coerce.
errOptional error output.
Returns
The coerced value, or invalid on failure.

◆ parseString()

Variant promeki::VariantSpec::parseString ( const String &  str,
Error *  err = nullptr 
) const

Parses a CLI-style string into a Variant of the correct type.

The spec's type list drives the parser — no template value is needed. For Enum types the spec's enumType() resolves value names. For polymorphic specs the types are tried in declaration order and the first successful parse wins.

Supported types: bool (true/false/yes/no/1/0), all integer and float widths, String, Size2Du32, FrameRate, Rational, Timecode, DateTime, Color, PixelFormat, PixelMemLayout, ColorModel, Enum, StringList, and SocketAddress (when network is enabled).

Parameters
strThe string to parse.
errOptional error output.
Returns
The parsed Variant, or an invalid Variant on failure.

◆ validate()

bool promeki::VariantSpec::validate ( const Variant &  value,
Error *  err = nullptr 
) const

Validates a value against this spec.

Checks performed in order:

  1. Type — if types() is non-empty, the value's type must be in the list.
  2. Range — if min/max are set and the value is numeric, it must fall within bounds.
  3. Enum type — if an Enum::Type is set and the value is an Enum, its type must match.
Parameters
valueThe value to validate.
errOptional error output:
  • Error::InvalidArgument if the type check fails.
  • Error::OutOfRange if the range check fails.
Returns
True if the value passes all checks.
