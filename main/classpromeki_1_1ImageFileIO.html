Detailed Description

Abstract backend for image file format I/O.

Subclasses implement format-specific loading and saving of images (e.g. PNG). Backends register themselves via PROMEKI_REGISTER_IMAGEFILEIO and are looked up by their format ID.

String promeki::ImageFileIO::mediaIoName ( ) const

Returns the preferred backend name for MediaIO registration.

The MediaIO registry exposes one MediaIOFactory per ImageFileIO backend; each backend carries the string used to identify it there ("ImgSeqDPX", "ImgSeqPNG", "ImgSeqCineon", ...). Defaults to "ImgSeq" + name() when the backend's constructor doesn't override it.

◆ registeredIDs()

static IDList promeki::ImageFileIO::registeredIDs ( )
static

Returns every registered backend ID in registration order.

Used by callers that need to enumerate all known image formats — e.g. the MediaIO layer wires up one MediaIOFactory per backend ID. Iterates the live registry, so calling this before every backend has registered (mid-static-init) returns a partial list; normal runtime use sees the complete set.

Returns
A list of all registered backend IDs.
