Detailed Description

Declarative description of a complete media pipeline.

A MediaPipelineConfig is the shareable data object that MediaPipeline consumes at build() time. It lists the MediaIO instances (stages) to instantiate, the frameReady → writeFrame edges between them (routes), and pipeline-wide Metadata.

Two serialization paths round-trip the same in-memory shape:

  • toJson / fromJson — stable, human-editable form used by preset files and GUI editors.
  • operator<< / operator>> on DataStream — compact binary form suitable for IPC and saved state.

validate performs both structural checks (route endpoints exist, no cycles, no orphans) and per-stage VariantSpec checks against the backend's registered config specs.

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized. MediaPipelineConfig::Ptr uses an atomic refcount and is safe to share across threads.

Per-stage role classification.

Replaces the old MediaIO::Mode that was removed when MediaIO was split into MediaIOSource / MediaIOSink / MediaIOPortGroup. The role still has to round-trip through JSON / DataStream because file-only stages (path-only with no registered backend) need to know whether to be opened for read or for write.

◆ frameCount()

const FrameCount & promeki::MediaPipelineConfig::frameCount ( ) const
inline

Returns the pipeline-wide target frame count.

A finite, non-empty FrameCount tells MediaPipeline to close each sink once it has received the requested number of frames; subsequent source reads are dropped on the floor. Any non-finite value (the default FrameCount::unknown, FrameCount::empty, or FrameCount::infinity) means "unlimited" — the pipeline runs until the sources naturally hit EOS.

For interframe-coded streams (VideoCodec::CodingTemporal) the cutoff respects GOP boundaries: once the target has been reached, the pipeline keeps writing until the next frame carries a MediaPayload::Keyframe flag, then stops before writing that keyframe. The sink therefore always receives a complete sequence of GOPs, even if that means overshooting the target by up to one GOP.

◆ statsWindowSize()

int promeki::MediaPipelineConfig::statsWindowSize ( ) const
inline

Returns the per-stage WindowedStat ring capacity.

Propagated to each stage's MediaConfig::StatsWindowSize during MediaPipeline::build / MediaPipeline::injectStage, so a single pipeline-wide knob controls the size of every MediaIO's per-command telemetry window. Stages that explicitly set MediaConfig::StatsWindowSize override the pipeline default. A value of zero disables tracking.

◆ validate()

Error promeki::MediaPipelineConfig::validate ( ) const

Validates the config for use by MediaPipeline::build.

Checks performed:

  • Every stage has a non-empty Stage::name and the names are unique.
  • Every stage has a non-empty Stage::type or Stage::path.
  • Every stage has a valid Stage::role.
  • Every route's from and to reference existing stage names.
  • The route graph is acyclic.
  • Every stage participates in at least one route (except single-stage configs).
  • Each stage's MediaConfig keys are recognized (via MediaIOFactory::unknownConfigKeys) and each set value passes its registered VariantSpec.

Findings are logged at warning level; the return value is Error::Invalid or Error::InvalidArgument on the first structural problem encountered, or Error::Ok if the config is well-formed.

Returns
Error::Ok when the config is valid.
