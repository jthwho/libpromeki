Detailed Description

template<CompiledString Name>
class promeki::StringRegistryItem< Name >

Lightweight handle identifying a string registered in StringRegistry.

Item wraps a uint64_t ID and provides access to the underlying string via the owning registry. Items are cheap to copy and can be compared by integer value.

Hoisted out of StringRegistry so that std::hash<StringRegistryItem<Name>> can be partially specialized — a hash specialization on a nested type (StringRegistry<Name>::Item) is a non-deduced context and would never be selected. StringRegistry<Name>::Item is preserved as a type alias, so existing code that spells the type as a member of the registry keeps working.

template<CompiledString Name>
static constexpr StringRegistryItem promeki::StringRegistryItem< Name >::literal ( const char *  name)
inlinestaticconstexpr

Creates a compile-time Item from a name literal.

Returns the pure FNV-1a hash of name without touching the runtime registry. Use this for well-known IDs that need to participate in constexpr machinery (switch labels, static_assert, template parameters). The resulting ID will match any Item later constructed from the same name via VariantDatabase::declareID, provided no hash collision was flagged at static-init time.

Note that literal() does not register the name for reverse lookup; name will return an empty string unless the same name was also registered via a runtime path.
