Detailed Description

template<typename K, typename V>
class promeki::Map< K, V >

Ordered associative container wrapping std::map.

Provides a Qt-inspired API over std::map with consistent naming conventions matching the rest of libpromeki.

Thread Safety
Conditionally thread-safe. Distinct instances may be used concurrently; concurrent access to a single instance must be externally synchronized.

Inserts or assigns a key-value pair.

Note
Unlike std::map::insert (which is no-op when the key is already present), this calls insert_or_assign — an existing entry is overwritten. This matches Qt's QMap::insert semantics and is almost always what callers want; use insertNew when no-overwrite is required.

Inserts a key-value pair only if key is absent.

Matches std::map::insert semantics: when the key is already present the existing value is left unchanged and the call returns false. Use this when overwrite is a logic error you want to detect rather than tolerate.
