Detailed Description

Uncompressed video payload — raster pixels in the format described by the base VideoPayload's ImageDesc.

The plane list on the base carries one BufferView per plane defined by the descriptor's PixelFormat — one entry for packed / interleaved formats (RGBA, YUYV), multiple entries for planar formats (Y/U/V), and multiple-views-on-one-buffer when a planar payload was read in a single allocation. The number of entries should match desc().planeCount, but that invariant is left to callers rather than being enforced by the payload itself so partial construction (create descriptor, attach planes later) stays ergonomic.

Every uncompressed payload is trivially keyframe-able — each frame decodes on its own — so isKeyframe is overridden to always return true regardless of whether the Keyframe flag has been set manually.

UncompressedVideoPayload is intentionally not final — a future CUDA- or GL-backed variant may extend it to tag its planes with device-memory provenance, for example.

Example — allocating from a descriptor
ImageDesc desc(1920, 1080, PixelFormat::RGBA8_sRGB);
auto buf = Buffer::Ptr::create(desc.pixelFormat().planeSize(0, desc));
BufferView plane0(buf, 0, buf->size());
auto payload = UncompressedVideoPayload::Ptr::create(
desc, plane0);

◆ allocate()

static Ptr promeki::UncompressedVideoPayload::allocate ( const ImageDesc &  desc)
static

Allocates a fresh payload with backing buffers sized for desc.

Convenience helper for producers that want to fill bytes directly into plane buffers they own. One Buffer is allocated per plane, sized to the descriptor's per-plane byte count (line pitch x height for interleaved and each plane of planar formats). Each plane's BufferView covers its buffer in full so downstream consumers see the whole-plane invariant.

Returns a null Ptr if the descriptor's pixel format or size is invalid.

◆ createPaintEngine()

PaintEngine promeki::UncompressedVideoPayload::createPaintEngine ( ) const

Creates a PaintEngine targeting this payload's pixel buffer.

Payload-native paint engine factory. The returned engine writes directly to the payload's plane-0 Buffer when its view covers the buffer in full (the common case), so paint operations are visible through any UncompressedVideoPayload sharing the same backing buffer.

Callers that need exclusive ownership of the pixel buffer (CoW-detach before paint) should duplicate the payload first — there is no ensureExclusive on UncompressedVideoPayload yet; that lands when the compositor stages flip onto the payload-native paint path.
