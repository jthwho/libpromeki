Detailed Description

Network sockets, address types, and AV-over-IP building blocks.

Raw POSIX socket layer (AbstractSocket, TcpSocket, TcpServer, UdpSocket, RawSocket), IPv4/IPv6/MAC address types, SocketAddress, RTP session and payload handling (L24, L16, RawVideo, JPEG), SDP session description, PrioritySocket for QoS, and MulticastManager. NetworkAddress provides a polymorphic wrapper over all address types and unresolved hostnames.

◆ HttpHandlerFunc

using promeki::HttpHandlerFunc = typedef std::function<void(const HttpRequest &request, HttpResponse &response)>

Lambda-style HTTP request handler.

Signature mirrors Go's http.HandlerFunc: the handler receives the request by const reference and writes its reply into the supplied HttpResponse, which the connection then serializes to the wire.

Use this directly with HttpRouter::route and HttpServer::route for the common case where a tiny lambda is the entire handler. When a handler needs explicit state, derive from HttpHandler instead.

◆ HttpMiddleware

using promeki::HttpMiddleware = typedef std::function<void(const HttpRequest &request, HttpResponse &response, std::function<void()> next)>

Middleware that wraps the next handler in the chain.

Middleware runs before the matched handler, in registration order. A middleware that wants the wrapped handler to run calls next(); a middleware that wants to short-circuit (e.g. an auth check that sets a 401 response) simply returns without calling next.

Example
server.use([](const HttpRequest &req, HttpResponse &res, auto next) {
if(!isAuthenticated(req)) {
res.setStatus(HttpStatus::Unauthorized);
return;
}
next();
});
