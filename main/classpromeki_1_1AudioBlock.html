Detailed Description

Base class for an audio processing block.

Defines the interface for an audio processing graph node. Each AudioBlock exposes a fixed number of named source and sink channels (counts supplied via Config) and is intended to be composed with other blocks into a processing chain.

Relationship to AudioBuffer

AudioBlock and AudioBuffer occupy different layers and are not interchangeable. AudioBlock is a pipeline node (an ObjectBase with sources/sinks, signals, and graph identity); AudioBuffer is a data container (a thread-safe PCM sample FIFO with format conversion). An AudioBlock typically uses one or more AudioBuffer instances internally to bridge its producer and consumer threads, but the two classes should not be unified.

Thread Safety
Thread-affine via ObjectBase. An AudioBlock instance must be used on the thread that created it; cross-thread interaction goes through ObjectBase signal/slot dispatch.
