Detailed Description

Describes a media format including frame rate, image layers, audio channels, and metadata.

MediaDesc is the complete descriptor for a media resource. It carries the frame rate, zero or more image layers (ImageDesc), zero or more audio channel groups (AudioDesc), and container-level metadata. A valid MediaDesc has a valid frame rate and at least one image or audio description.

static MediaDesc MediaDesc::fromSdp ( const SdpSession &  session)
static

Aggregates the per-m= descriptors from an SDP session.

Walks every media description in session, calls ImageDesc::fromSdp or AudioDesc::fromSdp on each one (based on its mediaType), and pushes any successful result into the corresponding list on the returned MediaDesc. The frame rate is left unset — SDP does not carry a frame rate attribute, so callers that need one must fill it in separately.

Media descriptions whose encoding name is not recognised by ImageDesc::fromSdp / AudioDesc::fromSdp are silently skipped, so the returned MediaDesc may be partial (or empty) for streams that use encodings libpromeki does not yet know about.

SdpSession MediaDesc::toSdp ( uint8_t  videoPayloadType = 96) const

Builds an SdpSession from this MediaDesc.

The inverse of fromSdp. Walks the image and audio lists and calls ImageDesc::toSdp / AudioDesc::toSdp on each entry, appending the resulting media descriptions to a new SdpSession.

Video payload types start at videoPayloadType (default 96) and increment for each image entry. Audio payload types start at audioPayloadType (default one past the last video PT).
