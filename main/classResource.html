Detailed Description

Compiled-in resource filesystem (Qt-style ":/...").

Resources are baked into the binary at build time via the cirf code generator and accessed at runtime through this class. Lookup is done by virtual path; the same code paths in File, FileIODevice and Dir transparently serve resources whenever they are given a path beginning with ":/" — so most consumers do not need to call into this class directly.

Path scheme
Resource paths follow Qt's QResource convention: a leading ":/" marker followed by the virtual path of the resource within the registered cirf root. For example, libpromeki's built-in FiraCode font is reachable at
":/.PROMEKI/fonts/FiraCodeNerdFontMono-Regular.ttf"

The .PROMEKI prefix marks built-in libpromeki resources as a hidden directory and avoids collisions with paths that consuming applications choose for their own resource sets.

Multiple resource sets
Each call to cirf_add_resources() in CMake produces an independent cirf_folder_t tree. Use registerRoot() to mount additional roots from a consuming application; use the PROMEKI_REGISTER_RESOURCES macro to mount one automatically at static-init time.
Lifetime and ownership
cirf data lives in the executable's .rodata segment for the lifetime of the program. Pointers returned by findFile() / findFolder() and Buffer views returned by data() are non-owning and never need freeing.

Registers a generated cirf root under an optional prefix.

With an empty prefix the root's contents are merged directly under ":/" — file lookups walk the root with their full virtual paths. With a prefix like "myapp" a file inside the root with virtual path "fonts/foo.ttf" becomes reachable at ":/myapp/fonts/foo.ttf" — the prefix is stripped from the lookup path before walking the root.

Backed by cirf_mount(). The prefix string is copied internally so the caller does not need to keep the original alive. Calling registerRoot() twice with the same prefix and the same root is a no-op; calling it with the same prefix and a different root replaces the previous mount.

Returns
Error::Ok on success, otherwise an error.
