Detailed Description

Abstract base class for stateful video decoders.

A VideoDecoder is the inverse of VideoEncoder: packets submitted via submitPacket feed an internal decode pipeline and uncompressed frames come back out of receiveFrame. The decoder may buffer several packets before producing its first frame (B-frame reordering, reference-frame dependencies).

Codec-family metadata lives on the VideoCodec wrapper returned by codec(); per-backend facts (supported output PixelFormats, bitrate limits, …) live on the registered BackendRecord and are reachable via VideoCodec::decoderSupportedOutputs.

Backend registration

Same idiom as VideoEncoder::registerBackend, but using this class's BackendRecord — see that method's documentation for the full registration example.

Session lifecycle
  1. Resolve a session via VideoCodec::createDecoder.
  2. Optionally call configure (most decoders infer parameters from the bitstream and need no explicit configuration).
  3. For each encoded packet, call submitPacket.
  4. After each submit, drain with receiveFrame until it returns an invalid Image.
  5. Call flush when the input stream ends, then drain again.
  6. Destroy the decoder.

Implementations are not required to be thread-safe.
