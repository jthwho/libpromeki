Detailed Description

Class for holding and manipulating timecode.

While this class supports all the capabilities of SMPTE timecode, it exceeds it in many ways. It makes no restrictions on hour counts above 23, or frame rates above 30. It will, of course, attempt to do the best it can when asked to output a SMPTE timecode (i.e. the hours will be modulo 24).

Example
// Create a 24fps timecode at 01:00:00:00
Timecode tc(Timecode::NDF24, 1, 0, 0, 0);
++tc; // advance to 01:00:00:01
// Convert to string
auto [str, err] = tc.toString(); // "01:00:00:01"
// From absolute frame number
Timecode tc2 = Timecode::fromFrameNumber(Timecode::NDF24, 86400);
// Parse from string
auto [tc3, err2] = Timecode::fromString("10:30:00:00");

Unpacks a 64-bit BCD time-address word into a Timecode.

Inverse of toBcd64. The supplied mode supplies the frame rate; the BCD word's drop-frame flag is resolved against mode according to these rules:

  • If the DF flag is clear, mode is used as-is.
  • If the DF flag is set and mode is invalid / unknown, the result is at VTC_FORMAT_29_97_DF (i.e. an unset DF flag combined with an unknown rate yields the canonical 29.97 DF rate).
  • If the DF flag is set and mode is at a rate that has a drop-frame sister format (29.97 NDF → 29.97 DF, 30 NDF → 30 DF, 59.94 NDF → 59.94 DF), the result is upgraded to that DF sister.
  • If mode is already at the matching DF rate, the result keeps that mode unchanged.
  • If the DF flag is set and mode is at a rate that does not support drop-frame counting (24, 25, integer 24/25-multiple HFR rates, etc.), an Error::ConversionFailed is returned because the BCD word and the requested mode are inconsistent.

The fmt argument selects which standard's bit positions are used to extract the digits and flag bits. The two variants only differ at bit 27 (see TimecodePackFormat); for typical Vitc/Ltc data exchanged through ImageDataEncoder, both produce identical digits.

Parameters
bcdThe 64-bit BCD word.
fmtWhich variant's bit interpretation to use (default: TimecodePackFormat::Vitc).
modeThe timecode mode the result should adopt (subject to the DF rules above).
Returns
A Result holding the unpacked Timecode on success, or an Error on inconsistency.

◆ fromString()

static Result< Timecode > promeki::Timecode::fromString ( const String &  str)
static

Parses a Timecode from its string representation.

Recognised inputs:

  • Standard SMPTE strings such as "01:00:00:00" (NDF) or "01:00:00;00" (DF, semicolon before frames).
  • The canonical invalid-timecode sentinel "--:--:--:--" — returns a default-constructed (invalid) Timecode with Error::Ok, so the round-trip toString() → fromString() is lossless for invalid timecodes.
  • An empty string — treated identically to the invalid sentinel: returns an invalid Timecode with Error::Ok.
  • Any other string that cannot be parsed returns a default-constructed Timecode and a non-ok error code.
Parameters
strThe string to parse (e.g. "01:00:00:00").
Returns
A Result holding the parsed Timecode on success or a default-constructed Timecode and an error code on parse failure.

◆ operator>()

bool promeki::Timecode::operator> ( const Timecode &  other) const
inline

Ordering operators.

Comparison strategy depends on the modes of the two operands:

  • If both sides carry the same Mode, digits alone determine the order. Drop-frame gaps do not affect digit ordering within a single mode (they only collapse frame counts).
  • If either side lacks a libvtc format (i.e. the timecode is invalid, or was parsed / constructed with an unknown frame rate), digits are used because no frame-number conversion is possible.
  • Otherwise the two sides are at different valid rates and we convert each to an absolute frame number using its own rate and compare those.

◆ toBcd64()

uint64_t promeki::Timecode::toBcd64 ( TimecodePackFormat  fmt = TimecodePackFormat::Vitc) const

Packs the timecode into the 64-bit BCD time-address word.

Produces the same set of fields that SMPTE 12M-1 (LTC) and SMPTE 12M-2 (VITC) carry in their respective time-address fields — eight BCD time digits, the binary groups (32 bits of user bits), the drop-frame flag, the color frame flag, and the binary group flags BGF0/BGF1/BGF2 — minus the wire-level framing (sync words, CRC, biphase mark transitions).

The bit positions inside the returned 64-bit word match the bit positions defined for the chosen variant in SMPTE 12M (so a downstream consumer that already knows "LTC bit 11 = color frame flag" can apply that knowledge directly to bit 11 of the returned word). The specification only differs at one bit position, bit 27 — see TimecodePackFormat for the table.

In TimecodePackFormat::Ltc mode this calls libvtc's vtc_ltc_pack and returns the lower 8 bytes of its 80-bit output (the 16-bit sync word at bits 64-79 is intentionally dropped — wire framing is the encoder's job, not this method's). In TimecodePackFormat::Vitc mode this packs directly so bit 27 carries the field marker bit (sourced from isFirstField), per SMPTE 12M-2 / 12-3.

The bit ordering in the returned 64-bit word is the same as in vtc_ltc_pack: bit 0 = LSB of frame units, bit 63 = MSB of user-bit nibble 8. Callers that want to transmit the word over a wire should pick a bit order convention and document it; libpromeki's ImageDataEncoder transmits MSB-first.

Parameters
fmtWhich variant's bit interpretation to use (default: TimecodePackFormat::Vitc).
Returns
The 64-bit BCD time-address word.

◆ toString()

Result< String > promeki::Timecode::toString ( const VtcStringFormat *  fmt = &VTC_STR_FMT_SMPTE) const

Converts the timecode to a string.

Always returns a non-empty, printable string:

  • An invalid timecode (no mode, not constructed with valid digits) returns "--:--:--:--", the canonical invalid-timecode sentinel. fromString recognises this sentinel and round-trips it back to a default-constructed invalid Timecode.
  • A format-less timecode (mode is valid but has no associated VtcFormat, e.g. constructed with fps=0) returns the bare digits "HH:MM:SS:FF" without a frame-rate suffix.
  • A valid timecode with a known format uses libvtc and the supplied fmt to produce the standard SMPTE-style string (e.g. "01:02:03:04" or "01:02:03;04" for drop-frame).
Parameters
fmtThe string format to use (default: SMPTE).
Returns
A Result holding the formatted string. The error code in the returned Result is Error::Ok for all three cases above; an error is only returned when libvtc itself reports an internal failure (extremely rare).
