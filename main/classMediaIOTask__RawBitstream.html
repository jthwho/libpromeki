Detailed Description

MediaIO sink that writes encoded packet payloads verbatim to a file.

MediaIOTask_RawBitstream is the minimum possible backend for capturing the output of a MediaIOTask_VideoEncoder — every MediaPacket on an incoming Frame is appended to the target file exactly as it arrived, with no container framing, no timestamps, no muxing, and no modification of the payload. The resulting file is a raw elementary stream — for H.264 / HEVC output from NVENC, that's Annex-B byte-stream form, which is exactly what ffplay / ffprobe / mpv will accept.

Registered with the extension list "h264" / "h265" / "hevc" so mediaplay's file-path auto-detection routes -d foo.h264 through this backend without requiring the caller to say RawBitstream explicitly.

Only sink mode (MediaIO::Sink) is supported in this first iteration — reading an elementary-stream file back into MediaPackets requires a codec-specific NAL-unit parser that belongs with the decoder work.

Example
// Capture NVENC output as a raw H.264 elementary stream:
MediaIO::Config cfg;
cfg.set(MediaConfig::Type, "RawBitstream");
cfg.set(MediaConfig::Filename, "/tmp/out.h264");
MediaIO *sink = MediaIO::create(cfg);
sink->open(MediaIO::Sink);
