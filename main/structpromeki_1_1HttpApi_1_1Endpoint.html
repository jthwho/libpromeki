Detailed Description

Self-describing HTTP endpoint.

The descriptor carries everything the catalog and OpenAPI generators need: route, method, presentation metadata (title/summary/tags), the parameter list, the success-response shape, and the declared error responses. Endpoints are immutable from the server's point of view — registering the same path/method twice is a programming error.

String promeki::HttpApi::Endpoint::path

Route pattern, relative to the parent HttpApi's prefix.

Paths registered through HttpApi are always interpreted relative to HttpApi::prefix. E.g. an endpoint with path == "/health" registered on an HttpApi with prefix "/api" lands on the underlying server at "/api/health". Stored verbatim in the catalog and /_openapi as the absolute server path, not the relative form, so clients see the URL they should actually call.
