Detailed Description

template<typename T>
class VariantLookup< T >::Key

Lightweight handle for a single registered key name.

A Key wraps the 64-bit FNV-1a hash of a registered name, so lookups that already know which terminal they want skip the per-call string hash. Use literal to build a constexpr handle from a name literal (no registry touch; the hash matches any name later registered via Registrar::scalar etc.). Use find to look up a name that may or may not be registered without inserting it. The regular Key(String) constructor is intentionally omitted because VariantLookup's registration path always flows through the Registrar macros; callers should use literal (compile-time) or find (runtime) to avoid accidentally creating stray Key entries.

Builds a Key from a name literal.

Pure FNV-1a hash; does not touch the runtime registry. The resulting id matches any name later registered via the Registrar, so literal pairs of Keys can be used in constexpr machinery (switch labels, static_assert, non-type template parameters).
