Detailed Description

Abstract base class for stateful audio decoders.

Inverse of AudioEncoder — encoded CompressedAudioPayload access units pushed via submitPayload feed an internal pipeline and PcmAudioPayload frames come back out of receiveAudioPayload. The decoder may buffer several packets before producing its first frame (codec start-up state, silence priming, reordering where present).

Codec-family metadata lives on the AudioCodec wrapper returned by codec(); per-backend facts (supported output formats, …) live on the registered BackendRecord and are reachable via AudioCodec::decoderSupportedOutputs.

Backend registration

Same idiom as AudioEncoder::registerBackend.

Session lifecycle
  1. Resolve a session via AudioCodec::createDecoder.
  2. Optionally call configure.
  3. For each encoded packet, call submitPayload.
  4. After each submit, drain with receiveAudioPayload until it returns a null Ptr.
  5. Call flush when the input stream ends, then drain again.
  6. Destroy the decoder.
Thread Safety
Conditionally thread-safe. Each pipeline thread should own its own decoder instance; concurrent access to a single instance is unsupported.
