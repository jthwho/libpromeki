Detailed Description

Installs handlers for termination signals (Ctrl-C, kill, etc.).

When installed, catches SIGINT, SIGTERM, SIGHUP, and SIGQUIT on POSIX systems (and CTRL_C_EVENT, CTRL_BREAK_EVENT, CTRL_CLOSE_EVENT on Windows) and turns them into a clean application shutdown:

  1. Application::quit is called with exit code 128+signo so any pump loop that polls Application::shouldQuit observes the quit request.
  2. A QuitItem is posted to Application::mainEventLoop so any thread blocked in EventLoop::exec wakes immediately with the same exit code.

On POSIX the handler uses the classic self-pipe trick: a sigaction hook forwards the incoming signal number as a single byte through a pipe, and a dedicated thread (named promeki-signals) reads the pipe and invokes the rest of the quit logic in a fully normal thread context. Only the one byte write goes through the async-signal handler, so no locks are taken and no allocation happens inside the signal path. Because the delivery mechanism is a process-wide sigaction rather than sigwait, the handler works correctly regardless of which thread the kernel picks to receive the signal or the order in which the application's threads are spawned.

Double-signal force-exit
By default, a second delivery of any caught termination signal calls std::_Exit(128+signo) so an unresponsive event loop can still be killed with a second Ctrl-C. Set LibraryOptions::SignalDoubleTapExit to false to disable this behaviour — useful for command-line tools that want a single Ctrl-C to abort immediately, or for applications that want to handle multiple signals themselves.

Controlled by LibraryOptions::TerminationSignalHandler (default true) and wired into Application automatically.

Example
// Automatic via Application (default):
int main(int argc, char **argv) {
Application app(argc, argv);
// SIGINT/SIGTERM/... now translate to Application::quit()
return Application::mainEventLoop()->exec();
}
// Manual control:
SignalHandler::install();
// ...
SignalHandler::uninstall();

◆ install()

static void SignalHandler::install ( )
static

Installs termination signal handlers.

On POSIX, creates a self-pipe, registers an async sigaction handler for SIGINT/SIGTERM/SIGHUP/SIGQUIT that writes the signal number to the pipe, and starts the dedicated reader thread that runs the rest of the quit logic in normal context.

On Windows, registers a SetConsoleCtrlHandler callback.

Safe to call multiple times — subsequent calls are no-ops while the handler is already installed.

◆ uninstall()

static void SignalHandler::uninstall ( )
static

Uninstalls the termination signal handlers.

On POSIX, restores the previous sigaction dispositions, pokes the reader thread with a sentinel byte, joins it, and closes the self-pipe. On Windows, removes the SetConsoleCtrlHandler callback.

Safe to call when the handler is not installed.

Warning
Must not be called from the reader thread itself.
