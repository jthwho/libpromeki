Detailed Description

MediaIOTask backend for QuickTime / ISO-BMFF container files.

Wraps the QuickTime engine to provide read and write access to .mov / .mp4 / .m4v container files through the MediaIO interface. Compressed sample bytes flow through the Frame as compressed Image objects (created via Image::fromCompressedData), with downstream consumers responsible for decoding via whichever codec implementation they prefer. Uncompressed video tracks (2vuy / v210 / etc.) are wrapped as raster Image objects.

Audio

PCM audio tracks are read and written. On read, each Frame carries an Audio object containing the PCM samples that correspond to the video frame duration; compressed (e.g. AAC) audio tracks are surfaced as compressed Audio objects created via Audio::fromCompressedData. On write, Frame audio is accumulated in an AudioBuffer FIFO and flushed to the engine writer in per-video-frame-aligned chunks. Source audio that differs from the on-disk storage format (e.g. float32 input → s16 little-endian on disk) is converted by the FIFO on push. The compressed audio write path is not yet implemented.

Probe

The format probe checks for an ftyp box in the first 16 bytes of the device and verifies the major brand against a small set of recognized values (qt, isom, mp41, mp42, iso2 — iso8, f4v).
