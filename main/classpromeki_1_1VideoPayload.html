Detailed Description

Abstract base for any video payload — compressed or uncompressed.

VideoPayload binds a MediaPayload's plane list to an ImageDesc that describes the video's format, geometry, scan mode, colour interpretation, and per-stream metadata. The descriptor is the single source of truth for "what is this video?"; concrete leaves (UncompressedVideoPayload and CompressedVideoPayload) extend this base with per-payload state that only makes sense for their side of the compressed / uncompressed split.

For compressed video the descriptor's PixelFormat is a compressed entry (e.g. PixelFormat::H264, PixelFormat::HEVC, PixelFormat::JPEG_XS_*); for uncompressed video it is a raster pixel format. A few structural ImageDesc fields (line padding and alignment) are meaningless on the compressed side, which is accepted as a harmless consequence of keeping a single descriptor type across both families.

VideoPayload is abstract — _promeki_clone and kind remain pure-virtual so the concrete leaves must supply a covariant clone. Subclassing concrete leaves for codec-specific specializations (a future NALBitstreamPayload for H.264 / HEVC access units, or a ProResPayload carrying frame-type-A/B metadata) is an intentional extension point.

Thread Safety
Conditionally thread-safe — same contract as MediaPayload. VideoPayload::Ptr is safe to hand off across threads (atomic refcount); concurrent mutation of a single instance must be externally synchronized.

◆ duration()

Duration promeki::VideoPayload::duration ( ) const
inlineoverridevirtual

Returns the per-payload wall-clock duration, or a zero Duration when unset.

Video payloads have no intrinsic rate — the enclosing pipeline (MediaIO / frame-rate converter) is the one that knows how long a frame represents. A zero-valued duration is the "not yet stamped" sentinel; MediaIO treats that case as a fill site and assigns one frame of the session rate.

Video payloads always support a duration — always returns true.

hasDuration is a type-level predicate that answers "is a duration meaningful for this payload kind?" rather than "has one been assigned?" Callers that want the latter should test duration().isZero().

Forwards to the descriptor's metadata.

Every video payload keeps its per-stream metadata (FrameRate, colorimetry, origination, …) on the ImageDesc that already travels with it. Implementing MediaPayload::metadata by forwarding here keeps one authoritative store per payload and lets generic consumers reach it uniformly via the base API.
